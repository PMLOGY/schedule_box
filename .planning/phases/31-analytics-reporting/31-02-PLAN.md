**1. GET /api/v1/analytics/employees:**
- Pattern: createRouteHandler, requiresAuth: true, requiredPermissions: [PERMISSIONS.EMPLOYEES_MANAGE]
- Tenant isolation via findCompanyId(userSub)
- Zod query schema: `days` (1-365, default 30)
- Query: SELECT employees.id, employees.uuid, employees.name, COUNT(bookings.id) as bookingCount, SUM(bookings.price - bookings.discountAmount) as totalRevenue FROM employees LEFT JOIN bookings ON bookings.employeeId = employees.id AND bookings.status = 'completed' AND bookings.startTime >= daysAgo AND bookings.deletedAt IS NULL WHERE employees.companyId = X AND employees.isActive = true AND employees.deletedAt IS NULL GROUP BY employees.id ORDER BY bookingCount DESC
- For occupancy approximation: calculate `occupancyPercent` per employee as `(bookingCount * avgServiceDuration) / (workingDaysInPeriod * 480)` where:
  - avgServiceDuration = average duration_minutes of the services this employee performed (from bookings JOIN services)
  - workingDaysInPeriod = min(days, business days approximation: days * 5/7)
  - 480 = 8 hours working day in minutes
  - Cap at 100%
- If no bookings for employee, occupancyPercent = 0
- Return: `Array<{ employeeId: string (uuid), employeeName: string, bookingCount: number, totalRevenue: number, occupancyPercent: number }>`

**1. Create analytics-scheduler.ts** following the exact pattern of billing-scheduler.ts:

```typescript
// Structure:
// - ANALYTICS_QUEUE_NAME = 'analytics-snapshots'
// - processAnalyticsSnapshots() function that:
//   1. Gets all active companies: SELECT id FROM companies WHERE isActive = true AND deletedAt IS NULL (companies table has no deletedAt, use isActive)
//   2. For each company, computes today's snapshot:
//      - Query bookings WHERE companyId = X AND DATE(startTime) = today
//      - COUNT total, completed, cancelled, no_show
//      - SUM revenue (price - discountAmount) for completed
//      - COUNT DISTINCT customerId for uniqueCustomers
//      - COUNT new customers (customers.createdAt = today AND companyId = X)
//      - AVG booking value = totalRevenue / completedBookings
//      - Top service by count
//   3. UPSERT into analytics_snapshots (ON CONFLICT company_id, snapshot_date DO UPDATE)
// - startAnalyticsScheduler() exports queue + worker
//   - Uses upsertJobScheduler with cron '0 * * * *' (every hour on the hour)
//   - Worker concurrency: 1
//   - Job attempts: 2, backoff: exponential 30s
```

Import from `@schedulebox/database`: `db, companies, bookings, customers, analyticsSnapshots`
Import from `drizzle-orm`: `eq, and, sql, count, isNull`

The UPSERT uses Drizzle's `onConflictDoUpdate` with target `[analyticsSnapshots.companyId, analyticsSnapshots.snapshotDate]`.

Log start/end with duration like billing-scheduler does.
