Create two new API routes following the exact pattern of existing analytics routes (createRouteHandler, PERMISSIONS.BOOKINGS_READ, findCompanyId, Zod query schema with days param).

**GET /api/v1/analytics/payment-methods:**
- Query `payments` table WHERE companyId = X AND status = 'paid' AND createdAt >= daysAgo
- GROUP BY `gateway` column
- Return: `Array<{ gateway: string, count: number, totalAmount: number }>` — shaped for a PieChart
- Include percentage calculation: `(count / totalCount * 100).toFixed(1)`

**GET /api/v1/analytics/top-services:**
- Query `bookings` JOIN `services` WHERE bookings.companyId = X AND bookings.status = 'completed' AND bookings.startTime >= daysAgo
- GROUP BY serviceId, ORDER BY SUM(price - discountAmount) DESC, LIMIT 10
- Return: `Array<{ serviceId: number, serviceName: string, bookingCount: number, totalRevenue: number }>` — shaped for a BarChart
- Use `services.name` for the service name, `services.uuid` for public ID

Both routes: use `z.coerce.number().int().min(1).max(365).default(30)` for the days query param. Use `PERMISSIONS.BOOKINGS_READ` (same as existing analytics routes). Format all numeric values as numbers (not strings) in the response.

<task type="auto">
  <name>Task 2: Booking detail and customer retention API routes (peak hours, cancellations, retention)</name>
  <files>

  <action>
Create three new API routes following the same pattern.

**GET /api/v1/analytics/peak-hours:**
- Query `bookings` WHERE companyId = X AND status IN ('completed', 'confirmed', 'pending') AND startTime >= daysAgo AND deletedAt IS NULL
- Use `EXTRACT(DOW FROM start_time)` as `dayOfWeek` (0=Sun..6=Sat) and `EXTRACT(HOUR FROM start_time)` as `hour`
- GROUP BY dayOfWeek, hour; COUNT(*) as bookingCount
- Return: `Array<{ dayOfWeek: number, hour: number, count: number }>` — a 7x24 sparse matrix for heatmap rendering
- The frontend will fill gaps with zeros

**GET /api/v1/analytics/cancellations:**
- Query `bookings` WHERE companyId = X AND startTime >= daysAgo AND deletedAt IS NULL
- GROUP BY `DATE(start_time)` — for each day, count total, cancelled, no_show
- Calculate rates: `cancelRate = cancelled / total`, `noShowRate = noShow / total`
- Return: `Array<{ date: string, total: number, cancelled: number, noShows: number, cancelRate: number, noShowRate: number }>`

**GET /api/v1/analytics/customer-retention:**
- Permission: `PERMISSIONS.CUSTOMERS_READ` (customer data)
- Repeat booking rate: Query `customers` WHERE companyId = X AND deletedAt IS NULL, count those with totalBookings > 1 vs total. Return `{ repeatRate: number, totalCustomers: number, repeatCustomers: number }`
- Customer churn: Count customers with lastVisitAt older than 90 days from now (dormant/churned). Return `{ churned: number, atRisk: number, active: number }` using the same thresholds as v_customer_metrics (90 days = at_risk, 180 days = dormant/churned)
- CLV distribution: Query customers WHERE companyId = X AND clvPredicted IS NOT NULL, bucket into ranges (0-500, 500-2000, 2000-5000, 5000-10000, 10000+) with COUNT per bucket. Return `Array<{ range: string, count: number }>`
- Combine all three into a single response object: `{ repeatBooking: {...}, churn: {...}, clvDistribution: [...] }`

All routes use the standard Zod days param (default 30). Customer retention uses `PERMISSIONS.CUSTOMERS_READ`.
  </action>
