  <action>
Create GET /api/v1/admin/analytics that returns SaaS health metrics. This route MUST enforce admin-only access.

**Authorization pattern:**
- Use createRouteHandler with requiresAuth: true
- Inside handler, check `user.role === 'admin'`. If not, throw ForbiddenError('Admin access required'). Import ForbiddenError from '@schedulebox/shared'.
- Do NOT use findCompanyId — admin routes are cross-tenant.

**Zod query schema:** `days` (1-365, default 30) for signup trend period.

**Response shape:**
```typescript
{
  mrr: number,           // Monthly Recurring Revenue in CZK
  arr: number,           // Annual Run Rate (MRR * 12)
  churnRate: number,     // % of subscriptions that expired/cancelled in period
  activeCompanies: number,
  totalCompanies: number,
  planDistribution: Array<{ plan: string, count: number, percentage: number }>,
  signupTrend: Array<{ date: string, count: number }>,  // daily new company signups
  mrrByPlan: Array<{ plan: string, mrr: number }>,
}
```

**MRR calculation:**
- Query subscriptions WHERE status = 'active'
- For monthly: SUM(priceAmount)
- For annual: SUM(priceAmount / 12)
- Total MRR = monthly sum + annual monthly equivalent

**Churn rate:**
- Count subscriptions that transitioned to 'expired' or 'cancelled' in the period (check subscription_events with eventType containing 'expired' or 'cancelled' AND createdAt >= daysAgo)
- Divide by total active subscriptions at period start (approximate: current active + churned)

**Plan distribution:**
- Query companies GROUP BY subscriptionPlan WHERE isActive = true
- Calculate percentage per plan

**Signup trend:**
- Query companies WHERE createdAt >= daysAgo GROUP BY DATE(createdAt) ORDER BY date
- Return daily count of new company signups

**MRR by plan:**
- GROUP BY subscriptions.plan WHERE status = 'active', SUM priceAmount (adjusted for annual)

Import from @schedulebox/database: db, companies, subscriptions, subscriptionEvents

<task type="auto">

  <action>
Create GET /api/v1/analytics/organization that returns aggregate analytics across all locations in a franchise owner's organization.

**Authorization pattern:**
- Use createRouteHandler with requiresAuth: true, requiredPermissions: [PERMISSIONS.BOOKINGS_READ]
- Resolve the user's organization: query organization_members WHERE userId = (user's internal ID from users table via UUID) AND role = 'franchise_owner'
- If user is not a franchise_owner in any org, throw ForbiddenError('Organization access required')
- Get all companyIds in the organization: query companies WHERE organizationId = org.id AND isActive = true

**To resolve user internal ID:** query `users` table WHERE uuid = user.sub to get users.id, then use that to find organization_members.

**Zod query schema:** `days` (1-365, default 30)

**Response shape:**
```typescript
{
  organizationId: string,     // org UUID
  organizationName: string,
  totals: {
    totalRevenue: number,
    totalBookings: number,
    completedBookings: number,
    cancelledBookings: number,
    noShows: number,
    uniqueCustomers: number,
  },
  locations: Array<{
    companyId: string,        // company UUID
    companyName: string,
    totalRevenue: number,
    totalBookings: number,
    completedBookings: number,
    occupancyApprox: number,  // same approximation as employee utilization
  }>,
}
```

**Query approach:**
- For each companyId in the org, query v_daily_booking_summary (or bookings table directly) WHERE companyId = X AND bookingDate >= daysAgo
- Aggregate per-location stats and org-level totals
- For occupancyApprox per location: count completed bookings, count active employees, approximate as (bookings * avgDuration) / (employees * workingDays * 480). This is the same V1 approximation documented in RESEARCH.md.
- For uniqueCustomers at org level: COUNT DISTINCT customerId across all bookings in all org companies (note: same customer may exist in multiple locations — this is org-level dedup by customerId, not by email)

Import from @schedulebox/database: db, organizations, organizationMembers, companies, bookings, employees, users, dailyBookingSummary

  </done>
