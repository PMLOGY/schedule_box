**Create `apps/web/app/api/v1/organizations/[id]/dashboard/route.ts`:**

**GET /api/v1/organizations/[id]/dashboard** — Per-location metrics.
- Authenticate user
- Verify user is franchise_owner of org [id] (location_managers should NOT see org-level dashboard)
- Get all company IDs in org via `findOrganizationCompanyIds(orgInternalId)`
- For each company, compute:
  - **bookings_count**: COUNT of bookings in current month WHERE companyId = X
  - **revenue_total**: SUM of payments.amount WHERE companyId = X AND status = 'completed' in current month
  - **occupancy_percent**: approximate as (total booked hours / total available hours) * 100 for current week. Use working_hours to get total available, bookings duration for booked. If too complex, return null and document it as "to be refined in Phase 31 Analytics".
- Also compute org-level aggregates: total bookings across all locations, total revenue, average occupancy

Use a single SQL query with GROUP BY for efficiency:
```sql
SELECT
  c.uuid as company_uuid,
  c.name as company_name,
  COUNT(b.id) as bookings_count,
  COALESCE(SUM(p.amount), 0) as revenue_total
FROM companies c
LEFT JOIN bookings b ON b.company_id = c.id AND b.start_time >= date_trunc('month', NOW())
LEFT JOIN payments p ON p.company_id = c.id AND p.status = 'completed' AND p.created_at >= date_trunc('month', NOW())
WHERE c.organization_id = $1 AND c.is_active = true
GROUP BY c.id, c.uuid, c.name
```

Use Drizzle ORM raw SQL (`sql` template) or query builder. Return array of `LocationMetrics` objects.

Response shape:

**Create `apps/web/app/api/v1/organizations/[id]/customers/route.ts`:**

**GET /api/v1/organizations/[id]/customers** — Cross-location customer search with dedup.
- Authenticate user
- Verify user is franchise_owner of org [id]
- Accept query params: `search` (string), `page` (int), `limit` (int)
- Get all companyIds in org via `findOrganizationCompanyIds(orgInternalId)`
- Query customers across all org companies with email-based deduplication:

Strategy: Use `DISTINCT ON (email)` (PostgreSQL-specific) to deduplicate customers across locations. For customers without email, use phone number. For those without both, treat as unique.

```sql
SELECT DISTINCT ON (COALESCE(email, phone, uuid::text))
  uuid, name, email, phone,
  total_bookings, total_spent, last_visit_at,
  company_id
FROM customers
WHERE company_id = ANY($companyIds)
  AND deleted_at IS NULL
  AND ($search IS NULL OR name ILIKE '%' || $search || '%' OR email ILIKE '%' || $search || '%' OR phone ILIKE '%' || $search || '%')
ORDER BY COALESCE(email, phone, uuid::text), last_visit_at DESC NULLS LAST
LIMIT $limit OFFSET $offset
```

Also include a `locations_visited` count: for each deduped customer (by email), count distinct company_ids they appear in.

Response shape: paginated response matching existing pattern, with added `locations_visited` field per customer.
