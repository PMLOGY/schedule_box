**Create `apps/web/app/api/v1/organizations/route.ts`:**

**GET /api/v1/organizations** — Return the user's organization with locations.
- Authenticate user
- Call `findOrganizationForUser(user.sub)` from org-scope.ts
- If user has no organization, return `{ data: null }`
- If user has organization, query organization with all companies (locations) that have `organizationId` matching
- Return organization details + locations array with company uuid, name, slug, address_city, is_active

**POST /api/v1/organizations** — Create a new organization.
- Authenticate user, require role = 'owner' (only company owners can create orgs)
- Validate body with `createOrganizationSchema` (name)
- Check user doesn't already belong to an organization (call findOrganizationForUser)
- If user already in an org, return ConflictError
- Generate slug from name (lowercase, hyphens, append random suffix)
- Get user's current company_id via `findCompanyId(user.sub)`
- Determine max_locations based on company's subscription plan:
  - free/essential: 1 (reject org creation — return PaymentRequiredError with message "Multi-location requires Growth or AI-Powered plan")
  - growth: 3
  - ai_powered: 10
- Insert into `organizations` table with ownerUserId = user's internal ID, maxLocations from plan
- Insert into `organization_members` with role = 'franchise_owner', companyId = null (all locations)
- Update user's current company to set `organizationId` = new org ID
- Return 201 with organization data

**Create `apps/web/app/api/v1/organizations/[id]/route.ts`:**

**GET /api/v1/organizations/[id]** — Get organization details.
- Authenticate user
- Validate params: id = organization UUID
- Verify user is a member of this organization (query organization_members)
- Return organization with full location list and member count

**PUT /api/v1/organizations/[id]** — Update organization name.
- Authenticate user
- Verify user is franchise_owner of this org
- Update name (and regenerate slug if name changed)
- Return updated organization

Both route files use `createRouteHandler` pattern. Use `paramsSchema` for the [id] parameter (z.object({ id: z.string().uuid() })).

**Create `apps/web/app/api/v1/organizations/[id]/locations/route.ts`:**

**GET /api/v1/organizations/[id]/locations** — List all locations in the org.
- Authenticate user
- Verify user is member of org [id] (query organization_members for user and org)
- Query companies WHERE organizationId = org's internal ID
- Return array of locations with uuid, name, slug, email, phone, address_*, is_active, created_at

**POST /api/v1/organizations/[id]/locations** — Add a new location.
- Authenticate user
- Verify user is franchise_owner of org [id] (location_managers cannot add locations)
- Validate body with `addLocationSchema` (name, slug, email, phone, address_*)
- Check slug uniqueness across all companies
- Count existing locations for this org. If count >= org.maxLocations, return PaymentRequiredError('Location limit reached for your plan. Upgrade to add more locations.')
- Insert into companies: name, slug, email, phone, address_street, address_city, address_zip, organizationId = org's internal ID, subscriptionPlan = same as the org owner's company plan
- Return 201 with the new company/location data

**Create `apps/web/app/api/v1/organizations/[id]/locations/[locationId]/route.ts`:**

**PUT /api/v1/organizations/[id]/locations/[locationId]** — Update location details.
- Authenticate user
- Verify user is franchise_owner of org [id]
- Validate body with `updateLocationSchema` (name, email, phone, address_*, is_active)
- locationId is the company UUID
- Verify the company belongs to the org
- Update company fields
- Return updated location

**DELETE /api/v1/organizations/[id]/locations/[locationId]** — Deactivate location.
- Authenticate user
- Verify user is franchise_owner of org [id]
- locationId is the company UUID
- Verify the company belongs to the org
- Set company.isActive = false (soft disable — NOT delete)
- DO NOT delete any data — historical bookings, customers, payments must all remain
- Return 200 with { deactivated: true, company_uuid: locationId }

NOTE: For all endpoints, the [id] param is the organization UUID and [locationId] is the company UUID. Use paramsSchema for validation.

**Create `apps/web/app/api/v1/organizations/[id]/members/route.ts`:**

**GET /api/v1/organizations/[id]/members** — List org members.
- Authenticate user
- Verify user is member of org [id]
- Query organization_members joined with users (for name, email) and optionally companies (for location name)
- Return array: { user_uuid, user_name, user_email, role, company_uuid (if location_manager), company_name (if location_manager), created_at }

**POST /api/v1/organizations/[id]/members** — Add a member.
- Authenticate user
- Verify user is franchise_owner of org [id] (only franchise owners can add members)
- Validate body with `addMemberSchema` (user_email, role, company_uuid)
- If role is 'location_manager', company_uuid is required. Validate it belongs to this org.
- If role is 'franchise_owner', company_uuid must be null (or ignored if provided — franchise owners have access to all locations)
- Find user by email. If not found, return NotFoundError('User with this email not found. They must register first.')
- Check if user is already a member of this org. If so, return ConflictError.
- Insert into organization_members
- Return 201 with member data

**DELETE /api/v1/organizations/[id]/members** — Remove a member (via body with user_uuid).
- Authenticate user
- Verify user is franchise_owner
- Cannot remove yourself (the org owner)
- Delete the organization_members record
- Return 204

Use `createRouteHandler` with `bodySchema` for POST and DELETE. Use `paramsSchema` for the [id] parameter.
