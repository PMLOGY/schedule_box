<task type="auto">
  <name>Task 1: Install next-intl and configure i18n middleware</name>
  <files>
    apps/web/package.json
    apps/web/i18n/request.ts
    apps/web/middleware.ts
    apps/web/next.config.mjs
  </files>
  <action>
    1. Install next-intl:
       ```bash
       cd apps/web && pnpm add next-intl
       ```

    2. Create `apps/web/i18n/request.ts`:
       ```typescript
       import { getRequestConfig } from 'next-intl/server';

       export default getRequestConfig(async ({ locale }) => ({
         messages: (await import(`../messages/${locale}.json`)).default,
       }));
       ```

    3. Create `apps/web/middleware.ts`:
       - Import createMiddleware from 'next-intl/middleware'
       - Configure locales: ['cs', 'sk', 'en'], defaultLocale: 'cs'
       - Set localePrefix to 'as-needed' (no /cs prefix for default locale)
       - Export config.matcher to exclude: api routes, _next, static files, images
       - Matcher: ['/((?!api|_next|.*\\..*).*)']

    4. Update `apps/web/next.config.mjs` to add next-intl plugin:
       - Import createNextIntlPlugin from 'next-intl/plugin'
       - Wrap existing config with withNextIntl('./i18n/request.ts')

    IMPORTANT: The middleware.ts file must NOT conflict with any existing middleware. Check if one exists first. If apps/web/middleware.ts already exists, merge the next-intl middleware into it.

    NOTE on routing: We are using next-intl WITHOUT the [locale] route segment approach. Instead, use the "middleware-based" approach where next-intl detects locale from cookies/headers and provides translations without changing URL structure. This avoids restructuring the entire app directory. The translations are loaded via the middleware and made available to components.
  </action>
  <verify>
    - `pnpm type-check --filter @schedulebox/web` passes
    - apps/web/middleware.ts exists with next-intl createMiddleware
    - apps/web/i18n/request.ts exports getRequestConfig
    - next.config.mjs includes withNextIntl wrapper
  </verify>
  <done>
    next-intl installed and configured with middleware-based locale detection. Czech is default locale, Slovak and English supported. No URL restructuring needed.
  </done>
</task>
