  <action>
    1. Install TanStack Table:
       ```bash
       cd apps/web && pnpm add @tanstack/react-table
       ```

    2. Create `apps/web/components/shared/data-table.tsx` (Client Component):
       - 'use client'
       - Generic component: DataTable<TData, TValue>
       - Props: columns (ColumnDef[]), data (TData[]), isLoading? (boolean), isError? (boolean), errorMessage? (string), onRetry? (() => void), emptyMessage? (string), pageSize? (number, default 10), onRowClick? (row => void)
       - Use useReactTable with: getCoreRowModel, getPaginationRowModel, getSortedRowModel
       - Render using shadcn/ui Table components (Table, TableHeader, TableBody, TableRow, TableHead, TableCell)
       - Sortable column headers: click to toggle asc/desc, show ArrowUpDown/ArrowUp/ArrowDown icon
       - Pagination footer: Previous/Next buttons, "Page X of Y" text, rows per page selector
       - When isLoading: show skeleton rows (3-5 rows of Skeleton components)
       - When isError: show error state with AlertTriangle icon, errorMessage (default: "Failed to load data"), and a "Try again" Button that calls onRetry. This covers UI-09 for API error states from TanStack Query.
       - When data empty (and not loading/error): show EmptyState component with emptyMessage
       - **Priority order:** isLoading > isError > empty > data. Only one state shown at a time.
       - Use useTranslations('table') for pagination labels and error/empty default text

    3. Create `apps/web/components/shared/empty-state.tsx`:
       - Props: icon? (LucideIcon), title (string), description? (string), action? ({ label: string, onClick: () => void } | { label: string, href: string })
       - Layout: centered flex column, icon (h-12 w-12, text-muted-foreground), title (text-lg font-semibold), description (text-sm text-muted-foreground), optional action Button
       - Padding: py-12
       - No 'use client' unless action onClick requires it (use 'use client' to be safe since action might have onClick)

    4. Create `apps/web/components/shared/loading-skeleton.tsx`:
       - Export multiple skeleton variants:
         - CardSkeleton: mimics StatCard (icon, title line, value line, trend line)
         - TableSkeleton: mimics DataTable (header row + N body rows with random-width skeletons)
         - PageSkeleton: full page with header + card grid + table
         - FormSkeleton: mimics form fields (label + input pairs)
       - Each uses Skeleton from shadcn/ui
       - Props for each: count? (number of items to show)
       - No 'use client' needed (pure presentational)

    5. Create `apps/web/components/shared/error-boundary.tsx` (Client Component):
       - 'use client'
       - React Error Boundary class component (React 18 doesn't have hooks-based error boundary)
       - Props: children, fallback? (ReactNode)
       - State: hasError (boolean), error (Error | null)
       - getDerivedStateFromError: set hasError=true
       - componentDidCatch: log error to console
       - Default fallback: centered card with warning icon, "Something went wrong" title, error message (in dev), "Try again" button that resets state
       - Use useTranslations inside a wrapper functional component for the fallback UI
  </action>

    - ErrorBoundary catches errors and shows recovery UI
