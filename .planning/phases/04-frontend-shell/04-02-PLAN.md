  <name>Task 1: Install state management dependencies and create Zustand stores</name>
  <files>
    apps/web/package.json
    apps/web/stores/auth.store.ts
    apps/web/stores/ui.store.ts
    apps/web/stores/calendar.store.ts
  </files>
  <action>
    1. Install dependencies in apps/web:
       ```bash
       cd apps/web
       pnpm add @tanstack/react-query zustand
       pnpm add -D @tanstack/react-query-devtools
       ```

    2. Create `apps/web/stores/auth.store.ts`:
       - Interface AuthState: user (User | null), accessToken (string | null), isAuthenticated (boolean)
       - User type: { id: string (UUID), email: string, firstName: string, lastName: string, role: string, companyId: string (UUID), companyName: string }
       - Actions: login(email, password) calls POST /api/v1/auth/login, stores user + accessToken, sets isAuthenticated=true
       - Actions: logout() calls POST /api/v1/auth/logout, clears state
       - Actions: setUser(user), setAccessToken(token), refreshToken() calls POST /api/v1/auth/refresh
       - Use persist middleware with name 'schedulebox-auth', partialize to only persist user (NOT tokens — tokens go in httpOnly cookies from API, accessToken in memory only)
       - Export useAuthStore

    3. Create `apps/web/stores/ui.store.ts`:
       - Interface UIState: sidebarCollapsed (boolean), sidebarMobileOpen (boolean), activeModal (string | null), modalData (unknown)
       - Actions: toggleSidebar(), setSidebarCollapsed(collapsed), toggleMobileSidebar(), closeMobileSidebar()
       - Actions: openModal(modalId, data?), closeModal()
       - Use persist middleware with name 'schedulebox-ui', partialize to only persist sidebarCollapsed
       - Export useUIStore

    4. Create `apps/web/stores/calendar.store.ts`:
       - Interface CalendarState: view ('resourceTimelineDay' | 'resourceTimelineWeek' | 'dayGridMonth'), selectedDate (Date), selectedEmployeeIds (string[]), showCancelled (boolean)
       - Actions: setView(view), setSelectedDate(date), toggleEmployee(id), setEmployeeFilter(ids), toggleShowCancelled()
       - NO persist middleware (calendar state resets on page refresh)
       - Export useCalendarStore

    IMPORTANT: All stores must use 'use client' is NOT needed in store files — they are plain TS modules. The 'use client' is only needed in React components that use them.
  </action>
  <verify>
    - `pnpm type-check --filter @schedulebox/web` passes
    - auth.store.ts exports useAuthStore with login, logout, setUser, refreshToken
    - ui.store.ts exports useUIStore with toggleSidebar, openModal, closeModal
    - calendar.store.ts exports useCalendarStore with setView, setSelectedDate
  </verify>
  <done>
    Three Zustand stores created: auth (user/tokens/login/logout), UI (sidebar/modal state), calendar (view/date/filters). All type-safe with TypeScript interfaces.
  </done>

<task type="auto">
  <name>Task 2: Create API client and TanStack Query provider</name>
  <files>
    apps/web/lib/api-client.ts

  <action>
    1. Create `apps/web/lib/api-client.ts`:
       - Class ApiClient with private request<T>(endpoint, options) method
       - Reads accessToken from useAuthStore.getState().accessToken
       - Base URL: process.env.NEXT_PUBLIC_API_URL || '/api/v1'
       - Auto-sets Content-Type: application/json and Authorization: Bearer {token}
       - On 401 response: attempt token refresh via useAuthStore.getState().refreshToken(), retry once, if still 401 call logout()
       - Methods: get<T>(endpoint, params?), post<T>(endpoint, data?), put<T>(endpoint, data?), patch<T>(endpoint, data?), delete<T>(endpoint)
       - For GET with params: construct URLSearchParams and append to URL
       - Error handling: parse error response body, throw structured error with { code, message, details, statusCode }
       - Export singleton: `export const apiClient = new ApiClient();`

    2. Create `apps/web/lib/query-client.ts`:
       - Export function `createQueryClient()` that returns new QueryClient with:
         - defaultOptions.queries.staleTime = 60 * 1000 (1 minute)
         - defaultOptions.queries.retry = 1
         - defaultOptions.queries.refetchOnWindowFocus = false
         - defaultOptions.mutations.onError: toast.error if sonner available (defer, just console.error for now)

    3. Create `apps/web/app/providers.tsx` (Client Component):
       - 'use client' directive
       - Use useState to create QueryClient (prevents sharing across requests)
       - Wrap children in QueryClientProvider
       - Include ReactQueryDevtools (initialIsOpen: false) in development only
       - Export function Providers({ children })

    4. NOTE: Do NOT update app/layout.tsx yet — that happens in Plan 05 when the full app shell layout is created. The providers.tsx is just prepared here.
  </action>

    - providers.tsx creates QueryClient inside useState (not at module level)
  </verify>

  </done>
