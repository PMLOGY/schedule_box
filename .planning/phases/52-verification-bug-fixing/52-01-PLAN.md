<task type="auto">
  <name>Task 1: Boot dev server and capture all startup errors</name>
  <files>apps/web/next.config.js, packages/database/src/index.ts, apps/web/lib/env.ts</files>
  <action>
1. Run `pnpm dev` from the project root and capture the first 60 seconds of output. Look for:
   - TypeScript compilation errors (red text, "Type error", "Cannot find module")
   - Database connection failures (Neon connection refused, invalid credentials, SSL errors)
   - Redis connection failures (Upstash URL invalid, unauthorized)
   - Missing environment variables (undefined, NEXT_PUBLIC_* missing)
   - Import resolution errors (module not found, circular dependency)
   - Next.js config errors (invalid next.config.js options)

2. For each error found, fix it at the source:
   - Missing env var: Add to .env.local with correct value from .env.example, or add a graceful fallback if it is optional
   - Import error: Fix the import path or install the missing package
   - DB connection: Fix connection string format, ensure Neon pooled/unpooled URLs are correct
   - Redis: Ensure UPSTASH_REDIS_REST_URL and UPSTASH_REDIS_REST_TOKEN are set
   - Next.js config: Fix deprecated options, remove invalid entries

3. After fixing, restart `pnpm dev` and confirm the server reaches "ready" with no errors.

4. Open http://localhost:3000 (or whatever port it binds to) in a headless check — use `curl -s -o /dev/null -w "%{http_code}" http://localhost:3000` to verify 200 response.

IMPORTANT: Do NOT skip errors or comment out code to make it compile. Fix the actual issue. If a service is unavailable (e.g., no Twilio credentials), ensure the code gracefully handles the missing config rather than crashing.
  </action>
  <verify>
    <automated>cd D:/Project/ScheduleBox && timeout 60 pnpm dev 2>&1 | head -100</automated>
  </verify>
  <done>Dev server starts without TypeScript errors, connection failures, or missing module errors. Server reaches "ready" state and responds with HTTP 200 on localhost.</done>
