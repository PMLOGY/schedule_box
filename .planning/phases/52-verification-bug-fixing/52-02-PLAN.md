  <action>
1. Navigate to /cs/register in the browser (use curl for API testing, read page source for SSR issues).

2. Test registration API directly:
   ```
   curl -X POST http://localhost:3000/api/v1/auth/register \
     -H "Content-Type: application/json" \
     -d '{"email":"verify-test@example.com","password":"TestPass123!","name":"Test Verify","companyName":"Verify Salon"}'
   ```
   Expected: 201 with user object + JWT token/cookie. If error, read the route handler and fix:
   - DB insert failure: Check schema matches, company_id generation, unique constraints
   - Validation error: Check Zod schema matches the request body
   - Auth error: Check JWT signing, cookie setting

3. After registration, verify the redirect sends to the onboarding wizard. Test the onboarding page loads:
   - GET /cs/dashboard/onboarding (or wherever onboarding lives) with auth cookie
   - Each wizard step should render without blank screen or JS error

   - Step 1: Company details (name, address, phone) — POST the data, verify it saves
   - Step 2: Business hours / working schedule — POST schedule data
   - Step 3: First service creation — POST service data
   - Step 4: Completion / redirect to dashboard
   For each step, check the API response and fix any failures.

5. Common bugs to watch for:
   - Onboarding state not tracked (user gets stuck in a loop)
   - Missing translations causing blank labels
   - Form validation too strict (rejects valid CZ phone numbers, ICs)
   - Wizard step components not imported or lazy-loaded incorrectly
   - Missing company_id in API calls (RLS violation)
