  <action>
    **globalSetup.ts** — Container lifecycle and schema setup:

    1. Import `PostgreSqlContainer`, `RedisContainer`, `RabbitMQContainer` from their respective @testcontainers modules
    2. Import `type { GlobalSetupContext } from 'vitest/node'`
    3. Import `drizzle` from `drizzle-orm/postgres-js`, `migrate` from `drizzle-orm/postgres-js/migrator`, `postgres` from `postgres`
    4. Import `readFileSync` from `node:fs`, `resolve` from `node:path`

    5. `setup(project: GlobalSetupContext)`:
       - Start all 3 containers in parallel via `Promise.all`:
         - PostgreSqlContainer('postgres:16-alpine').withDatabase('schedulebox_test').withUsername('test_superuser').start()
         - RedisContainer('redis:7-alpine').start()
         - RabbitMQContainer('rabbitmq:3.13-management-alpine').start()
       - Get `databaseUrl` from `pgContainer.getConnectionUri()`
       - Run Drizzle migrations: `migrate(drizzle(postgres(databaseUrl, { max: 1 })), { migrationsFolder: resolve(__dirname, '../../packages/database/src/migrations') })`
       - Close migration client after migrate completes
       - Apply ALL SQL files from apply-sql.ts in order (read the file list from that source). Use `readFileSync` + `sqlClient.unsafe()` for each file:
         1. `packages/database/src/rls/functions.sql`
         2. `packages/database/src/functions/updated-at.sql`
         3. `packages/database/src/functions/customer-metrics.sql`
         4. `packages/database/src/functions/marketplace-rating.sql`
         5. `packages/database/src/functions/coupon-usage.sql`
         6. `packages/database/src/functions/double-booking.sql`
         7. `packages/database/src/functions/soft-delete.sql`
         8. `packages/database/src/functions/deferred-fks.sql`
         9. `packages/database/src/functions/audit-trail.sql`
         10. `packages/database/src/functions/composite-indexes.sql`
         11. `packages/database/src/rls/policies.sql`
       - Close SQL client after all files applied
       - Create non-superuser role for RLS tests:
         ```sql
         DO $$ BEGIN
           IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'test_app') THEN
             CREATE ROLE test_app LOGIN PASSWORD 'test_app_pwd';
           END IF;
         END $$;
         GRANT ALL ON ALL TABLES IN SCHEMA public TO test_app;
         GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO test_app;
         GRANT EXECUTE ON ALL FUNCTIONS IN SCHEMA public TO test_app;
         ```
       - Provide connection strings via `project.provide()`:
         - `'DATABASE_URL'` = superuser URL (for seeding, bypasses RLS)
         - `'DATABASE_URL_APP'` = URL with test_app credentials (for RLS tests)
           Construct by replacing username and password in the superuser URI
         - `'REDIS_URL'` = redisContainer.getConnectionUrl()
         - `'RABBITMQ_URL'` = rabbitContainer.getAmqpUrl()
       - Add console.log statements for progress: "Starting containers...", "Running migrations...", "Applying SQL files...", "Ready."

    6. `teardown()`: Stop all 3 containers in parallel via Promise.all

    7. Declare the provide keys for TypeScript. Either add `declare module 'vitest'` augmentation at top of globalSetup.ts:
       ```typescript
       declare module 'vitest' {
         export interface ProvidedContext {
           DATABASE_URL: string;
           DATABASE_URL_APP: string;
           REDIS_URL: string;
           RABBITMQ_URL: string;
         }
       }
       ```

    **test-db.ts** — DB connection factory and helpers:

    1. `createTestDb()` — Returns `{ client, db }` using superuser DATABASE_URL from `inject('DATABASE_URL')`. Use `postgres(url, { max: 10 })` and `drizzle(client, { schema })` importing `* as schema from '@schedulebox/database'`.

    2. `createTestAppDb()` — Same but uses DATABASE_URL_APP (non-superuser for RLS tests). Returns `{ client, db }`.

    3. `setRlsContext(client, { companyId, userRole, userId })` — Runs `SET LOCAL app.company_id`, `SET LOCAL app.user_role`, `SET LOCAL app.user_id` on the given client. Use `SET LOCAL` (not `SET`) so context is scoped to the transaction.

    4. `truncateAllTables(client)` — Runs `TRUNCATE companies CASCADE` which cascades to all tenant tables (companies is the root FK reference). This is the fastest cleanup method since all tables ultimately reference companies.

    **seed-helpers.ts** — Test data factories:

    Import Drizzle schema from `@schedulebox/database`. Each factory takes a `db` (Drizzle instance) and optional overrides, returns the inserted row.

    1. `seedCompany(db, overrides?)` — Insert into companies with defaults: `{ name: 'Test Company', slug: 'test-company', email: 'test@example.com', ...overrides }`. Return full row via `.returning()`.

    2. `seedUser(db, { companyId, ...overrides })` — Insert into users with defaults: `{ companyId, email: 'user@test.com', passwordHash: 'hashed', role: 'owner', ...overrides }`. Return full row.

    3. `seedService(db, { companyId, ...overrides })` — Insert into services with defaults: `{ companyId, name: 'Test Service', durationMinutes: 60, price: '500.00', currency: 'CZK', isActive: true, ...overrides }`. Return full row.

    4. `seedEmployee(db, { companyId, ...overrides })` — Insert into employees with defaults: `{ companyId, name: 'Test Employee', email: 'emp@test.com', isActive: true, ...overrides }`. Return full row.

    5. `seedEmployeeService(db, { employeeId, serviceId })` — Insert into employee_services junction table.

    6. `seedCustomer(db, { companyId, ...overrides })` — Insert into customers with defaults: `{ companyId, name: 'Test Customer', email: 'customer@test.com', ...overrides }`. Return full row.

    7. `seedBooking(db, { companyId, customerId, serviceId, employeeId, startTime, endTime, ...overrides })` — Insert into bookings with defaults for status ('pending'), source ('online'), price ('500.00'), currency ('CZK'). Return full row.

    All factories use `.values({...}).returning()` to get the inserted row back. Use unique-ish defaults with Date.now() suffix for emails to avoid unique constraint violations when called multiple times.
