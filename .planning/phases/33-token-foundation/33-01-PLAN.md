<task type="auto">
  <name>Task 2: Responsive degradation and accessibility fallbacks</name>
  <files>apps/web/app/globals.css</files>
  <action>
Add three blocks to the END of globals.css (after the gradient mesh dark mode overrides from Task 1):

**1. Responsive blur degradation (DSYS-06):**
```css
@media (max-width: 767px) {
  .glass-surface {
    backdrop-filter: blur(8px);
    -webkit-backdrop-filter: blur(8px);
  }
  .glass-surface-subtle {
    backdrop-filter: blur(4px);
    -webkit-backdrop-filter: blur(4px);
  }
  .glass-surface-heavy {
    backdrop-filter: blur(12px);
    -webkit-backdrop-filter: blur(12px);
  }
}
```
These override the plugin-registered blur values on mobile. Values are HARDCODED (not CSS variables) because -webkit-backdrop-filter cannot use variables (Safari MDN#25914). Both prefixed and unprefixed forms must be present because autoprefixer does NOT process Tailwind plugin CSS.

**2. prefers-reduced-transparency fallback (DSYS-07):**
```css
@media (prefers-reduced-transparency: reduce) {
  .glass-surface,
  .glass-surface-subtle,
  .glass-surface-heavy {
    backdrop-filter: none;
    -webkit-backdrop-filter: none;
    background: hsl(var(--card));
    border: 1px solid hsl(var(--border));
    box-shadow: var(--shadow-md);
  }
}
```
This reuses existing shadcn tokens (--card, --border, --shadow-md) for the opaque fallback. No new tokens needed.

**3. WCAG scrim via ::before pseudo-element (DSYS-07):**
```css
@supports (backdrop-filter: blur(1px)) or (-webkit-backdrop-filter: blur(1px)) {
  .glass-surface,
  .glass-surface-subtle,
  .glass-surface-heavy {
    position: relative;
  }

  .glass-surface::before,
  .glass-surface-subtle::before,
  .glass-surface-heavy::before {
    content: '';
    position: absolute;
    inset: 0;
    border-radius: inherit;
    background: rgba(255, 255, 255, 0.04);
    pointer-events: none;
    z-index: 0;
  }

  .dark .glass-surface::before,
  .dark .glass-surface-subtle::before,
  .dark .glass-surface-heavy::before {
    background: rgba(0, 0, 0, 0.08);
  }
}
```
NOTE: `position: relative` is baked into glass classes here (per research recommendation in Open Question 1) so that the ::before scrim positions correctly without requiring components to remember to add it. The @supports guard ensures this only activates when backdrop-filter is actually supported.

CRITICAL: The order matters — @media responsive MUST come after the base glass utilities (from plugin), and @media prefers-reduced-transparency MUST come after @media responsive so it wins in specificity cascade.
