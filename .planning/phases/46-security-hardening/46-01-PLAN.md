  <behavior>
    sanitize.test.ts:
    - sanitizeText('<script>alert(1)</script>') returns empty string
    - sanitizeText('<b>bold</b>') returns 'bold' (strips all tags)
    - sanitizeText('hello &amp; world') returns 'hello &amp; world' (preserves entities)
    - sanitizeRichText('<b>bold</b><script>x</script>') returns '<b>bold</b>' (allows safe tags, strips script)
    - sanitizeRichText('<a href="x" onclick="evil()">link</a>') strips onclick attribute

    hibp.test.ts (mock fetch):
    - isPasswordBreached('password') returns true (mock HIBP response with matching suffix)
    - isPasswordBreached('uniqueSecurePassword123!') returns false (mock HIBP response without matching suffix)
    - isPasswordBreached returns false when HIBP API returns 500 (fail-open)
    - isPasswordBreached returns false when fetch throws network error (fail-open)

    ssrf.test.ts:
    - isPrivateIP('http://10.0.0.1/hook') returns true
    - isPrivateIP('http://192.168.1.1/hook') returns true
    - isPrivateIP('http://172.16.0.1/hook') returns true
    - isPrivateIP('http://127.0.0.1/hook') returns true
    - isPrivateIP('http://169.254.1.1/hook') returns true
    - isPrivateIP('https://api.example.com/hook') returns false
    - isPrivateIP('invalid-url') returns true (treat invalid as unsafe)
    - validateWebhookUrl throws for private IPs
    - validateWebhookUrl does not throw for public URLs
  </behavior>
  <action>
    Install isomorphic-dompurify and @types/dompurify:
    ```
    pnpm --filter @schedulebox/web add isomorphic-dompurify
    pnpm --filter @schedulebox/web add -D @types/dompurify
    ```

    Create `apps/web/lib/security/sanitize.ts`:
    - Import DOMPurify from 'isomorphic-dompurify'
    - Export `sanitizeText(input: string): string` — ALLOWED_TAGS: [], ALLOWED_ATTR: []
    - Export `sanitizeRichText(input: string): string` — ALLOWED_TAGS: ['b','i','em','strong','a','p','br','ul','ol','li'], ALLOWED_ATTR: ['href','target','rel']

    Create `apps/web/lib/auth/hibp.ts`:
    - Import createHash from 'crypto'
    - Export `isPasswordBreached(password: string): Promise<boolean>`
    - SHA-1 hash the password, split into 5-char prefix + suffix
    - Fetch `https://api.pwnedpasswords.com/range/{prefix}` with { headers: { 'Add-Padding': 'true' }, cache: 'no-store' }
    - Parse response lines, check if any suffix matches
    - Wrap in try/catch — on ANY error, log warning and return false (fail-open)

    Create `apps/web/lib/security/ssrf.ts`:
    - Define PRIVATE_RANGES array of regex patterns covering: 127.x, 10.x, 172.16-31.x, 192.168.x, 169.254.x, 0.x, 100.64-127.x (CGNAT), ::1, fc00:, fd..:, fe80:
    - Export `isPrivateIP(urlString: string): boolean` — parse URL, test hostname against all patterns, return true on invalid URL
    - Export `validateWebhookUrl(url: string): void` — throw error if isPrivateIP returns true
    - Export Zod refinement: `webhookUrlSchema` = z.string().url().refine(url => !isPrivateIP(url), { message: 'Webhook URL must not point to a private IP address' })

    Write tests FIRST (RED), then implement (GREEN). Follow TDD cycle.
  </action>

  <action>
    **XSS Sanitization (SEC-02):**
    In `reviews/route.ts` POST handler — after Zod validation, before DB insert:
    - Import `sanitizeText` from `@/lib/security/sanitize`
    - Sanitize `comment` field: `const cleanComment = sanitizeText(input.comment)`
    - Use `cleanComment` in the insert statement instead of raw `input.comment`
    - If a reply endpoint exists in the same file or a nested route, sanitize `reply` field the same way

    Also check and sanitize in any route that writes to:
    - `notification_templates.body_template` — use `sanitizeRichText` (allows basic formatting)
    - `companies.description` — use `sanitizeRichText`

    **HIBP Breach Check (SEC-04):**
    In `auth/register/route.ts` POST handler — after Zod body validation, BEFORE `hashPassword`:
    - Import `isPasswordBreached` from `@/lib/auth/hibp`
    - `const breached = await isPasswordBreached(input.password)`
    - If breached, throw ValidationError with message: 'This password has appeared in known data breaches. Please choose a different password.'
    - Add Czech translation for this error message in cs.json under `auth.errors.breachedPassword`

    In `auth/change-password/route.ts` POST handler — after old password verification, BEFORE updating:
    - Same pattern: check `isPasswordBreached(body.new_password || body.newPassword)` (check actual field name in schema)
    - If breached, throw same ValidationError

    **SSRF Protection (SEC-05):**
    In `automation/rules/route.ts` POST handler — when `actionType === 'webhook'`:
    - Import `validateWebhookUrl` from `@/lib/security/ssrf`
    - After Zod validation, check `if (input.actionType === 'webhook' && input.actionConfig?.url)`
    - Call `validateWebhookUrl(input.actionConfig.url)` — it throws on private IPs
    - Same check in `automation/rules/[id]/route.ts` PUT/PATCH handler

    IMPORTANT: Read each route file first to understand the exact variable names, schema shapes, and insertion patterns before modifying. Do not assume — the field names may differ from the examples above.
