**Step 1: Run the build**

Run `pnpm build` from the project root. This executes `pnpm --filter @schedulebox/web build` which runs `next build`.

**Step 2: Analyze errors**

Capture all build output. Categorize errors:
- TypeScript type errors (TS2xxx codes)
- Missing imports or modules
- ESLint errors that block build
- Next.js specific warnings (dynamic server usage, etc.)

**Step 3: Fix errors systematically**

For each error, apply the minimal fix:

- **Missing type imports**: Add explicit type imports where needed
- **Unused variables**: Remove or prefix with underscore
- **Type mismatches**: Fix the type, not suppress with `as any` (unless the type is genuinely `any` from a third-party lib)
- **Missing module errors**: Check if the import path is wrong or if the file was moved
- **ESLint errors**: Fix the code (don't disable rules unless truly necessary — e.g., a legitimate `any` from external API)
- **next/dynamic or server component issues**: Ensure 'use client' directive is present where needed

Do NOT:
- Add `@ts-ignore` or `@ts-expect-error` unless absolutely no other option
- Change business logic — only fix type/lint issues
- Add new dependencies
- Modify the build configuration (next.config.mjs is fine as-is)

**Step 4: Re-run build**

After fixes, run `pnpm build` again. Repeat fix cycle until build succeeds with exit code 0.

**Step 5: Verify standalone output**

Confirm that `apps/web/.next/standalone/server.js` exists after successful build. Confirm that `apps/web/.next/static` directory exists.
