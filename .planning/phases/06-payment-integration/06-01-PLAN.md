  <name>Task 1: Payment Zod Schemas and TypeScript Types</name>
  <files>
    packages/shared/src/schemas/payment.ts

    packages/shared/src/types/index.ts

  <action>
    Create payment validation schemas following the exact pattern from packages/shared/src/schemas/booking.ts:

    1. **packages/shared/src/schemas/payment.ts** — Create these Zod schemas:
       - `paymentStatusEnum`: z.enum(['pending', 'paid', 'failed', 'refunded', 'partially_refunded'])
       - `paymentGatewayEnum`: z.enum(['comgate', 'qrcomat', 'cash', 'bank_transfer', 'gift_card'])
       - `paymentCreateSchema`: { booking_id (number, required), gateway (paymentGatewayEnum), amount (number, positive), currency (string, 3 chars, default 'CZK') }
       - `comgateCreateSchema`: { booking_id (number, required) } — Comgate payment initiation (amount derived from booking)
       - `qrPaymentGenerateSchema`: { booking_id (number, required) } — QR code generation
       - `paymentRefundSchema`: { amount (number, positive, optional for full refund), reason (string, max 500 chars, required) }
       - `paymentListQuerySchema`: page (z.coerce.number, min 1, default 1), limit (z.coerce.number, 1-100, default 20), status (paymentStatusEnum optional), gateway (paymentGatewayEnum optional), date_from (z.string().date() optional), date_to (z.string().date() optional), booking_id (z.coerce.number optional)
       - `invoiceStatusEnum`: z.enum(['draft', 'issued', 'paid', 'cancelled'])

    2. **packages/shared/src/types/payment.ts** — Create TypeScript types:
       - `PaymentStatus` = z.infer of paymentStatusEnum
       - `PaymentGateway` = z.infer of paymentGatewayEnum
       - `Payment` type matching API spec: { id, uuid, companyId, bookingId, customerId, amount (string, decimal), currency, status, gateway, gatewayTransactionId, refundAmount, refundReason, paidAt, refundedAt, createdAt, updatedAt }
       - `PaymentCreate` = z.infer of paymentCreateSchema
       - `PaymentRefund` = z.infer of paymentRefundSchema
       - `PaymentListQuery` = z.infer of paymentListQuerySchema
       - `ComgateCreateResponse` = { transactionId: string, redirectUrl: string }
       - `QrPaymentResponse` = { qrCodeBase64: string, spdString: string }
       - `Invoice` type: { id, uuid, companyId, paymentId, customerId, invoiceNumber, amount, taxAmount, currency, status, issuedAt, dueAt, pdfUrl, createdAt }
       - `InvoiceStatus` = z.infer of invoiceStatusEnum

    3. Update **packages/shared/src/schemas/index.ts** — Add re-exports for payment schemas (schemas only, no types, per 05-01 pattern)
    4. Update **packages/shared/src/types/index.ts** — Add re-exports for payment types

    IMPORTANT: Follow the same separation pattern from Phase 5: schemas/ files export ONLY Zod schemas, types/ files export types. Do NOT export types from schema files (causes TS2308 module conflicts).
  </action>
  <verify>
    Run `pnpm exec tsc --noEmit -p packages/shared/tsconfig.json` — zero errors.
    Verify imports work: schemas importable from @schedulebox/shared, types importable from @schedulebox/shared.
  </verify>
  <done>
