  <name>Task 1: Payment List, Detail, and Manual Create Endpoints</name>
  <files>
    apps/web/app/api/v1/payments/route.ts
    apps/web/app/api/v1/payments/[id]/route.ts
  </files>
  <action>
    Create the standard payment CRUD endpoints following the pattern from apps/web/app/api/v1/customers/route.ts.

    1. **apps/web/app/api/v1/payments/route.ts** — GET + POST handlers:

       **GET /api/v1/payments** — List payments:
       - Use createRouteHandler with requiresAuth: true
       - Validate query with paymentListQuerySchema from @schedulebox/shared
       - Query payments table filtered by companyId (tenant isolation)
       - Apply optional filters: status, gateway, date_from/date_to (on createdAt), booking_id
       - Support pagination: page, limit, offset calculation
       - Order by createdAt DESC (newest first)
       - Return paginatedResponse with: data (payment array), meta (total, page, limit, totalPages)
       - Each payment in list includes: uuid, amount, currency, status, gateway, gatewayTransactionId, paidAt, createdAt
       - Do NOT include gatewayResponse in list (too large, save for detail view)

       **POST /api/v1/payments** — Create manual payment:
       - Use createRouteHandler with requiresAuth: true, admin/owner permissions
       - Validate request body with paymentCreateSchema
       - This is for MANUAL payment recording (cash, bank_transfer) — not for Comgate/QR (those have dedicated endpoints)
       - Validate booking_id exists and belongs to company
       - Validate gateway is 'cash' or 'bank_transfer' (Comgate and QR use their own endpoints)
       - Create payment record with status='paid', paidAt=new Date() (manual payments are immediately confirmed)
       - Publish createPaymentCompletedEvent
       - Call handlePaymentCompleted from SAGA handlers (auto-confirms booking)
       - Create invoice via createInvoiceForPayment
       - Return created payment with status 201

    2. **apps/web/app/api/v1/payments/[id]/route.ts** — GET handler:

       **GET /api/v1/payments/{id}** — Payment detail:
       - Use createRouteHandler with requiresAuth: true
       - Extract payment UUID from path params
       - Query payment by UUID with companyId filter
       - JOIN with booking (uuid, status, startTime, endTime, serviceName), customer (uuid, name, email), and invoice (uuid, invoiceNumber, status)
       - Return full payment detail including gatewayResponse (admin debugging info)
       - Return related booking and customer info for context
       - If not found, return 404

    IMPORTANT: Always filter by companyId. Use uuid (not internal serial id) in path params and responses per project convention (never expose SERIAL IDs).
  </action>
  <verify>

    routes export GET and POST (payments/route.ts) and GET (payments/[id]/route.ts).
    List supports pagination and filtering.
  </verify>
  <done>
    Payment list with filtering and pagination. Payment detail with related booking/customer/invoice. Manual payment creation for cash/bank transfers with auto-confirmation and invoice generation.
  </done>
