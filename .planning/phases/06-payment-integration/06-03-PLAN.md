  <files>

  <action>
    Create a Comgate API client module that encapsulates all HTTP communication with Comgate.

    **apps/web/app/api/v1/payments/comgate/client.ts:**

    1. `initComgatePayment(params)` — Create payment on Comgate:
       - Params: { price (number, in CZK), currency (string), label (string), refId (string — booking UUID), email (string), redirectUrl (string), callbackUrl (string) }
       - Comgate API endpoint: `https://payments.comgate.cz/v1.0/create` (POST, application/x-www-form-urlencoded)
       - Request params: merchant (COMGATE_MERCHANT_ID env), test (true if NODE_ENV !== 'production'), price (amount in hellers = price * 100), curr, label, refId, email, method='ALL', prepareOnly=true, lang='cs', url (redirectUrl), callback (callbackUrl)
       - NOTE: Comgate uses `prepareOnly=true` to get a redirect URL without auto-redirect
       - Response: URL-encoded string like `code=0&message=OK&transId=ABC123&redirect=https://...`
       - Parse response with URLSearchParams
       - If code !== '0', throw AppError with PAYMENT_GATEWAY_ERROR
       - Return { transactionId: string, redirectUrl: string }

    2. `getComgatePaymentStatus(transId)` — Check payment status:
       - Endpoint: `https://payments.comgate.cz/v1.0/status` (POST)
       - Params: merchant, transId, secret (COMGATE_SECRET env)
       - Return parsed status

    3. `refundComgatePayment(transId, amount?)` — Initiate refund:
       - Endpoint: `https://payments.comgate.cz/v1.0/refund` (POST)
       - Params: merchant, transId, secret, amount (optional, in hellers for partial refund)
       - Return success/failure

    4. `verifyComgateSignature(rawBody, signature)` — Verify webhook signature:
       - Use crypto.createHmac('sha256', COMGATE_SECRET).update(rawBody).digest('hex')
       - Use crypto.timingSafeEqual for constant-time comparison
       - Return boolean
       - Handle Buffer length mismatch gracefully (return false)

    Environment variables used: COMGATE_MERCHANT_ID, COMGATE_SECRET
    Add COMGATE_MERCHANT_ID and COMGATE_SECRET to .env.example with placeholder values.
    Also add COMGATE_API_URL=https://payments.comgate.cz (allows override for testing).

    Use native `fetch()` (not axios) — already available in Node.js 20. Set 15-second timeout using AbortController.

<task type="auto">
  <name>Task 3: Comgate Webhook Endpoint</name>
  <files>

  <action>
    Create the Comgate webhook receiver endpoint with full idempotency and signature verification.

    **apps/web/app/api/v1/webhooks/comgate/route.ts** — POST handler:

    This is a PUBLIC endpoint (no auth — called by Comgate servers). Do NOT use createRouteHandler with requiresAuth.

    1. Read raw body using `await req.text()` BEFORE any JSON parsing (critical for signature verification)
    2. Get signature from header (check Comgate docs — likely `x-comgate-signature` or similar header)
    3. Verify signature using verifyComgateSignature(rawBody, signature) — return 401 if invalid
    4. Parse the body (Comgate sends application/x-www-form-urlencoded format: `transId=ABC&status=PAID&price=15000`)
    5. Extract transId and status from parsed body
    6. Idempotency check: call checkWebhookIdempotency(transId, 'comgate', parsedPayload)
       - If alreadyProcessed === true, return 200 immediately (prevent retry loops)
    7. Find payment by gateway transaction ID using findPaymentByGatewayTx('comgate', transId)
       - If not found, log warning and return 200 (Comgate might send webhook before our payment record exists in edge cases)
    8. Process based on Comgate status:
       - If status === 'PAID':
         - Call updatePaymentStatus(payment.id, 'paid', { paidAt: new Date(), gatewayResponse: parsedPayload })
         - Publish createPaymentCompletedEvent with { paymentUuid, bookingUuid, companyId, amount, currency, gateway: 'comgate', gatewayTransactionId: transId, paidAt }
       - If status === 'CANCELLED' or status === 'AUTHORIZED' (authorized but not captured — treat as pending):
         - If CANCELLED: call updatePaymentStatus(payment.id, 'failed', { gatewayResponse: parsedPayload })
         - Publish createPaymentFailedEvent with { paymentUuid, bookingUuid, companyId, gateway: 'comgate', reason: 'Payment cancelled by customer' }
    9. Call markWebhookCompleted(transId)
    10. Return 200 immediately (Comgate expects response within 5 seconds)

    Wrap entire handler in try/catch. On ANY error, log it but still return 200 to prevent Comgate retry loops from hammering the endpoint. Mark webhook as 'failed' status.

    IMPORTANT: Do NOT use express.json() or any JSON parsing middleware before reading raw body. Next.js API routes provide req.text() which gives raw body — use this.
  </action>
