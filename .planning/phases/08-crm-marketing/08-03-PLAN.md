    **Step 2: Add import row schema to customer validations**
    Add to `apps/web/validations/customer.ts`:
    ```typescript
    export const customerImportRowSchema = z.object({
      name: z.string().min(1, 'Name is required').max(255),
      email: z.string().email('Invalid email').optional().or(z.literal('')).transform(v => v || undefined),
      phone: z.string().max(50).optional().or(z.literal('')).transform(v => v || undefined),
      date_of_birth: z.string().date('Invalid date').optional().or(z.literal('')).transform(v => v || undefined),
      notes: z.string().optional().or(z.literal('')).transform(v => v || undefined),
    });
    export type CustomerImportRow = z.infer<typeof customerImportRowSchema>;
    ```
    The `.or(z.literal(''))` handles empty CSV cells, and `.transform(v => v || undefined)` converts empty strings to undefined so they become NULL in the database.

    **Step 3: Replace import scaffold**
    Completely rewrite `apps/web/app/api/v1/customers/import/route.ts`:

    **POST /api/v1/customers/import**:
    - Use createRouteHandler with requiresAuth: true, requiredPermissions: [PERMISSIONS.CUSTOMERS_CREATE]
    - Call findCompanyId(user.sub)
    - Extract file from request: `const formData = await req.formData(); const file = formData.get('file');`
    - Validate: if no file or not File instance, throw ValidationError('CSV file is required')
    - Validate file size: if file.size > 10 * 1024 * 1024 (10MB), throw ValidationError('File size exceeds 10MB limit')
    - Read file content as text: `const text = await file.text()`
    - Parse with PapaParse synchronously (since we already have the text):
      ```typescript
      const parsed = Papa.parse(text, {
        header: true,
        skipEmptyLines: true,
        transformHeader: (header: string) => header.trim().toLowerCase(),
      });
      ```
    - Initialize results: `{ imported: 0, skipped: 0, errors: [] as Array<{ row: number, error: string }> }`
    - Validate max rows: if parsed.data.length > 500000, throw ValidationError('CSV exceeds 500,000 row limit')
    - Process rows in batches:
      ```typescript
      const BATCH_SIZE = 1000;
      const validRows: Array<{ name: string; email?: string; phone?: string; dateOfBirth?: string; notes?: string; companyId: number; source: string }> = [];

      for (let i = 0; i < parsed.data.length; i++) {
        const row = parsed.data[i];
        const validation = customerImportRowSchema.safeParse(row);
        if (!validation.success) {
          results.errors.push({ row: i + 2, error: validation.error.issues[0].message });
          results.skipped++;
          if (results.errors.length >= 100) {
            // Cap errors to prevent response from being too large
            results.errors.push({ row: 0, error: `Error reporting capped at 100. ${parsed.data.length - i - 1} remaining rows not checked.` });
            break;
          }
          continue;
        }
        validRows.push({
          name: validation.data.name,
          email: validation.data.email,
          phone: validation.data.phone,
          dateOfBirth: validation.data.date_of_birth,
          notes: validation.data.notes,
          companyId,
          source: 'import',
        });
      }
      ```
    - Batch insert valid rows:
      ```typescript
      for (let i = 0; i < validRows.length; i += BATCH_SIZE) {
        const chunk = validRows.slice(i, i + BATCH_SIZE);
        const result = await db
          .insert(customers)
          .values(chunk)
          .onConflictDoNothing({ target: [customers.companyId, customers.email] });
        // Note: onConflictDoNothing means duplicates are silently skipped
        // We count all valid rows as "imported" (includes skipped duplicates)
      }
      results.imported = validRows.length;
      ```
    - Return successResponse with results: `{ imported: results.imported, skipped: results.skipped, errors: results.errors, total_rows: parsed.data.length }`

    Create `apps/web/app/api/v1/customers/[id]/anonymize/route.ts`:

    **DELETE /api/v1/customers/[id]/anonymize**:
    - Use createRouteHandler with paramsSchema: customerIdParamSchema (from @/validations/customer), requiredPermissions: [PERMISSIONS.CUSTOMERS_DELETE]
    - Call findCompanyId(user.sub)
    - Find customer by UUID and companyId and deletedAt IS NULL. If not found, throw NotFoundError('Customer not found')
    - Anonymize customer data by updating:
      ```typescript
      await db
        .update(customers)
        .set({
          name: `Deleted User ${customer.id}`,
          email: null,
          phone: null,
          dateOfBirth: null,
          notes: null,
          marketingConsent: false,
          deletedAt: new Date(),
          updatedAt: new Date(),
        })
        .where(eq(customers.id, customer.id));
      ```
    - Also remove all tag associations for this customer:
      ```typescript
      await db
        .delete(customerTags)
        .where(eq(customerTags.customerId, customer.id));
      ```
    - The anonymization intentionally PRESERVES:
      - `totalBookings` and `totalSpent` (aggregate business analytics, not PII)
      - `healthScore`, `clvPredicted`, `noShowCount` (AI metrics, not PII)
      - The customer row itself (soft delete, not hard delete) for referential integrity with bookings
    - Return noContentResponse() (204 No Content)

    NOTE: This is different from the existing DELETE /customers/[id] which only sets deletedAt. This endpoint additionally anonymizes all PII fields. The difference:
    - DELETE /customers/[id] = soft delete (preserves data, can be restored)
    - DELETE /customers/[id]/anonymize = GDPR erasure (irreversible, anonymizes PII)
