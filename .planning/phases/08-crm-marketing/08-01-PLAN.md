  <action>
    Create Zod validation schemas in `apps/web/validations/coupon.ts`:
    - `couponCreateSchema`: code (string, .toUpperCase() transform, 3-50 chars), description (optional, max 255), discount_type (enum: 'percentage' | 'fixed'), discount_value (number, positive), min_booking_amount (number, >=0, optional, default 0), max_uses (number int positive, optional — NULL = unlimited), max_uses_per_customer (number int positive, optional, default 1), applicable_service_ids (array of int, optional — NULL = all services), valid_from (string datetime, optional), valid_until (string datetime, optional), is_active (boolean, default true)
    - `couponUpdateSchema`: all fields from create but all optional, code still transforms toUpperCase if provided
    - `couponQuerySchema`: page (coerce int, min 1, default 1), limit (coerce int, min 1, max 100, default 20), search (string optional — searches code and description), is_active (coerce boolean, optional)
    - `couponIdParamSchema`: id (string.uuid for UUID lookup)
    - Export all schemas and inferred types

    Create `apps/web/app/api/v1/coupons/route.ts` with GET and POST:

    **GET /api/v1/coupons** (list with pagination):
    - Use createRouteHandler with requiresAuth: true, requiredPermissions: [PERMISSIONS.COUPONS_MANAGE]
    - Call findCompanyId(user.sub) for tenant isolation
    - Parse query params with validateQuery(couponQuerySchema, req)
    - Query coupons table WHERE companyId = companyId
    - If search provided, add ILIKE on code and description
    - If is_active filter provided, add eq(coupons.isActive, filter)
    - Order by createdAt DESC
    - Apply pagination (offset, limit)
    - Count total for pagination meta
    - Return paginatedResponse with coupon data (use uuid as public id, never expose serial id)

    **POST /api/v1/coupons** (create):
    - Use createRouteHandler with bodySchema: couponCreateSchema, requiredPermissions: [PERMISSIONS.COUPONS_MANAGE]
    - Call findCompanyId(user.sub)
    - Check for duplicate code within company: SELECT from coupons WHERE companyId AND code = body.code (already uppercase from Zod). If exists, throw ConflictError('Coupon code already exists')
    - Insert into coupons with all fields from body + companyId
    - Return createdResponse with the new coupon (uuid as id)

    Create `apps/web/app/api/v1/coupons/[id]/route.ts` with GET, PUT, DELETE:

    **GET /api/v1/coupons/[id]** (detail):
    - Use createRouteHandler with paramsSchema: couponIdParamSchema, requiredPermissions: [PERMISSIONS.COUPONS_MANAGE]
    - Find coupon by UUID and companyId. If not found, throw NotFoundError
    - Return successResponse with coupon data

    **PUT /api/v1/coupons/[id]** (update):
    - Use createRouteHandler with paramsSchema: couponIdParamSchema, bodySchema: couponUpdateSchema, requiredPermissions: [PERMISSIONS.COUPONS_MANAGE]
    - Find coupon by UUID and companyId. If not found, throw NotFoundError
    - If code changed, check for duplicate within company (exclude current coupon id)
    - Update coupon with body fields + updatedAt: new Date()
    - Return successResponse with updated coupon

    **DELETE /api/v1/coupons/[id]**:
    - Use createRouteHandler with paramsSchema: couponIdParamSchema, requiredPermissions: [PERMISSIONS.COUPONS_MANAGE]
    - Find coupon by UUID and companyId. If not found, throw NotFoundError
    - Hard delete the coupon (CASCADE will remove coupon_usage records)
    - Return noContentResponse()

    Follow the exact same import patterns as apps/web/app/api/v1/customers/route.ts — import from @schedulebox/database, @schedulebox/shared, @/lib/middleware/route-handler, @/lib/db/tenant-scope, @/lib/middleware/rbac, @/lib/utils/response, @/validations/coupon.

    Map database columns to API response: use uuid as 'id', snake_case for field names (discount_type, discount_value, max_uses, etc.), format dates as ISO strings. Never expose serial id.

<task type="auto">
  <name>Task 2: Coupon validation endpoint</name>
  <files>
    apps/web/validations/coupon.ts

  <action>
    Add `couponValidateSchema` to `apps/web/validations/coupon.ts`:
    - code (string, .toUpperCase() transform)
    - service_id (number int, positive)
    - customer_id (string uuid — customer UUID for per-customer usage check)

    Create `apps/web/app/api/v1/coupons/validate/route.ts`:

    **POST /api/v1/coupons/validate**:
    - Use createRouteHandler with bodySchema: couponValidateSchema, requiredPermissions: [PERMISSIONS.COUPONS_MANAGE]
    - Call findCompanyId(user.sub)
    - Find coupon by code (already uppercase) and companyId and isActive=true. If not found, return successResponse({ data: { valid: false, message: 'Invalid coupon code' } })
    - Check expiration: if validFrom exists and now < validFrom, return { valid: false, message: 'Coupon not yet valid' }. If validUntil exists and now > validUntil, return { valid: false, message: 'Coupon has expired' }
    - Check global usage limit: if maxUses is not null and currentUses >= maxUses, return { valid: false, message: 'Coupon usage limit reached' }
    - Check per-customer usage: resolve customer UUID to internal ID (SELECT from customers WHERE uuid = body.customer_id AND companyId). If customer not found, throw NotFoundError('Customer not found'). Then count from coupon_usage WHERE couponId = coupon.id AND customerId = customer.id. If count >= maxUsesPerCustomer, return { valid: false, message: 'You have already used this coupon' }
    - Check service applicability: if applicableServiceIds is not null and the array does NOT include body.service_id, return { valid: false, message: 'Coupon not valid for this service' }
    - If all checks pass, return successResponse({ data: { valid: true, discount_type: coupon.discountType, discount_value: Number(coupon.discountValue), min_booking_amount: Number(coupon.minBookingAmount), message: 'Coupon applied successfully' } })

    NOTE on service_id: The validate endpoint receives the internal service ID (integer) since this is called from the booking flow server-side where internal IDs are available. The applicableServiceIds array in the coupons table stores integer service IDs.

    Import couponUsage from @schedulebox/database alongside coupons and customers.
    Use sql<number>`count(*)` for the per-customer usage count query (same pattern as 08-RESEARCH.md).
  </action>
