1. Create `apps/web/lib/push/automation-push-handler.ts`:
   - Import sendPushToUser from '@/lib/push/web-push-service'
   - Import db, notifications, users, customers from '@schedulebox/database'
   - Import eq, and from 'drizzle-orm'

   - Export `async function executeAutomationPush(params: { ruleId: number; companyId: number; customerId?: number; bookingId?: number; actionConfig: { title?: string; body?: string; url?: string } }): Promise<{ success: boolean; sent: number; notificationId: number | null }>`:
     a. Determine target userId: if customerId provided, look up the user record associated with that customer (match by email+companyId in users table)
     b. Build push payload: title from actionConfig.title or 'ScheduleBox', body from actionConfig.body or 'Mate novou notifikaci', url from actionConfig.url or '/dashboard'
     c. Call sendPushToUser(targetUserId, payload)
     d. Insert notification record: channel 'push', companyId, customerId, bookingId, subject: payload.title, body: payload.body, status: result.sent > 0 ? 'sent' : 'failed'
     e. Return { success: result.sent > 0, sent: result.sent, notificationId }
     f. If no targetUserId found (customer has no user account), return { success: false, sent: 0, notificationId: null } — log warning

2. Wire into automation executor:
   - Check if `apps/web/app/api/v1/automation/execute/route.ts` exists. If not, search for the automation execution logic (grep for 'send_email' action handler or automation executor).
   - In the automation execution switch/if block that handles actionType:
     - Find where 'send_email' is handled
     - Add a case for 'send_push' that calls executeAutomationPush with the rule's actionConfig, customerId, bookingId
     - Log the result to automation_logs table (same pattern as send_email logging)
   - If no automation executor exists yet (it may just be DB schema + CRUD), create `apps/web/app/api/v1/automation/execute/route.ts` as a POST endpoint:
     - requiresAuth: true, requiredPermissions: [PERMISSIONS.SETTINGS_MANAGE]
     - Body: { ruleId: number }
     - Fetches the rule, determines action type, dispatches to appropriate handler
     - For send_push: calls executeAutomationPush
     - For send_email: existing email handler (or no-op if not wired yet)
     - Inserts automation_log record with status and result
     - Returns execution result
