  <action>
Create two public (no auth required) marketplace endpoints following the createRouteHandler pattern.

**GET /api/v1/marketplace/listings (public catalog):**
- `requiresAuth: false` (public endpoint per API spec - `security: []`)
- Validate query params with `marketplaceSearchQuerySchema`
- Build WHERE conditions: `is_active = true`
- If `category` provided: add `eq(marketplaceListings.category, category)`
- If `city` provided: add `ilike(marketplaceListings.addressCity, city)`
- If `search` provided: add `ilike` on title OR description
- If `lat` AND `lng` provided: calculate distance using raw SQL
  - Distance formula: Use Haversine via SQL since PostGIS may not be available:
    ```sql
    (6371 * acos(cos(radians($lat)) * cos(radians(latitude)) * cos(radians(longitude) - radians($lng)) + sin(radians($lat)) * sin(radians(latitude)))) AS distance_km
    ```
  - If `radius_km` provided: filter WHERE distance_km <= radius_km
  - Note: This is a fallback approach. If PostGIS is available, use `ST_Distance` and `ST_DWithin` instead. For MVP without PostGIS, Haversine in SQL is acceptable for the expected scale (<10k listings).
- Sorting: if `sort_by = 'distance'` AND lat/lng provided, ORDER BY distance. If `sort_by = 'rating'`, ORDER BY averageRating DESC. If `sort_by = 'name'`, ORDER BY title ASC.
- Pagination: offset-based with `page` and `limit`
- Return `paginatedResponse` with listings (UUID as id, never expose SERIAL)
- Map response: include `distance` field only when lat/lng provided

**GET /api/v1/marketplace/listings/[id] (public detail):**
- `requiresAuth: false`
- Accept UUID in path params (not SERIAL ID)
- Find listing by UUID where `is_active = true`
- Return 404 if not found
- Return listing with UUID as id, include all fields
- Include company services and employees count if possible (or defer to Phase 13)

Use `sql` template from drizzle-orm for raw SQL in distance calculations. Import `db, marketplaceListings` from `@schedulebox/database`. Use `successResponse` and `paginatedResponse` from `@/lib/utils/response`.
