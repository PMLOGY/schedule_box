  <action>
Create a RabbitMQ consumer that listens for review.created events and updates marketplace listing aggregate ratings.

**review-rating-sync.ts:**
- Import consumer helper from `@schedulebox/events` (createConsumer or equivalent pattern from Phase 7 consumers)
- Import `db`, `reviews`, `marketplaceListings` from `@schedulebox/database`
- Import `eq`, `and`, `sql`, `avg`, `count` from `drizzle-orm`
- Import `ReviewCreatedPayload` from `@schedulebox/events`

- `handleReviewCreated(payload: ReviewCreatedPayload)`:
  1. Extract `companyId` from payload
  2. Calculate aggregate rating: SELECT AVG(rating), COUNT(*) FROM reviews WHERE companyId = companyId AND isPublished = true AND deletedAt IS NULL
  3. Update marketplace listing: UPDATE marketplace_listings SET averageRating = calculatedAvg (rounded to 2 decimal places), reviewCount = calculatedCount, updatedAt = new Date() WHERE companyId = companyId
  4. If no marketplace listing exists for this company: log info and skip (company hasn't created a listing yet)
  5. Log: "Updated marketplace rating for company ${companyId}: ${avg} (${count} reviews)"

- `startReviewRatingSyncConsumer()`:
  1. Create consumer binding to `review.created` routing key on `schedulebox.events` topic exchange
  2. Queue name: `review-rating-sync`
  3. On message: parse CloudEvent payload, call handleReviewCreated
  4. ACK message after successful processing
  5. On error: log error, NACK without requeue (prevent infinite retry loops on bad data)
  6. Return cleanup function for graceful shutdown

- Export both `handleReviewCreated` (for testing) and `startReviewRatingSyncConsumer` (for lifecycle)
