**apps/web/public/widget/embed.js:**
Plain JavaScript file (no build step, no React, no TypeScript - must work on any website). Served as static file from /widget/embed.js.

```javascript
// ScheduleBox Booking Widget Loader
// Usage: <script src="https://app.schedulebox.cz/widget/embed.js"></script>
//        <schedulebox-widget data-company="my-salon" data-theme="light"></schedulebox-widget>
```

- Define `ScheduleBoxWidget` as a custom element extending HTMLElement
- `connectedCallback()`:
  1. Read attributes: `data-company` (required), `data-theme` (optional, default 'light'), `data-locale` (optional, default 'cs'), `data-width` (optional, default '100%'), `data-height` (optional, default '600px')
  2. If `data-company` missing: show error message in element
  3. Create shadow DOM for style isolation
  4. Create iframe element:
     - src: `${window.location.protocol}//app.schedulebox.cz/embed/${companySlug}?theme=${theme}&locale=${locale}`
     - For development: use relative URL `/embed/${companySlug}?theme=${theme}&locale=${locale}` (detect via script src or data-base-url attribute)
     - style: width from data-width, min-height from data-height, border none, border-radius 8px
     - sandbox: 'allow-scripts allow-same-origin allow-forms allow-popups'
     - allow: 'payment' (for potential payment flow)
  5. Add PostMessage listener (origin-validated):
     - `BOOKING_COMPLETED`: dispatch CustomEvent 'booking-complete' with booking detail
     - `RESIZE`: update iframe height to match content
     - `ERROR`: dispatch CustomEvent 'widget-error' with error detail
     - Validate event.origin matches expected app URL
  6. Append iframe to shadow DOM
- `disconnectedCallback()`: remove PostMessage listener
- Register: `customElements.define('schedulebox-widget', ScheduleBoxWidget)`
- Include inline CSS in shadow DOM for default styling (loading spinner while iframe loads)

**GET /api/v1/public/widget/config/[slug]:**
- No auth required (public endpoint)
- Find company by slug
- If not found: return 404
- Query marketplace listing for branding (colors, logo)
- Query active services (name, duration, price)
- Return config object: companyName, companySlug, logo, primaryColor, services (minimal: name, uuid, duration, price), locale defaults
- Set CORS headers: `Access-Control-Allow-Origin: *` (widget can be embedded anywhere)
- Set cache headers: `Cache-Control: public, max-age=300` (5 min cache for config)

**layout.tsx:**
- Minimal layout with NO navigation, NO header, NO footer
- Just `<html><body>{children}</body></html>` with Tailwind CSS
- Include base styles and font loading
- Meta viewport for mobile responsiveness
- CSP meta tag: `default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; frame-ancestors *;`
- Read `theme` from searchParams (light/dark) and apply appropriate Tailwind classes

**page.tsx:**
- Server component that fetches company data via direct Drizzle queries (same DB access as server component)
- Accept `company_slug` from params, `theme` and `locale` from searchParams
- If company not found: render "Company not found" message
- Render compact booking widget UI:
  1. **Company header**: Logo (if available), company name, average rating
  2. **Service list**: Scrollable list of active services with name, duration, price, "Select" button
  3. **Selected service state**: When service selected, show date picker placeholder and CTA
  4. Note: Full booking wizard integration (date/time picker, customer form) will connect to existing booking wizard components or redirect to the full booking flow. For this plan, implement service selection and a "Continue to booking" link that opens the full public booking page in a new tab.
- PostMessage to parent on key events:
  - `window.parent.postMessage({ type: 'RESIZE', height: document.body.scrollHeight }, '*')` after render
  - `window.parent.postMessage({ type: 'BOOKING_STARTED', service: selectedService }, '*')` on service select
- Use client components where interactivity needed (service selection, PostMessage)
- Style with Tailwind CSS, compact spacing (this is an embedded widget, not a full page)
- Support `theme=dark` by toggling Tailwind dark mode classes

The widget page lives OUTSIDE the [locale] routing to avoid middleware interference. It's at `/embed/[company_slug]` not `/[locale]/embed/[company_slug]`. The locale is passed as a query param instead.

IMPORTANT: Ensure this route doesn't conflict with existing Next.js middleware (which handles locale routing). The /embed path should be excluded from locale middleware if needed. Check the existing middleware.ts file and add /embed to the excluded paths if present.
