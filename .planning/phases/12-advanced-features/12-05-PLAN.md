  <action>
Create white-label app management endpoints following createRouteHandler pattern.

**GET /api/v1/apps/whitelabel:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.WHITELABEL_MANAGE]`
- Find company ID via `findCompanyId(user.sub)`
- Query whitelabelApps WHERE companyId matches
- If no app exists, return `successResponse` with `null`
- Return app config with UUID (never SERIAL ID), all fields

**POST /api/v1/apps/whitelabel:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.WHITELABEL_MANAGE]`
- Validate body with `whitelabelAppCreateSchema`
- Find company ID via `findCompanyId`
- Check if app already exists for company (UNIQUE constraint): query WHERE companyId
- If exists: return 409 ConflictError('App already exists for this company. Use PUT to update.')
- INSERT into whitelabelApps with companyId and validated data
- Default iosStatus='draft', androidStatus='draft'
- Return 201 with created app (UUID)

**PUT /api/v1/apps/whitelabel:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.WHITELABEL_MANAGE]`
- Validate body with `whitelabelAppUpdateSchema`
- Find company ID via `findCompanyId`
- Find existing app by companyId
- If not found: return 404
- UPDATE with validated data, set updatedAt=new Date()
- Do NOT allow updating iosStatus/androidStatus via PUT (these are system-controlled)
- Return updated app (UUID)

**POST /api/v1/apps/whitelabel/build:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.WHITELABEL_MANAGE]`
- Find company ID via `findCompanyId`
- Find existing app by companyId
- If not found: return 404 ('Create app configuration first')
- Validate app has required fields: appName must be set
- UPDATE app SET iosStatus='building', androidStatus='building', lastBuildAt=new Date()
- Note: Actual build execution is deferred to Phase 15 DevOps (RabbitMQ consumer, Expo CNG, Fastlane). For now, just update status to 'building' as a placeholder.
- Return 202 with `{ build_id: app.uuid, status: 'building', message: 'Build queued. Actual build infrastructure coming in Phase 15.' }`

Import `whitelabelApps` from `@schedulebox/database`. Import `PERMISSIONS` from `@/lib/middleware/rbac`. Use `ConflictError` from `@schedulebox/shared` for duplicate app creation.
