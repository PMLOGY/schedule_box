  <action>
Create 4 Zod schema files following the established schema-only export pattern (see loyalty.ts for reference). Schema fields must match database columns from packages/database/src/schema/*.ts.

**marketplace.ts:**
- `marketplaceListingCreateSchema`: title (string 3-255), description (optional string max 2000), category (optional string max 100), subcategory (optional string max 100), addressStreet (optional string max 255), addressCity (optional string max 100), addressZip (optional string max 20), latitude (optional z.coerce.number -90..90), longitude (optional z.coerce.number -180..180), images (optional string array), priceRange (optional enum '$','$$','$$$','$$$$'), featured (optional boolean), verified (optional boolean)
- `marketplaceListingUpdateSchema`: same fields as create but all optional (partial)
- `marketplaceSearchQuerySchema`: search (optional string), category (optional string), city (optional string), lat (optional z.coerce.number), lng (optional z.coerce.number), radius_km (optional z.coerce.number default 10, min 1, max 100), sort_by (optional enum 'rating','distance','name' default 'rating'), page (optional z.coerce.number default 1), limit (optional z.coerce.number default 20, max 50)

**review.ts:**
- `reviewCreateSchema`: bookingUuid (z.string().uuid()), rating (z.number().int().min(1).max(5)), comment (optional string min 10, max 1000)
- `reviewReplySchema`: reply (z.string().min(1).max(2000))
- `reviewListQuerySchema`: rating_min (optional z.coerce.number int 1-5), page (optional z.coerce.number default 1), limit (optional z.coerce.number default 20 max 50), status (optional enum 'pending','approved','rejected')

**video.ts:**
- `videoProviderEnum`: z.enum(['zoom', 'google_meet', 'ms_teams'])
- `videoMeetingCreateSchema`: bookingUuid (z.string().uuid()), provider (videoProviderEnum)
- `videoMeetingListQuerySchema`: page (optional z.coerce.number default 1), limit (optional z.coerce.number default 20)

**whitelabel.ts:**
- `whitelabelAppCreateSchema`: appName (string 2-100), bundleId (optional string max 255), logoUrl (optional string url max 500), primaryColor (optional string regex /^#[0-9A-Fa-f]{6}$/ default '#3B82F6'), secondaryColor (optional string regex /^#[0-9A-Fa-f]{6}$/ default '#1E40AF'), features (optional z.object({ booking: z.boolean(), loyalty: z.boolean(), push: z.boolean() }))
- `whitelabelAppUpdateSchema`: same as create but all optional (partial)

Update `packages/shared/src/schemas/index.ts` to re-export all new schemas.

Use z.coerce.number() for query parameters (automatic string-to-number conversion, following Phase 5 pattern). Export only schemas from schema files (types inferred separately in types/ files to prevent TS2308 module conflicts).

Create 4 type files that infer TypeScript types from the Zod schemas created in Task 1. Follow the established dual type definition pattern from loyalty.ts (response types + input types).

**marketplace.ts:**
- `MarketplaceListingCreate` = z.infer from marketplaceListingCreateSchema
- `MarketplaceListingUpdate` = z.infer from marketplaceListingUpdateSchema
- `MarketplaceSearchQuery` = z.infer from marketplaceSearchQuerySchema
- `MarketplaceListing` = response type with id, uuid, companyId, title, description, category, subcategory, addressStreet, addressCity, addressZip, latitude, longitude, images, averageRating, reviewCount, priceRange, featured, verified, isActive, createdAt, updatedAt, distance (optional number for geo-search results)

**review.ts:**
- `ReviewCreate` = z.infer from reviewCreateSchema
- `ReviewReply` = z.infer from reviewReplySchema
- `ReviewListQuery` = z.infer from reviewListQuerySchema
- `Review` = response type with id, uuid, companyId, customerId, bookingId, serviceId, employeeId, rating, comment, redirectedTo, isPublished, reply, repliedAt, deletedAt, createdAt, updatedAt
- `ReviewStatus` = 'pending' | 'approved' | 'rejected'

**video.ts:**
- `VideoProvider` = 'zoom' | 'google_meet' | 'ms_teams'
- `VideoMeetingCreate` = z.infer from videoMeetingCreateSchema
- `VideoMeetingListQuery` = z.infer from videoMeetingListQuerySchema
- `VideoMeeting` = response type with id, uuid, companyId, bookingId, provider, meetingUrl, meetingId, hostUrl, password, startTime, durationMinutes, status, createdAt, updatedAt
- `VideoMeetingStatus` = 'scheduled' | 'started' | 'ended' | 'cancelled'

**whitelabel.ts:**
- `WhitelabelAppCreate` = z.infer from whitelabelAppCreateSchema
- `WhitelabelAppUpdate` = z.infer from whitelabelAppUpdateSchema
- `WhitelabelApp` = response type with uuid, appName, bundleId, logoUrl, primaryColor, secondaryColor, features, iosStatus, androidStatus, iosAppStoreUrl, androidPlayStoreUrl, lastBuildAt, createdAt, updatedAt
- `WhitelabelBuildStatus` = 'draft' | 'building' | 'submitted' | 'published' | 'rejected'

Update `packages/shared/src/types/index.ts` to re-export all new types with `export type { ... } from './marketplace'` etc. Never expose SERIAL IDs in response types - use UUID for public-facing identifiers.
