  <action>
Create the video provider abstraction layer in packages/shared/src/video-providers/.

**VideoProvider.interface.ts:**
- Export `VideoMeetingResult` interface: meetingUrl (string), hostUrl (string), meetingId (string), password (string | null), providerResponse (Record<string, unknown>)
- Export `VideoProvider` interface with methods:
  - `createMeeting(params: { topic: string; startTime: Date; durationMinutes: number; hostEmail: string }): Promise<VideoMeetingResult>`
  - `deleteMeeting(meetingId: string): Promise<void>`
- Export `VideoProviderError` class extending Error with code property

**ZoomProvider.ts:**
- Implements VideoProvider
- Constructor takes `accountId`, `clientId`, `clientSecret`
- `createMeeting`: POST to `https://api.zoom.us/v2/users/me/meetings` with type=2 (scheduled), timezone='Europe/Prague', waiting_room=true
- `deleteMeeting`: DELETE to `https://api.zoom.us/v2/meetings/{meetingId}`
- Private `getAccessToken()`: Server-to-Server OAuth via `https://zoom.us/oauth/token` with grant_type=account_credentials. Use in-memory cache with 55-minute expiry (tokens last 1 hour). Use Buffer.from for Basic auth header.
- On API error: throw VideoProviderError with provider-specific message

**GoogleMeetProvider.ts:**
- Implements VideoProvider
- Constructor takes `clientEmail`, `privateKey`
- `createMeeting`: Create calendar event with conferenceData using Google Calendar API
  - POST to `https://www.googleapis.com/calendar/v3/calendars/primary/events?conferenceDataVersion=1`
  - Include `conferenceData: { createRequest: { requestId: randomUUID, conferenceSolutionKey: { type: 'hangoutsMeet' } } }`
  - Auth: Generate JWT from service account credentials, exchange for access token
  - Extract meetingUrl from conferenceData.entryPoints[0].uri
  - meetingId = event.id
- `deleteMeeting`: DELETE calendar event
- Private `getAccessToken()`: JWT-based service account auth with caching

**MSTeamsProvider.ts:**
- Implements VideoProvider
- Constructor takes `tenantId`, `clientId`, `clientSecret`
- `createMeeting`: POST to `https://graph.microsoft.com/v1.0/me/onlineMeetings`
  - Body: subject, startDateTime, endDateTime, lobbyBypassSettings
  - Auth: Client credentials flow via `https://login.microsoftonline.com/{tenantId}/oauth2/v2.0/token`
  - Extract meetingUrl from joinWebUrl, hostUrl from joinWebUrl
- `deleteMeeting`: DELETE online meeting
- Private `getAccessToken()`: Client credentials OAuth with caching

**index.ts:**
- Export `createVideoProvider(provider: 'zoom' | 'google_meet' | 'ms_teams'): VideoProvider`
- Factory reads env vars:
  - zoom: ZOOM_ACCOUNT_ID, ZOOM_CLIENT_ID, ZOOM_CLIENT_SECRET
  - google_meet: GOOGLE_CLIENT_EMAIL, GOOGLE_PRIVATE_KEY
  - ms_teams: MS_TEAMS_TENANT_ID, MS_TEAMS_CLIENT_ID, MS_TEAMS_CLIENT_SECRET
- If required env vars missing for requested provider: throw VideoProviderError('PROVIDER_NOT_CONFIGURED', `${provider} credentials not configured`)
- Re-export all interfaces and providers

All providers use fetch() for HTTP calls (consistent with existing api-client.ts pattern). No external SDK dependencies needed - use raw HTTP APIs. This avoids package installation and keeps the bundle lean.

Create video meeting API endpoints using createRouteHandler pattern.

**POST /api/v1/video/meetings:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.BOOKINGS_UPDATE]`
- Validate body with `videoMeetingCreateSchema` (bookingUuid, provider)
- Find company ID via `findCompanyId(user.sub)`
- Resolve booking by UUID, verify it belongs to company
- Verify booking status is 'confirmed' or 'pending' (no meetings for cancelled/completed)
- Check if video meeting already exists for this booking (SELECT from videoMeetings WHERE bookingId) - return 409 if exists
- Create video provider via `createVideoProvider(provider)` from packages/shared
- Wrap provider.createMeeting() in try/catch:
  - On success: INSERT into videoMeetings table with companyId, bookingId (SERIAL), provider, meetingUrl, hostUrl, meetingId, password, startTime (from booking), durationMinutes (from booking's service duration), status='scheduled', providerResponse
  - On VideoProviderError with code 'PROVIDER_NOT_CONFIGURED': return 503 with message "Video provider not configured"
  - On other error: return 502 with message "Failed to create video meeting"
- Return 201 with video meeting data (UUID, meetingUrl, hostUrl, password)

**GET /api/v1/video/meetings/[id]:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.BOOKINGS_READ]`
- Accept UUID in path param
- Find company ID, query videoMeetings by UUID WHERE companyId matches
- Return 404 if not found
- Return meeting detail with UUID, meetingUrl, hostUrl, provider, status, startTime, durationMinutes

**DELETE /api/v1/video/meetings/[id]:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.BOOKINGS_UPDATE]`
- Accept UUID in path param
- Find company ID, find meeting by UUID WHERE companyId matches
- Return 404 if not found
- Try to delete meeting via provider: `createVideoProvider(meeting.provider).deleteMeeting(meeting.meetingId)`
  - Wrap in try/catch - if provider deletion fails, log error but still update local status
- UPDATE videoMeetings SET status='cancelled', updatedAt=new Date()
- Return 204

Import `videoMeetings` from `@schedulebox/database`. Import `createVideoProvider` from `@schedulebox/shared/video-providers` (or the appropriate import path based on package exports). All video provider calls are wrapped in try/catch with graceful degradation.
