  <action>
Create review list and create endpoints following createRouteHandler pattern.

**GET /api/v1/reviews:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.BOOKINGS_READ]` (reviews are tied to bookings)
- Find company ID via `findCompanyId(user.sub)`
- Validate query with `reviewListQuerySchema`
- Query reviews WHERE `companyId = companyId` AND `deletedAt IS NULL`
- If `rating_min` provided: filter WHERE `rating >= rating_min`
- If `status` provided: filter by isPublished (approved = isPublished true, pending/rejected = map accordingly)
  - Note: reviews table has `isPublished` boolean, not a status column. Map: 'approved' = isPublished true, 'pending' = isPublished false AND reply IS NULL, 'rejected' = isPublished false AND deletedAt IS NOT NULL
  - Simplification: treat `isPublished = false` as pending (needs moderation)
- JOIN with customers to get customer name, JOIN with services to get service name
- Order by createdAt DESC
- Return paginatedResponse with review data (UUID, not SERIAL ID)

**POST /api/v1/reviews:**
- `requiresAuth: true` (customer role)
- Validate body with `reviewCreateSchema` (bookingUuid, rating, comment)
- Resolve booking by UUID: query bookings WHERE `uuid = bookingUuid`
- Validate:
  1. Booking exists and belongs to user's customer record (find customer by user ID)
  2. Booking status = 'completed' (only completed bookings can be reviewed)
  3. No existing review for this bookingId (UNIQUE check): query reviews WHERE `bookingId = booking.id` - return 409 DUPLICATE_REVIEW if found
- Auto-moderation logic:
  1. Count customer's existing reviews to determine if first-time reviewer
  2. `needsModeration = true` if: rating <= 3 OR isFirstReview OR (comment && comment.length < 20)
  3. Set `isPublished = !needsModeration`
- INSERT review with: companyId (from booking), customerId (from booking), bookingId (SERIAL from resolved booking), serviceId (from booking), employeeId (from booking), rating, comment, isPublished, redirectedTo (if rating >= 4: 'google', else 'internal')
- Publish review.created event using `createReviewCreatedEvent` from `@schedulebox/events` (fire-and-forget, wrapped in try/catch)
- Return 201 with review data and `redirect_to` field (null for low ratings, 'google' redirect URL placeholder for 4-5 stars)

Important: The customer needs to be identified from the JWT. Use the user's sub (UUID) to find their customer record. If user has no customer record, return 403.

Create review detail/delete and reply endpoints.

**GET /api/v1/reviews/[id]:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.BOOKINGS_READ]`
- Accept UUID in path param `id`
- Find company ID via `findCompanyId`
- Query review by UUID WHERE companyId matches AND deletedAt IS NULL
- JOIN customer name, service name, employee name
- Return 404 if not found
- Return review detail with all fields

**DELETE /api/v1/reviews/[id]:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.SETTINGS_MANAGE]` (only owner/admin can delete reviews)
- Accept UUID in path param
- Find company ID via `findCompanyId`
- Soft delete: UPDATE review SET `deletedAt = new Date()`, `isPublished = false` WHERE uuid matches AND companyId matches
- Return 404 if review not found
- Return 204 on success

**POST /api/v1/reviews/[id]/reply:**
- `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.SETTINGS_MANAGE]` (only owner/admin can reply)
- Accept UUID in path param
- Validate body with `reviewReplySchema` (reply text)
- Find company ID via `findCompanyId`
- Find review by UUID WHERE companyId matches AND deletedAt IS NULL
- Return 404 if not found
- UPDATE review SET `reply = data.reply`, `repliedAt = new Date()`, `updatedAt = new Date()`
- If review was pending (isPublished = false), this is also an implicit approval: SET `isPublished = true`
- Return updated review

All endpoints use UUID in path params per API conventions (never expose SERIAL ID). Import `reviews` from `@schedulebox/database`.
