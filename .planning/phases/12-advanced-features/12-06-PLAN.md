  <action>
Create 3 public (no auth) API endpoints for the public booking page data.

**GET /api/v1/public/company/[slug]:**
- No auth required (public endpoint)
- Find company by slug from `companies` table WHERE `slug = params.slug`
- If not found: return 404
- Find marketplace listing for this company (LEFT JOIN or separate query)
- Return company info: name, slug, description (from marketplace listing), address (from marketplace listing), averageRating, reviewCount, images, category, priceRange
- Do NOT expose companyId (SERIAL). Return company UUID if needed.
- Include settings.logoUrl, settings.primaryColor if available from company settings

**GET /api/v1/public/company/[slug]/services:**
- No auth required
- Find company by slug
- If not found: return 404
- Query services WHERE companyId matches AND isActive=true AND deletedAt IS NULL
- Return services: uuid, name, description, durationMinutes, price, currency, category (from service_categories), isOnline
- Order by category name, then service name

**GET /api/v1/public/company/[slug]/reviews:**
- No auth required
- Find company by slug
- If not found: return 404
- Query reviews WHERE companyId matches AND isPublished=true AND deletedAt IS NULL
- JOIN customers for first name (anonymize: show first name + last initial or "Verified customer")
- Return: uuid, rating, comment, reply, repliedAt, createdAt, customerName (anonymized)
- Order by createdAt DESC
- Pagination: page and limit query params
- Also return aggregate: averageRating, reviewCount, rating distribution (count per star 1-5)

All endpoints use `NextResponse.json()` directly (not createRouteHandler since these are public with no auth). Or use createRouteHandler with `requiresAuth: false` for consistency. Either approach works - prefer createRouteHandler for consistency with the codebase pattern.

Import `companies` from `@schedulebox/database` (from auth.ts schema). Import `services`, `reviews`, `marketplaceListings` as needed.
