1. **Create services/ai/app/services/voice_processor.py** -- Audio processing pipeline:

   Constants:
   - MAX_AUDIO_SIZE = settings.MAX_AUDIO_SIZE_MB * 1024 * 1024 (from config)
   - ALLOWED_CONTENT_TYPES = {"audio/webm", "audio/wav", "audio/mp3", "audio/mpeg", "audio/mp4", "audio/m4a", "audio/ogg"}

   Class `VoiceProcessor`:
   - `async def process(self, audio: UploadFile, language: str, company_id: int) -> dict`:

     **Step 1: Validate audio**
     - Read audio bytes: `content = await audio.read()`
     - Check size: if `len(content) > MAX_AUDIO_SIZE` return error response with `{"error": "audio_too_large", "intent": "unknown", "transcript": None, "fallback": True}`
     - Log file size and content type for debugging

     **Step 2: Transcribe with Whisper**
     - Call `transcribe_audio(audio_bytes=content, filename=audio.filename or "recording.webm", language=language)` from openai_client
     - Wrap in try/except: on failure return `{"transcript": None, "intent": "unknown", "error": "transcription_failed", "fallback": True}`
     - Check transcript quality: if empty or `len(transcript.strip()) < 3` return `{"transcript": transcript, "intent": "unknown", "error": "empty_transcription", "entities": None, "fallback": True}`

     **Step 3: Extract intent with GPT-4**
     - Call `extract_intent(transcript)` from openai_client
     - Wrap in try/except: on failure return `{"transcript": transcript, "intent": "unknown", "entities": None, "error": "intent_extraction_failed", "fallback": True}`

     **Step 4: Return structured result**
     ```python
     return {
         "transcript": transcript,
         "intent": intent_result["intent"],
         "entities": intent_result["entities"],
         "confidence": intent_result["confidence"],
         "confirmation_needed": intent_result["intent"] == "create_booking",
         "error": None,
         "fallback": False,
     }
     ```

   Use logging throughout. Import from `..services.openai_client` and `..config`.

2. **Create services/ai/app/routers/voice.py** -- FastAPI router:

   Follow exact pattern of existing routers (predictions.py, optimization.py).

   ```python
   from fastapi import APIRouter, UploadFile, File, Form, HTTPException
   from ..services.voice_processor import VoiceProcessor
   from ..schemas.responses import VoiceProcessResponse

   router = APIRouter(prefix="/voice", tags=["voice"])

   @router.post("/process", response_model=VoiceProcessResponse)
   async def process_voice(
       audio: UploadFile = File(..., description="Audio file (webm, wav, mp3, mp4, m4a, ogg)"),
       language: str = Form(default="cs", description="Language code: cs, sk, en"),
       company_id: int = Form(..., description="Company ID for context"),
   ):
       """
       Process voice audio for booking intent extraction.

       Pipeline: Audio -> Whisper STT (transcription) -> GPT-4 NLU (intent extraction)

       Returns transcript, intent (create_booking/cancel_booking/check_availability/unknown),
       extracted entities (service, date, time, employee, customer), and confidence score.

       Does NOT create bookings -- returns intent for Node.js to handle via existing booking flow.
       """
       processor = VoiceProcessor()
       result = await processor.process(audio, language, company_id)
       return result
   ```

   The router accepts multipart/form-data with audio file and metadata fields.
   The response_model validates the output against VoiceProcessResponse Pydantic model.
   No authentication on the Python side -- auth is handled by the Node.js layer (createRouteHandler).

   Follows the existing API route pattern (see ai/optimization/upselling/route.ts):

   - RBAC via `PERMISSIONS` from `@/lib/middleware/rbac`
   - Uses `successResponse` from `@/lib/utils/response`

   Special handling for multipart/form-data:
   - Since createRouteHandler expects JSON body parsing by default, this route needs to handle multipart differently
   - Use createRouteHandler WITHOUT bodySchema (audio comes as FormData, not JSON)
   - Inside handler, manually parse the request:
     ```typescript
     const formData = await request.formData();
     const audioFile = formData.get('audio') as File | null;
     const language = (formData.get('language') as string) || 'cs';
     ```
   - Validate audio file exists and is within size limit (10MB)
   - Validate language is one of 'cs', 'sk', 'en'

   Build a new FormData to proxy to the AI service:
   ```typescript
   const proxyFormData = new FormData();
   proxyFormData.append('audio', audioFile);
   proxyFormData.append('language', language);
   proxyFormData.append('company_id', String(companyId));
   ```

   Call circuit breaker:
   ```typescript
   try {
     const result = await processVoice.fire(proxyFormData);
     return successResponse(result);
   } catch {
     const fallback = getVoiceProcessFallback();
     return successResponse(fallback);
   }
   ```

   Import `processVoice` from `@/lib/ai/client` and `getVoiceProcessFallback` from `@/lib/ai/fallback`.

   Note: The `request` object is available in createRouteHandler's handler -- check how the existing route handler passes the raw request. If createRouteHandler doesn't pass the raw Request, use a custom handler wrapper that accesses it. Follow whatever pattern the existing codebase uses for accessing the raw request.

   The route intentionally does NOT create a booking. It returns the AI's interpretation. The frontend will display a confirmation UI and call the existing POST /bookings endpoint to create the actual booking.
