1. **Create services/ai/app/services/followup_prompts.py** -- Prompt templates and budget:

   Define `FOLLOW_UP_TEMPLATES` dict with 4 template types, each having:
   - `system`: System prompt instructing GPT-4 to write Czech marketing email
   - `context_template`: Template string with {placeholders} for customer context

   **post_visit template:**
   - System: "You are a friendly marketing assistant for a Czech service business. Write a personalized follow-up email in Czech for a customer who recently visited. Keep the tone warm and personal. Include a subtle upsell if the customer data suggests it. Output JSON with 'subject' and 'body' fields. Body should be plain text with line breaks."
   - Context template: Customer name, last visit date, service, total visits, health score, preferred services, business name

   **re_engagement template:**
   - System: "You are a marketing assistant for a Czech service business. Write a re-engagement email in Czech for a customer who hasn't visited in a while. Be warm but not pushy. Mention what they're missing. Optionally include a time-limited offer. Output JSON with 'subject' and 'body' fields."
   - Context template: Customer name, days inactive, preferred services, total visits, total spent CZK, health score + category, business name

   **upsell template:**
   - System: "You are a marketing assistant for a Czech service business. Write an upsell email in Czech suggesting a complementary service. Reference the customer's history to make it personal. Output JSON with 'subject' and 'body' fields."
   - Context template: Customer name, recent services, recommended service, recommendation reason, business name

   **birthday template:**
   - System: "You are a marketing assistant for a Czech service business. Write a birthday congratulation email in Czech with a special offer. Be celebratory and personal. Include a birthday discount or gift. Output JSON with 'subject' and 'body' fields."
   - Context template: Customer name, total visits, loyalty tier, preferred services, business name

   Define `build_prompt(template_type: str, customer_context: dict) -> tuple[str, str]`:
   - Gets template from FOLLOW_UP_TEMPLATES
   - Formats context_template with customer_context dict (use .format_map with defaultdict returning "N/A" for missing keys)
   - Returns (system_prompt, formatted_context)

   Define `check_token_budget(system_prompt: str, context: str, model: str, max_tokens: int = 2000) -> str`:
   - Uses tiktoken to count tokens
   - If total exceeds max_tokens, truncates context to fit
   - Returns (possibly truncated) context string

2. **Create services/ai/app/routers/followup.py** -- FastAPI router:

   ```python
   import json
   import logging
   from collections import defaultdict
   from fastapi import APIRouter, HTTPException
   from ..schemas.requests import FollowUpRequest
   from ..schemas.responses import FollowUpResponse
   from ..services.openai_client import generate_followup_text
   from ..services.followup_prompts import FOLLOW_UP_TEMPLATES, build_prompt, check_token_budget
   from ..config import settings

   logger = logging.getLogger(__name__)
   router = APIRouter(prefix="/followup", tags=["followup"])

   # Simple in-memory rate limiter (per company, per day)
   # In production, use Redis for distributed rate limiting
   _rate_limits: dict[str, int] = defaultdict(int)
   _rate_limit_day: str = ""

   def _check_rate_limit(company_id: int) -> bool:
       """Check if company has exceeded daily follow-up limit."""
       import datetime
       nonlocal _rate_limit_day  # Actually use global
       today = datetime.date.today().isoformat()
       global _rate_limits, _rate_limit_day
       if _rate_limit_day != today:
           _rate_limits.clear()
           _rate_limit_day = today
       key = str(company_id)
       if _rate_limits[key] >= settings.MAX_FOLLOWUP_PER_DAY:
           return False
       _rate_limits[key] += 1
       return True
   ```

   Actually, use a cleaner approach: module-level dict with reset logic.

   ```python
   @router.post("/generate", response_model=FollowUpResponse)
   async def generate_followup(request: FollowUpRequest):
       """
       Generate personalized follow-up email content using AI.

       Uses gpt-4o-mini for cost-effective email generation.
       Enforces token budget (2000 tokens max prompt) and daily rate limit per company.
       Returns subject + body in Czech.
       """
       # Rate limit check
       if not _check_rate_limit(request.company_id):
           return FollowUpResponse(
               subject="",
               body="",
               error="rate_limit_exceeded",
               fallback=True,
           )

       # Validate template type
       if request.type not in FOLLOW_UP_TEMPLATES:
           return FollowUpResponse(
               subject="",
               body="",
               error="invalid_template_type",
               fallback=True,
           )

       try:
           # Build prompt from template + customer context
           system_prompt, context = build_prompt(
               request.type,
               request.customer_context.model_dump(),
           )

           # Enforce token budget
           context = check_token_budget(
               system_prompt, context,
               model=settings.OPENAI_FOLLOWUP_MODEL,
           )

           # Generate with GPT-4o-mini
           result = await generate_followup_text(
               system_prompt=system_prompt,
               context=context,
           )

           return FollowUpResponse(
               subject=result["subject"],
               body=result["body"],
               model=result.get("model", settings.OPENAI_FOLLOWUP_MODEL),
               tokens_used=result.get("tokens_used", 0),
               fallback=False,
           )
       except Exception as e:
           logger.error(f"Follow-up generation failed: {e}")
           return FollowUpResponse(
               subject="",
               body="",
               error="generation_failed",
               fallback=True,
           )
   ```

   The rate limiter uses module-level state. For MVP this is fine (single process). Document that Redis-based rate limiting should be added for production multi-process deployment.

1. **Create apps/web/app/api/v1/ai/follow-up/route.ts** -- POST endpoint following exact pattern of optimization routes:

   ```typescript
   /**
    * AI Follow-Up Email Generation Endpoint
    * POST /api/v1/ai/follow-up
    *
    * Generates personalized follow-up email content using GPT-4o-mini.
    * Accepts customer context and template type, returns subject + body in Czech.
    * Uses circuit breaker: returns empty content (200) when AI unavailable.
    * Non-critical, advisory -- never blocks other operations.
    */
   ```

   Pattern (matching upselling/route.ts exactly):
   - `import { createRouteHandler } from '@/lib/middleware/route-handler'`
   - `import { PERMISSIONS } from '@/lib/middleware/rbac'`
   - `import { successResponse } from '@/lib/utils/response'`
   - `import { generateFollowUp } from '@/lib/ai/client'`
   - `import { getFollowUpFallback } from '@/lib/ai/fallback'`
   - `import { followUpRequestSchema } from '@schedulebox/shared'`

   Use createRouteHandler with:
   - `bodySchema: followUpRequestSchema`
   - `requiresAuth: true`
   - `requiredPermissions: [PERMISSIONS.SETTINGS_MANAGE]` -- Follow-up generation is an admin/owner feature, not customer-facing. Use SETTINGS_MANAGE (same as AI health endpoint in Phase 10-04).

   Handler:
   ```typescript
   handler: async ({ body, companyId }) => {
     try {
       // Add company_id from auth context (not from body)
       const request = { ...body, company_id: companyId };
       const result = await generateFollowUp.fire(request);
       return successResponse(result);
     } catch {
       // Follow-up is advisory -- return 200 with fallback
       const fallback = getFollowUpFallback({ ...body, company_id: companyId ?? 0 });
       return successResponse(fallback);
     }
   }
   ```

   Note: company_id comes from the authenticated user's context (resolved by createRouteHandler via findCompanyId), NOT from the request body. This prevents users from generating follow-ups for other companies.
