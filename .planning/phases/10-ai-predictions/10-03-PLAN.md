  <action>
    **services/ai/app/services/feature_store.py:** Create Redis-backed feature store:
    - Use `redis.asyncio` for async Redis client.
    - Module-level `_redis_client = None`.
    - `async def get_redis_client()`: Lazy-initialize Redis client from settings.REDIS_URL. Return client. Handle connection errors gracefully (log warning, return None).
    - `async def get_booking_features(booking_id: int) -> dict | None`: Try to get from Redis key `features:booking:{booking_id}`. If cached, return json.loads(cached). If not cached or Redis unavailable, return None (caller will use request-provided features or compute).
    - `async def cache_booking_features(booking_id: int, features: dict, ttl: int = 3600)`: Store features in Redis with 1-hour TTL. Handle errors gracefully (just log, don't fail).
    - `async def get_customer_features(customer_id: int) -> dict | None`: Same pattern with key `features:customer:{customer_id}`.
    - `async def cache_customer_features(customer_id: int, features: dict, ttl: int = 3600)`: Same pattern.
    - `async def close()`: Close Redis connection if open.

    **services/ai/app/routers/predictions.py:** Create predictions router with 4 endpoints:

    `POST /predictions/no-show` (response_model=NoShowPredictionResponse):
    - Accept NoShowPredictionRequest body.
    - Get no-show model from model_loader.get_no_show_model().
    - If model is None AND request.features is None, return fallback response (no_show_probability=0.15, confidence=0.0, risk_level="low", model_version="fallback", fallback=True).
    - Try to get features from feature_store.get_booking_features(request.booking_id).
    - If no cached features and request.features provided, use request.features.
    - If no features at all, use default features.
    - Call model.predict(features) (or heuristic if no model).
    - Cache features if computed.
    - Return NoShowPredictionResponse with fallback=False.
    - Wrap in try/except: on any error, return fallback response with fallback=True.

    `POST /predictions/clv` (response_model=CLVPredictionResponse):
    - Same pattern as no-show but for CLV prediction.
    - Get clv model from model_loader.get_clv_model().
    - Fallback: clv_predicted=0, confidence=0.0, segment="low", fallback=True.

    `POST /predictions/health-score` (response_model=HealthScoreResponse):
    - Accept HealthScorePredictionRequest body.
    - Get health_score calculator from model_loader.get_health_score_model().
    - Health score calculator is always available (pure RFM, no serialized model needed).
    - Call calculator.calculate(request.recency_days, request.frequency, request.monetary).
    - Return HealthScoreResponse with fallback=False.
    - On error, return fallback with health_score=50, category="good".

    `POST /predictions/health-score/batch` (response_model=BatchHealthScoreResponse):
    - Accept BatchHealthScoreRequest body (list of customers).
    - Get health_score calculator.
    - Call calculator.calculate_batch() for all customers.
    - Return BatchHealthScoreResponse with list of predictions.

    Register all routes with `router = APIRouter(prefix="/predictions", tags=["predictions"])`.

    IMPORTANT: Also update services/ai/app/main.py to include the predictions router if not already done:
    ```python
    app.include_router(predictions.router, prefix="/api/v1", tags=["predictions"])
    ```

    **services/ai/scripts/train_no_show.py:** Create XGBoost no-show training script:
    - Import xgboost, pandas, numpy, sklearn (TimeSeriesSplit, roc_auc_score, precision_score, recall_score, f1_score), joblib, json, datetime.
    - `load_training_data(api_url: str) -> pd.DataFrame`: Placeholder that makes HTTP GET to `{api_url}/api/internal/features/training/no-show` using httpx. For now, generate synthetic training data (500 rows) with realistic feature distributions if API unavailable. Features: booking_lead_time_hours (exponential, mean 48h), customer_no_show_rate (beta, mean 0.12), customer_total_bookings (poisson, mean 5), day_of_week (uniform 0-6), hour_of_day (normal, mean 14, std 3), is_weekend (from day_of_week), service_duration_minutes (choice [30, 45, 60, 90, 120]), service_price (uniform 300-3000), is_first_visit (bernoulli 0.3), has_payment (bernoulli 0.6), days_since_last_visit (exponential, mean 30). Target: no_show (bernoulli, probability = sigmoid of linear combination of customer_no_show_rate * 3 + is_first_visit * 0.5 - has_payment * 1.0 + random noise)).
    - `engineer_features(df: pd.DataFrame) -> tuple[pd.DataFrame, pd.Series]`: Extract feature columns in correct order (matching NoShowPredictor.FEATURE_COLUMNS), return X and y.

Train with TimeSeriesSplit(n_splits=3). Save model to `{output_dir}/no_show_v1.0.0.joblib`. Print metrics (AUC, precision, recall, F1). Update metadata.json.

Target: future_clv = total_spent * (1 + log(booking_frequency+1)) * (1 - no_show_rate) + noise.
