  <name>Task 2: Create circuit breaker factory, fallback functions, and AI HTTP client</name>
  <files>

  <action>
    **apps/web/lib/ai/circuit-breaker.ts:** Create the generic circuit breaker factory:
    - Import CircuitBreaker from 'opossum' (default import).
    - Define `AI_CIRCUIT_BREAKER_OPTIONS` constant: timeout=5000, errorThresholdPercentage=50, resetTimeout=30000, rollingCountTimeout=10000, rollingCountBuckets=10, volumeThreshold=5.
    - Export `createAICircuitBreaker<TArgs, TResult>(fn, fallbackFn, customOptions?)`: Creates and returns a CircuitBreaker instance. Attach the fallbackFn via `breaker.fallback()`. Add event listeners for 'open' (console.warn), 'halfOpen' (console.info), 'close' (console.info), 'fallback' (console.warn with result). Return the breaker.
    - Export `getCircuitBreakerHealth(breaker)`: Returns AIServiceHealth object with status (healthy/degraded/unhealthy based on circuit state), state (OPEN/CLOSED/HALF_OPEN), and stats (successes, failures, fallbacks, timeouts from breaker.stats). Check breaker.opened for OPEN, breaker.halfOpen for HALF_OPEN, else CLOSED.

    **apps/web/lib/ai/fallback.ts:** Create fallback functions that return sensible defaults:
    - `getNoShowFallback(request: NoShowPredictionRequest): NoShowPredictionResponse` — Returns { booking_id: request.booking_id, no_show_probability: 0.15, confidence: 0.0, risk_level: 'low', model_version: 'fallback', fallback: true }.
    - `getCLVFallback(request: CLVPredictionRequest): CLVPredictionResponse` — Returns { customer_id: request.customer_id, clv_predicted: 0, confidence: 0.0, segment: 'low', model_version: 'fallback', fallback: true }. Note: caller should compute from customer.total_spent * 2.5 if available.
    - `getHealthScoreFallback(request: HealthScorePredictionRequest): HealthScoreResponse` — Returns { customer_id: request.customer_id, health_score: 50, category: 'good', model_version: 'fallback', fallback: true }.

    **apps/web/lib/ai/client.ts:** Create the AI service HTTP client with circuit breaker wrapping:
    - Get AI_SERVICE_URL from `process.env.AI_SERVICE_URL` with fallback to 'http://localhost:8000'.
    - Create private async functions (NOT exported directly):
      - `callNoShowAPI(request: NoShowPredictionRequest): Promise<NoShowPredictionResponse>` — POST to `${AI_SERVICE_URL}/api/v1/predictions/no-show` with JSON body. Throw on non-2xx status.
      - `callCLVAPI(request: CLVPredictionRequest): Promise<CLVPredictionResponse>` — POST to `${AI_SERVICE_URL}/api/v1/predictions/clv`.
      - `callHealthScoreAPI(request: HealthScorePredictionRequest): Promise<HealthScoreResponse>` — POST to `${AI_SERVICE_URL}/api/v1/predictions/health-score`.
    - Create circuit breaker instances (module-level singletons):
      - `export const predictNoShow = createAICircuitBreaker(callNoShowAPI, getNoShowFallback)`
      - `export const predictCLV = createAICircuitBreaker(callCLVAPI, getCLVFallback)`
      - `export const predictHealthScore = createAICircuitBreaker(callHealthScoreAPI, getHealthScoreFallback)`
    - Export `getAIServiceStatus(): AIServiceHealth` that returns health status from the no-show circuit breaker (representative of AI service health).

    IMPORTANT: Use `fetch()` (not axios) for HTTP calls to stay consistent with existing codebase patterns. The response should be parsed as JSON with proper error handling (check response.ok, throw Error with status on failure).
  </action>
