    - `BatchHealthScoreRequest`: customers (list of HealthScorePredictionRequest)

    - `NoShowPredictionResponse`: booking_id (int), no_show_probability (float), confidence (float), risk_level (Literal["low", "medium", "high"]), model_version (str), fallback (bool)
    - `CLVPredictionResponse`: customer_id (int), clv_predicted (float), confidence (float), segment (Literal["low", "medium", "high", "premium"]), model_version (str), fallback (bool)
    - `HealthScoreResponse`: customer_id (int), health_score (int, ge=0, le=100), category (Literal["excellent", "good", "at_risk", "churning"]), rfm_details (optional dict with r_score, f_score, m_score), model_version (str), fallback (bool)
    - `BatchHealthScoreResponse`: predictions (list of HealthScoreResponse)
    - `ModelInfoResponse`: model_name (str), model_version (str), trained_at (optional str), features (list of str), metrics (optional dict)

Wrap in try/except per model - log warning if file not found but continue.     - `def is_models_loaded()`: Returns bool.

    - `predict(self, features: dict) -> dict`: Extract features in correct order from dict, fill missing with defaults (booking_lead_time_hours=24, customer_no_show_rate=0.15, customer_total_bookings=1, day_of_week=1, hour_of_day=10, is_weekend=0, service_duration_minutes=60, service_price=500, is_first_visit=1, has_payment=0, days_since_last_visit=999). If self.model is None, return heuristic prediction: calculate probability as weighted combination of customer_no_show_rate (weight 0.6) + first_visit_factor (0.05 if first visit) + lead_time_factor (higher prob for very short or very long lead times) + no_payment_factor (0.1 if no payment). Clamp to [0, 1]. If model exists, use model.predict_proba() to get probability. Determine risk_level: >= 0.5 "high", >= 0.3 "medium", else "low". Return dict with no_show_probability, confidence (0.82 if model, 0.4 if heuristic), risk_level.
    - `@classmethod from_file(cls, path: str)`: Load model from joblib file, return instance.

    **services/ai/app/models/clv.py:** Create `CLVPredictor` class:
    - `__init__(self, model=None, model_version="v1.0.0")`: Store Random Forest model and version.
    - `FEATURE_COLUMNS` class attribute: list of 8 features (total_bookings, total_spent, avg_booking_value, days_since_first_visit, days_since_last_visit, booking_frequency, no_show_rate, service_diversity).
    - `predict(self, features: dict) -> dict`: Extract features, fill missing with defaults. If self.model is None, return heuristic: CLV = total_spent * 2.5 * (1 - no_show_rate) * frequency_factor (log scale). If model exists, use model.predict() for regression output. Determine segment: >= 50000 "premium", >= 20000 "high", >= 5000 "medium", else "low". Return dict with clv_predicted, confidence (0.75 if model, 0.3 if heuristic), segment.
    - `@classmethod from_file(cls, path: str)`: Load model from joblib file.

    **services/ai/app/models/health_score.py:** Create `HealthScoreCalculator` class:
    - `__init__(self, model_version="v1.0.0")`: No ML model needed - pure RFM calculation.
    - `WEIGHTS` class attribute: dict with recency=0.40, frequency=0.35, monetary=0.25.
    - `CATEGORIES` class attribute: dict mapping score ranges to categories: >= 80 "excellent", >= 60 "good", >= 40 "at_risk", else "churning".
    - `calculate(self, recency_days: float, frequency: int, monetary: float) -> dict`: Calculate RFM scores. Recency: score inversely proportional to days (0 days = 100, 365+ days = 0, linear interpolation). Frequency: score proportional to frequency (0 = 0, 20+ = 100, linear interpolation). Monetary: score proportional to monetary (0 = 0, 100000+ CZK = 100, linear interpolation). Weighted sum: R * 0.4 + F * 0.35 + M * 0.25 (0-100 scale). Round to integer. Categorize using CATEGORIES thresholds. Return dict with health_score, category, rfm_details (r_score, f_score, m_score).
    - `calculate_batch(self, customers: list[dict]) -> list[dict]`: Calculate health scores for multiple customers.

    - `compute_clv_features(customer_data: dict) -> dict`: Compute the 8 features needed for CLV prediction from raw customer data.
    - `compute_rfm(customer_data: dict) -> tuple[float, int, float]`: Extract recency_days, frequency, monetary from customer data dict.
