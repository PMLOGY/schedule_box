<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure Twilio account and Czech phone number</name>
  <action>
The user must:

1. **Create or verify Twilio account** at https://console.twilio.com/
   - If trial account: upgrade to paid (trial cannot send to unverified numbers in CZ)
   - Note: Trial accounts are limited to verified phone numbers only

2. **Purchase a Czech mobile number**
   - Twilio Console -> Phone Numbers -> Buy a Number
   - Country: Czech Republic
   - Capabilities: SMS (voice optional)
   - Choose a +420 6xx or 7xx prefix number (~$12/month)
   - Note: Czech Sender ID registration may be needed for T-Mobile/O2 (3-week wait). For immediate testing, a purchased number works.

3. **Get credentials**
   - Account SID: Twilio Console -> Account Info
   - Auth Token: Twilio Console -> Account Info

4. **Set environment variables** in `.env.local`:
   ```
   TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   TWILIO_AUTH_TOKEN=your-auth-token
   TWILIO_FROM_NUMBER=+420xxxxxxxxx
   ```

5. **Set in Railway** (production):
   - Add TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, TWILIO_FROM_NUMBER to the notification-worker service environment variables
  </action>
  <resume-signal>Type "credentials configured" with the TWILIO_FROM_NUMBER value (phone number only, no secrets) so Claude can verify the setup.</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Verify SMS delivery and set up cost monitoring trigger</name>
  <files></files>
  <action>
After the user confirms Twilio credentials are configured:

1. **Verify Twilio client initializes** by checking env vars are set:
   ```bash
   grep "TWILIO_ACCOUNT_SID" .env.local | head -1
   # Should show ACxxx... (not empty)
   ```

2. **Send a test SMS** to verify delivery works. Create a small test script and run it:
   ```typescript
   // Quick test: send SMS via the existing sms-sender module
   import { sendSMS } from './services/notification-worker/src/services/sms-sender.js';
   ```
   Or use the Twilio CLI / a simple curl:
   ```bash
   npx tsx -e "
   const twilio = require('twilio');
   const client = twilio(process.env.TWILIO_ACCOUNT_SID, process.env.TWILIO_AUTH_TOKEN);
   client.messages.create({
     from: process.env.TWILIO_FROM_NUMBER,
     to: '+420XXXXXXXXX', // User's test number
     body: 'ScheduleBox test: Pripominame termin zitra v 14:00. Salon Krasa'
   }).then(m => console.log('Sent:', m.sid)).catch(e => console.error('Error:', e));
   "
   ```
   Ask the user for their phone number to send the test to.

3. **Run the usage trigger setup script**:
   ```bash
   npx tsx scripts/setup-twilio-usage-trigger.ts
   ```
   This creates a monthly recurring trigger at $50 (or SMS_BUDGET_ALERT_THRESHOLD) that POSTs to /api/v1/webhooks/twilio-usage when threshold is reached.

4. **Verify the trigger was created** by listing triggers:
   ```bash
   npx tsx -e "
   const twilio = require('twilio');
   const client = twilio(process.env.TWILIO_ACCOUNT_SID, process.env.TWILIO_AUTH_TOKEN);
   client.usage.triggers.list().then(triggers => {
     triggers.forEach(t => console.log(t.sid, t.friendlyName, '$' + t.triggerValue, t.recurring));
   });
   "
   ```
  </action>

1. Test SMS was received on the user's phone
2. Usage trigger SID was returned by setup script
3. Usage trigger list shows "ScheduleBox SMS Monthly Budget Alert"
  </verify>
