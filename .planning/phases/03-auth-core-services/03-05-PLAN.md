  <files>

  <action>
    Install dependencies: `cd apps/web && pnpm add qrcode && pnpm add -D @types/qrcode`
    (otplib should already be installed from plan 04)

    Create `apps/web/lib/auth/mfa.ts`:
    - Import { authenticator } from 'otplib'
    - Import QRCode from 'qrcode'
    - Import { nanoid } from 'nanoid'
    - Import db, users schema, eq from drizzle-orm
    - Configure authenticator: digits=6, step=30

    setupMFA(userId: number, userEmail: string):
    - Generate secret: authenticator.generateSecret()
    - Generate 10 backup codes: Array.from({ length: 10 }, () => nanoid(16))
    - Create otpauth URL: authenticator.keyuri(userEmail, 'ScheduleBox', secret)
    - Generate QR code: QRCode.toDataURL(otpauthUrl)
    - Store secret in user record (mfaSecret=secret, but do NOT enable MFA yet — enabled after verify)
    - Return { secret, qr_code_url: qrCodeUrl, backup_codes: backupCodes }

    verifyMFACode(secret: string, code: string): boolean
    - authenticator.verify({ token: code, secret })
    - Return result, catch errors and return false

    enableMFA(userId: number, code: string): Promise<boolean>
    - Fetch user to get mfaSecret
    - If no mfaSecret, return false
    - Verify code against secret
    - If valid, update user: mfaEnabled=true
    - Return true/false

    Create `apps/web/app/api/v1/auth/mfa/setup/route.ts`:
    - Export POST handler (requires auth)
    - Find user by UUID (from JWT sub) to get internal userId and email
    - Call setupMFA(userId, userEmail)
    - Return 200 { data: { secret, qr_code_url, backup_codes } }

    Create `apps/web/app/api/v1/auth/mfa/verify/route.ts`:
    - Export POST handler (requires auth)
    - Validate body with mfaSetupVerifySchema (from validations/auth.ts)
    - Find user by UUID to get internal userId
    - Call enableMFA(userId, code)
    - If success, return 200 { message: 'MFA enabled successfully' }
    - If failure, return 400 ValidationError('Invalid MFA code')
  </action>
  <verify>
    Run `cd apps/web && pnpm tsc --noEmit` — compiles without errors.
    Verify mfa.ts exports setupMFA, verifyMFACode, enableMFA.
    Verify MFA setup endpoint returns secret, qr_code_url, backup_codes.
  </verify>
  <done>
    MFA setup generates TOTP secret + QR code + backup codes. MFA verify enables MFA after code validation. Ready for use in login flow.
  </done>
</task>

    Create `apps/web/app/api/v1/settings/api-keys/route.ts`:
    - Import createRouteHandler from middleware
    - Import nanoid, crypto for key generation
    - Import apiKeys schema from @schedulebox/database

    Export GET = createRouteHandler({
      requiresAuth: true,
      requiredPermissions: ['settings.manage'],
      handler: async ({ user }) => {
        - Find user's companyId from UUID
        - Query api_keys WHERE companyId AND isActive=true
        - Return list with: id, name, key_prefix, scopes, created_at, last_used_at, expires_at
        - NEVER return keyHash
      }
    })

    Export POST = createRouteHandler({
      bodySchema: z.object({ name: z.string().min(1).max(100), scopes: z.array(z.string()).optional().default([]) }),
      requiresAuth: true,
      requiredPermissions: ['settings.manage'],
      handler: async ({ body, user }) => {
        - Generate API key: `sb_live_${nanoid(32)}` (prefix 'sb_live_' for live keys)
        - Extract prefix: first 10 chars of key
        - Hash key with SHA-256
        - Find user's companyId
        - Insert into api_keys: companyId, name, keyHash, keyPrefix, scopes, isActive=true
        - Return 201 { data: { id, name, key: fullApiKey, key_prefix, created_at } }
        - IMPORTANT: The full key is returned ONLY on creation — never retrievable again
      }
    })

    Create `apps/web/app/api/v1/settings/api-keys/[id]/route.ts`:
    Export DELETE = createRouteHandler({
      requiresAuth: true,
      requiredPermissions: ['settings.manage'],
      handler: async ({ req, user }) => {
        - Extract id from params (await params)
        - Parse id as integer
        - Find user's companyId
        - Update api_keys SET isActive=false WHERE id AND companyId (tenant isolation!)
        - If no rows affected, throw NotFoundError
        - Return 204 noContentResponse()
      }
    })
