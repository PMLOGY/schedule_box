<task type="auto">
  <name>Task 2: Create API response utilities and error handler wrapper</name>
  <files>
    apps/web/lib/utils/errors.ts

  <action>
    Create `apps/web/lib/utils/response.ts`:
    - Import NextResponse from 'next/server'
    - Import AppError, PaginationMeta from '@schedulebox/shared'
    - successResponse<T>(data: T, status = 200) — returns NextResponse.json({ data }, { status })
    - createdResponse<T>(data: T) — returns successResponse(data, 201)
    - noContentResponse() — returns new NextResponse(null, { status: 204 })
    - paginatedResponse<T>(data: T[], meta: PaginationMeta) — returns NextResponse.json({ data, meta })
    - errorResponse(error: AppError) — returns NextResponse.json({ error: { code: error.code, message: error.message, details: error.details } }, { status: error.statusCode })
    - validationErrorResponse(errors: Array<{ path: string; message: string }>) — returns NextResponse.json with code VALIDATION_ERROR, status 400, and errors array as details. IMPORTANT: Never include raw input values in error details (security — prevents password leaking in logs)

    Create `apps/web/lib/utils/errors.ts`:
    - Import AppError from '@schedulebox/shared'
    - handleRouteError(error: unknown) — if error is AppError, return errorResponse(error). If error is Error, log to console.error and return errorResponse with InternalError. If unknown, log and return InternalError response.
    - This is a catch-all error handler for use in route handlers.
  </action>
