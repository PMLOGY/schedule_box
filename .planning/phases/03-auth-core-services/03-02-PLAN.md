<task type="auto">
  <name>Task 2: Create JWT and password authentication utilities</name>
  <files>
    apps/web/lib/auth/jwt.ts
    apps/web/lib/auth/password.ts
  </files>
  <action>
    Install dependencies first:
    ```
    cd apps/web && pnpm add jsonwebtoken argon2 nanoid
    pnpm add -D @types/jsonwebtoken
    ```

    Create `apps/web/lib/auth/jwt.ts`:
    - Import jsonwebtoken, nanoid, crypto, redis client, db client, schema tables (refreshTokens, rolePermissions, permissions, users)
    - Import drizzle-orm operators (eq, and)
    - Define JWTPayload interface matching documentation (sec 25.1): sub (user UUID string), iss 'schedulebox', aud 'schedulebox-api', exp, iat, company_id (number), role (string), permissions (string[]), mfa_verified (boolean)
    - JWT_SECRET from process.env.JWT_SECRET (throw if missing in production, use fallback 'dev-secret-change-me' in dev)
    - ACCESS_TOKEN_EXPIRY = '15m'
    - REFRESH_TOKEN_EXPIRY_DAYS = 30

    generateTokenPair(userId: number, userUuid: string, companyId: number, roleId: number, roleName: string, mfaVerified: boolean):
    - Fetch permissions for roleId from role_permissions JOIN permissions
    - Sign access token with claims: sub=userUuid, company_id=companyId, role=roleName, permissions=permissionNames, mfa_verified
    - Generate refresh token: nanoid(64)
    - Hash refresh token with SHA-256
    - Store hash in refresh_tokens table with userId, expiresAt (now + 30 days), revoked=false
    - Return { accessToken, refreshToken, expiresIn: 900 }

    verifyJWT(token: string): Promise<JWTPayload>:
    - Check Redis blacklist first (`blacklist:{token}`)
    - If blacklisted, throw UnauthorizedError('Token has been revoked')
    - jwt.verify with issuer 'schedulebox', audience 'schedulebox-api'
    - Return decoded payload
    - On any error, throw UnauthorizedError('Invalid or expired token')

    rotateRefreshToken(refreshToken: string):
    - Hash token with SHA-256
    - Use db.transaction with SELECT FOR UPDATE on refresh_tokens row (prevents race condition — see research pitfall #2)
    - If not found, revoked, or expired, throw UnauthorizedError
    - Revoke old token (set revoked=true)
    - Fetch user details including role name
    - Generate new token pair via generateTokenPair
    - Return new pair

    blacklistToken(token: string):
    - Decode token (without verification) to get exp
    - Calculate remaining TTL = exp - now
    - If TTL > 0, redis.setex(`blacklist:{token}`, ttl, '1')
    - If TTL <= 0, token is already expired, no-op

    Create `apps/web/lib/auth/password.ts`:
    - Import argon2, db, schema tables (users, passwordHistory), drizzle operators
    - ARGON2_OPTIONS: type argon2.argon2id, memoryCost 65536, timeCost 3, parallelism 4 (matches doc sec 24.1)
    - PASSWORD_HISTORY_LIMIT = 5

    hashPassword(password: string): Promise<string> — hash with ARGON2_OPTIONS
    verifyPassword(hash: string, password: string): Promise<boolean> — argon2.verify, return false on error
    checkPasswordHistory(userId: number, newPassword: string): Promise<boolean> — fetch last 5 hashes, verify against each, return true if password is NOT in history (safe to use), false if found in history
    updatePassword(userId: number, newPassword: string): Promise<void> — hash, update users.passwordHash and passwordChangedAt, insert into passwordHistory, clean up old history entries beyond limit 5

    IMPORTANT: Use `import argon2 from 'argon2'` (default import). For nanoid v5, use `import { nanoid } from 'nanoid'`.
  </action>
