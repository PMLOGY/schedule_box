    - Build `service_id_to_idx` mapping dict from service_ids on init.
    - Method `recommend(current_service_id: int, customer_history: list[int], n_recommendations: int = 3) -> list[dict]`:
      - If similarity_matrix is None or current_service_id not in mapping, return `_popularity_fallback()`.
      - Get similarity scores row for current service.
      - Zero out scores for already-booked services (customer_history) and current service (set to -1).
      - Get top-N indices by score, filter out those with score <= 0.
      - Return list of `{"service_id": int, "confidence": float, "reason": "collaborative_filtering"}`.
      - If fewer than n_recommendations, pad with `_popularity_fallback()`.

    - Classmethod `build_from_bookings(booking_data: list[dict]) -> UpsellRecommender`: Build sparse customer-service matrix from `[{"customer_id": int, "service_id": int, "count": int}]`, compute cosine similarity on item-item (transposed), extract popularity fallback (top 10 by booking count).

    **services/ai/app/models/pricing.py:** Create PricingOptimizer class for Thompson Sampling MAB:
    - Import numpy, json.
    - Class constant `N_ARMS = 5` (price discretized into 5 levels).
    - Constructor accepts `state: dict` (arm parameters per context) and `model_version`.
    - State format: `{"context_key": {"alpha": [1.0]*5, "beta": [1.0]*5}}`.
    - Method `get_optimal_price(service_id: int, price_min: float, price_max: float, hour_of_day: int, day_of_week: int, utilization: float, base_price: float | None = None) -> dict`:
      - Compute context_key from `_context_key(service_id, hour_of_day, day_of_week, utilization)`.
      - Initialize state for context_key if missing.
      - Sample from Beta(alpha[i], beta[i]) for each arm.
      - Select arm with highest sample.
      - Map arm to price using `np.linspace(price_min, price_max, N_ARMS)`.
      - **CRITICAL 30% daily constraint:** If base_price is provided, clamp optimal_price to `[base_price * 0.7, base_price * 1.3]`. If clamped, adjust confidence down.
      - Compute confidence from total observations on selected arm.
      - Return `{"optimal_price": float, "arm_selected": int, "confidence": float, "context_key": str, "constrained": bool}`.
    - Method `update_reward(context_key: str, arm: int, reward: bool)`: Increment alpha (success) or beta (failure).
    - Method `_context_key(service_id, hour, day, utilization) -> str`: Format as `"{service_id}:{day}:{hour//4}:{util_bucket}"` where util_bucket is "low"/<0.3, "mid"/<0.7, "high".
    - Methods `save_state(path)` and classmethod `load_state(path)` for JSON persistence.

    **services/ai/app/routers/optimization.py:** Create new optimization router with 4 endpoints:

    ```python
    router = APIRouter(prefix="/optimization", tags=["optimization"])
    ```

    `POST /optimization/upselling` (response_model=UpsellResponse):
    - Accept UpsellRequest body.
    - Get upselling model from model_loader.get_upselling_model().
    - If model is None, return fallback: empty recommendations, fallback=True.
    - Call model.recommend(request.current_service_id, request.customer_history or [], n_recommendations=3).
    - Return UpsellResponse.
    - On any error, return empty recommendations with fallback=True.

    `POST /optimization/pricing` (response_model=DynamicPricingResponse):
    - Accept DynamicPricingRequest body.
    - Get pricing model from model_loader.get_pricing_model().
    - If model is None, return fallback: optimal_price=(price_min+price_max)/2, fallback=True.
    - Call model.get_optimal_price(...) passing all fields including base_price.
    - Return DynamicPricingResponse with service_id from request.
    - On error, return midpoint price with fallback=True.

    `POST /optimization/capacity` (response_model=CapacityForecastResponse):
    - Accept CapacityForecastRequest body.
    - Get capacity model from model_loader.get_capacity_model().
    - If model is None, return empty forecast and suggestions with fallback=True.
    - Call model.forecast(request.days_ahead).
    - Call model.suggest_schedule_changes(forecast_data, request.current_capacity).
    - Return CapacityForecastResponse.
    - On error, return empty with fallback=True.

    `POST /optimization/reminder-timing` (response_model=ReminderTimingResponse):
    - Accept ReminderTimingRequest body.
    - Get reminder timing model from model_loader.get_reminder_timing_model().
    - If model is None, return fallback: minutes_before=1440, confidence=0.0, fallback=True.
    - Call model.get_optimal_timing(request.customer_id, request.notification_channel).
    - Return ReminderTimingResponse with customer_id from request.
    - On error, return 1440 min fallback.

    All endpoints: wrap in try/except, log errors, return typed fallback response.
