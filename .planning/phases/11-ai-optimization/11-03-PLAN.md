  <action>
    **services/ai/scripts/train_upselling.py:** Create similarity matrix training script:
    - Imports: numpy, pandas, json, datetime, os, sys, joblib, logging.
    - Add parent directory to sys.path for model class imports.
    - `generate_synthetic_bookings(n_customers: int = 200, n_services: int = 15) -> list[dict]`:
      - Create realistic booking data with customer-service frequency patterns.
      - Some customers prefer related services (e.g., customer who books haircut also books coloring).
      - Create "affinity groups" of 3-4 services that are frequently booked together.
      - Return list of `{"customer_id": int, "service_id": int, "count": int}`.
    - `load_training_data(api_url: str | None = None) -> list[dict]`:
      - If api_url provided, try GET `{api_url}/api/internal/features/training/upselling` using httpx.
      - If API unavailable or no URL, use `generate_synthetic_bookings()`.
    - `train_model(output_dir: str = "models")`:
      - Load data.
      - Import UpsellRecommender from app.models.upselling.
      - Call `UpsellRecommender.build_from_bookings(data)`.
      - Save similarity_matrix, service_ids, and popularity_fallback as a dict via joblib to `{output_dir}/upselling_v1.0.0.joblib`.
      - Print model dimensions and top service pairs by similarity.
      - Update metadata.json with training timestamp and metrics.
    - `if __name__ == '__main__': train_model()`

    **services/ai/scripts/train_pricing.py:** Create MAB state initialization script:
    - Imports: numpy, json, datetime, os, sys, logging.
    - `generate_synthetic_pricing_data(n_services: int = 15) -> dict`:
      - For each service, generate synthetic pricing observations across time contexts.
      - Create state dict with alpha/beta priors reflecting realistic patterns:
        - Popular hours (10am-2pm): higher base success rate.
        - Low-demand times (early morning, late evening): lower prices perform better.
        - Weekend vs weekday differences.
      - Instead of flat priors (alpha=1, beta=1), use informative priors (e.g., alpha=5, beta=3 for mid-price during peak hours).
    - `load_training_data(api_url: str | None = None) -> dict`:
      - If api_url, try to GET historical booking-at-price data.
      - Else, use `generate_synthetic_pricing_data()`.
    - `train_model(output_dir: str = "models")`:
      - Load or generate state.
      - Save to `{output_dir}/pricing_state.json`.
      - Print summary: number of contexts, total observations.
    - `if __name__ == '__main__': train_model()`
  </action>

      - Return DataFrame with 'ds' (datetime) and 'y' (booking count) columns.
    - `load_training_data(api_url: str | None = None) -> pd.DataFrame`:
      - If api_url, try to GET hourly booking aggregates.
      - Else, use `generate_synthetic_booking_counts()`.

      - Load data.

    - `generate_synthetic_notification_data(n_customers: int = 100) -> list[dict]`:

      - Each record: `{"customer_id": int, "channel": "email"|"sms", "minutes_before": int, "was_opened": bool}`.
      - Model realistic patterns: most customers open notifications sent 2-4 hours before. SMS has higher open rate than email. Some customers prefer morning notifications.
    - `load_training_data(api_url: str | None = None) -> list[dict]`:
      - If api_url, try GET notification history.
      - Else, use synthetic data.
    - `train_model(output_dir: str = "models")`:
      - Load data.
      - Import ReminderTimingOptimizer from app.models.reminder_timing.
      - Call `ReminderTimingOptimizer.optimize_from_data(data)`.
      - Save via `optimizer.save(f"{output_dir}/reminder_timing.json")`.
      - Print summary: number of customer clusters optimized, average optimal timing.
    - `if __name__ == '__main__': train_model()`
