<task type="auto">
  <name>Task 1: Install Vitest dependencies and create shared configuration</name>
  <files>
    package.json
    vitest.shared.ts
    vitest.workspace.ts
  </files>
  <action>
1. Install Vitest and coverage dependencies at root workspace level:
   `pnpm add -Dw vitest @vitest/coverage-v8 @vitest/ui happy-dom vite-tsconfig-paths`

2. Install React testing dependencies in apps/web:
   `pnpm add -D --filter @schedulebox/web @vitejs/plugin-react @testing-library/react @testing-library/dom @testing-library/user-event @testing-library/jest-dom`

3. Create `vitest.shared.ts` at project root with:
   - `defineConfig` from vitest/config
   - `globals: true` (so describe/it/expect available without import)
   - `environment: 'happy-dom'` as default (packages override to 'node' as needed)
   - Coverage configuration using `v8` provider with reporters: `['text', 'json', 'html', 'lcov']`
   - Coverage thresholds: `lines: 80, functions: 80, branches: 80, statements: 80`
   - Coverage exclude patterns: `node_modules/**, **/*.config.{js,ts,mjs}, **/*.d.ts, **/mocks/**, **/__tests__/setup.*, **/index.ts` (barrel files)
   - `restoreMocks: true` for automatic mock cleanup between tests
   - Do NOT include setupFiles here (packages define their own)

4. Create `vitest.workspace.ts` at project root with:
   - `defineWorkspace` importing from 'vitest/config'
   - Glob patterns: `['apps/*', 'packages/*', 'services/*']`
   - This tells Vitest to look for vitest.config.ts in each package

5. Add test scripts to root `package.json`:
   - `"test"`: `"vitest run"`
   - `"test:unit"`: `"vitest run"`
   - `"test:watch"`: `"vitest"`
   - `"test:ui"`: `"vitest --ui"`
   - `"test:coverage"`: `"vitest run --coverage"`

NOTE: Use `vitest/config` imports (not `vitest`). The workspace feature uses `defineWorkspace`. Research confirms Vitest 4.0 still supports `vitest.workspace.ts` file (the `projects` config is an alternative, not a replacement).
  </action>
  <verify>
Run `pnpm vitest --version` to confirm Vitest is installed. Run `pnpm test -- --passWithNoTests` or check that the command exits 0 with no test files found (not an error). Check that vitest.shared.ts and vitest.workspace.ts exist and have no TypeScript errors via `npx tsc --noEmit --skipLibCheck vitest.shared.ts` (or equivalent check).
  </verify>
  <done>
Vitest 4.0+ installed at root, vitest.shared.ts has coverage thresholds at 80%, vitest.workspace.ts references all workspace packages, root package.json has test/test:unit/test:watch/test:ui/test:coverage scripts.
  </done>
</task>

   - Merge with package-specific settings:
     - `test.include: ['src/**/*.test.ts']`
     - `test.environment: 'node'` (pure utility functions, no DOM needed)
     - Add `vite-tsconfig-paths` plugin if tsconfig has paths

2. Create `packages/events/vitest.config.ts`:
   - Same pattern as shared, merge with base config
   - `test.include: ['src/**/*.test.ts']`
   - `test.environment: 'node'`

3. Create `apps/web/vitest.config.ts`:
   - Merge shared config with app-specific:
   - `test.environment: 'happy-dom'` (React component tests need DOM)
   - `test.include: ['**/*.test.ts', '**/*.test.tsx']`
   - `test.exclude: ['node_modules', '.next', 'tests/e2e/**']` (exclude Playwright E2E)
   - Add `@vitejs/plugin-react` plugin
   - Add `vite-tsconfig-paths` plugin for `@/` path alias resolution
   - Add setupFiles pointing to `./vitest.setup.ts` (will be created in Plan 03 for MSW)

4. Create `services/notification-worker/vitest.config.ts`:
   - Same pattern, merge with base config
   - `test.environment: 'node'`
   - `test.include: ['src/**/*.test.ts']`

5. Create a minimal smoke test `packages/shared/src/utils/index.test.ts`:
   - Import `generateSlug` from './index'
   - Write one test: `test('generateSlug converts Czech diacritics', () => { expect(generateSlug('Salon Krasa')).toBe('salon-krasa'); })`
   - This verifies the entire test pipeline works end-to-end

6. Add `"test"` script to each package's package.json: `"test": "vitest run"`, `"test:watch": "vitest"`

   - services/notification-worker/package.json (add to existing scripts)

IMPORTANT: Do NOT create apps/web/vitest.setup.ts yet (Plan 03 creates it with MSW). The setupFiles reference in apps/web/vitest.config.ts should be commented out or conditional until the file exists.
