<task type="auto">
  <name>Task 2: Fix CI coverage command to use per-package execution</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Change the CI test job's coverage command from `pnpm test:coverage` to `pnpm -r --if-present test:coverage`.

    Root cause: Vitest 4.0 workspace mode (`pnpm test:coverage` at root) aggregates all package files into a combined report and does NOT propagate per-package threshold failures. The per-package vitest configs define thresholds via vitest.shared.ts but these only enforce when running each package individually.

    `pnpm -r --if-present test:coverage` runs each package's `test:coverage` script sequentially (respecting pnpm workspace topology), and each failing package exits with code 1, which pnpm -r propagates as the overall exit code. This matches how `pnpm --filter @schedulebox/events test -- --coverage` already exits code 1 when thresholds are violated.

    In .github/workflows/ci.yml, update the "Run unit tests with coverage" step:

    From:
    ```yaml
    - name: Run unit tests with coverage
      run: pnpm test:coverage
    ```

    To:
    ```yaml
    - name: Run unit tests with coverage
      run: pnpm -r --if-present test:coverage
    ```

    Leave the coverage artifact upload step unchanged (it already references packages/shared/coverage/, packages/events/coverage/, apps/web/coverage/ which are the per-package output directories — these will still be populated by per-package runs).

    Do not change any other CI jobs (lint, build, build-ai). Only the single `run:` line in the test job changes.
  </action>
  <verify>
    Verify the change is in the file:
    ```
    grep "pnpm -r --if-present test:coverage" .github/workflows/ci.yml
    ```
    Expected: line found.

    Verify old command is gone:
    ```
    grep "^        run: pnpm test:coverage" .github/workflows/ci.yml
    ```
    Expected: no match.

    Also run the full test suite locally to confirm all packages pass:
    ```

    ```
    Expected: All packages exit 0, overall EXIT: 0 with coverage at 100% for measured files in shared and events packages.

    Note: If any package lacks a `test:coverage` script, `--if-present` causes pnpm to skip it silently rather than failing. This is the safe fallback for packages like database that may not yet have tests.
  </verify>
