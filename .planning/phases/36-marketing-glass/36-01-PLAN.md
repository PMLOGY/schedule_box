/* Aurora animation for hero section */
@keyframes aurora {
  0%, 100% { background-position: 0% 50%; }
  50% { background-position: 100% 50%; }
}

.aurora-bg {
  background: linear-gradient(
    135deg,
    rgba(59, 130, 246, 0.15) 0%,
    rgba(99, 102, 241, 0.12) 25%,
    rgba(168, 85, 247, 0.08) 50%,
    rgba(59, 130, 246, 0.10) 75%,
    rgba(99, 102, 241, 0.15) 100%
  );
  background-size: 300% 300%;
  animation: aurora 18s ease infinite;
}

.dark .aurora-bg {
  background: linear-gradient(
    135deg,
    rgba(59, 130, 246, 0.08) 0%,
    rgba(99, 102, 241, 0.06) 25%,
    rgba(168, 85, 247, 0.04) 50%,
    rgba(59, 130, 246, 0.05) 75%,
    rgba(99, 102, 241, 0.08) 100%
  );
  background-size: 300% 300%;
  animation: aurora 18s ease infinite;
}

@media (prefers-reduced-motion: reduce) {
  .aurora-bg {

  }

Modify `hero-section.tsx` to add gradient text on the h1 and the aurora animation background element. Keep it as a server component (`async function`) — the aurora is pure CSS, no JS hooks needed.

1. **Hero section wrapper:** Change the outer `<section>` className from:
   `overflow-hidden py-16 md:py-24 lg:py-32`
   to:
   `relative overflow-hidden py-16 md:py-24 lg:py-32`
   The `relative` is needed to contain the `position: absolute` aurora div.

2. **Aurora background div:** Add immediately inside the `<section>`, before the `<div className="mx-auto...">`:
   ```tsx
   <div
     className="absolute inset-0 aurora-bg opacity-60 dark:opacity-30"
     aria-hidden="true"
   />
   ```
   This div is positioned absolute within the section, clipped by `overflow-hidden`, and uses the `.aurora-bg` class from globals.css for the animated gradient. It has `aria-hidden="true"` for accessibility. The opacity values (0.60 light / 0.30 dark) keep the aurora subtle enough to not degrade text legibility.

3. **Hero content z-index:** Add `relative` to the inner content wrapper `<div className="mx-auto max-w-6xl px-4">` so it becomes `<div className="relative mx-auto max-w-6xl px-4">`. This ensures the text content renders above the aurora background div (both are position:relative/static, but explicit relative ensures the content participates in stacking).

4. **Gradient text on h1:** Change the `<h1>` className from:
   `mt-6 text-3xl font-bold tracking-tight md:text-4xl lg:text-5xl xl:text-6xl`
   to:
   `mt-6 text-3xl font-bold tracking-tight md:text-4xl lg:text-5xl xl:text-6xl bg-gradient-to-r from-blue-600 to-indigo-600 bg-clip-text text-transparent`
   This creates the blue-to-indigo gradient text effect. The gradient text gets legibility from contrast against the near-white background (aurora opacity is too low to meaningfully shift background luminance).

5. **Do NOT:** Add `'use client'` directive. Do NOT wrap aurora in a motion component. Do NOT apply `glass-surface` to the section element. Do NOT use `position: fixed` on the aurora div.
