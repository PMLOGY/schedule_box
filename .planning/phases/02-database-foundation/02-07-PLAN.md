    ```sql
    -- Helper function to get company_id from session
    CREATE OR REPLACE FUNCTION current_company_id() RETURNS INTEGER AS $$
    BEGIN
      RETURN NULLIF(current_setting('app.company_id', true), '')::INTEGER;
    END;
    $$ LANGUAGE plpgsql STABLE SECURITY DEFINER;

    -- Helper function to get user role from session
    CREATE OR REPLACE FUNCTION current_user_role() RETURNS TEXT AS $$
    BEGIN
      RETURN current_setting('app.user_role', true);
    END;
    $$ LANGUAGE plpgsql STABLE SECURITY DEFINER;

    -- Helper function to get user_id from session
    CREATE OR REPLACE FUNCTION current_user_id() RETURNS INTEGER AS $$
    BEGIN
      RETURN NULLIF(current_setting('app.user_id', true), '')::INTEGER;
    END;
    $$ LANGUAGE plpgsql STABLE SECURITY DEFINER;
    ```

    - coupons, coupon_usage, gift_cards, gift_card_transactions
    - loyalty_programs, loyalty_tiers, loyalty_cards (note: loyalty_transactions and rewards reference program, not company directly — skip RLS or add via join)
    - notification_templates, notifications
    - reviews

    - marketplace_listings
    - video_meetings
    - whitelabel_apps

    - analytics_events, audit_logs, competitor_data

    For EACH table with company_id, add:
    ```sql
    ALTER TABLE {table_name} ENABLE ROW LEVEL SECURITY;
    CREATE POLICY tenant_isolation_{table_name} ON {table_name}
      USING (company_id = current_company_id());
    ```

    For tables WITHOUT company_id that are tenant-scoped via FK (loyalty_tiers via program_id, loyalty_transactions via card_id, loyalty_cards via program_id, rewards via program_id, automation_logs via rule_id, coupon_usage indirectly):

    - automation_logs: Skip RLS (accessed through rule which has RLS)

    - coupon_usage: No company_id. Skip — accessed through coupon/booking.
    - password_history, refresh_tokens: User-scoped, not tenant-scoped. Skip RLS.
    - customer_tags: No company_id. Skip — accessed through customer.
    - employee_services: No company_id. Skip — accessed through employee.
    - service_resources: No company_id. Skip — accessed through service.
    - role_permissions: Global. Skip.

    Add admin bypass policy for each RLS-enabled table:
    ```sql
    CREATE POLICY admin_bypass_{table_name} ON {table_name}
      USING (current_user_role() = 'admin');
    ```

    Add customer-specific booking policy:
    ```sql
    CREATE POLICY customer_own_bookings ON bookings
      FOR SELECT
      USING (
        current_user_role() = 'customer'
        AND customer_id IN (
          SELECT id FROM customers WHERE user_id = current_user_id()
        )
      );
    ```
