  <action>
    Create drizzle.config.ts using defineConfig from drizzle-kit:
    - schema: './src/schema/index.ts'
    - out: './src/migrations'
    - dialect: 'postgresql'
    - dbCredentials.url from process.env.DATABASE_URL
    - verbose: true, strict: true

    Create src/db.ts:
    - Import drizzle from 'drizzle-orm/postgres-js'
    - Import postgres from 'postgres'
    - Create migrationClient with max: 1 for migration transactions
    - Create queryClient with max: 10, idle_timeout: 20, connect_timeout: 10
    - Export `db` instance with drizzle(queryClient) — schema will be added as schemas are created
    - Export `migrationClient` for use in migrate.ts
    - Export type `Database = typeof db`

    Create src/migrate.ts:
    - Import drizzle from 'drizzle-orm/postgres-js'
    - Import migrate from 'drizzle-orm/postgres-js/migrator'
    - Import postgres from 'postgres'
    - Load DATABASE_URL from environment (use dotenv if --env-file not available)
    - Create single-connection client
    - Run migrate with migrationsFolder: './src/migrations'
    - Close connection after completion
    - Log progress: "Running migrations..." and "Migrations complete"

    Create src/schema/index.ts as empty barrel file:
    ```typescript
    // Schema barrel file - exports added as schema files are created
    ```

    Update src/index.ts to re-export from db.ts and schema:
    ```typescript
    export { db, migrationClient } from './db.js';
    export type { Database } from './db.js';
    export * from './schema/index.js';
    ```

    IMPORTANT: Use .js extensions in imports (ESM requires this with TypeScript).
  </action>
