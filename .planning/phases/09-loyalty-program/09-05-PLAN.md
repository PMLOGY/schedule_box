**Add Points Route** (`apps/web/app/api/v1/loyalty/cards/[id]/add-points/route.ts`):

- **POST**: Manually add points to a loyalty card.
  - Validate `[id]` param as UUID
  - Validate body with `addPointsSchema` (points: number, description?: string)
  - Look up card by UUID, validate ownership via program -> companyId
  - Call `earnPoints(card.id, body.points, body.description || 'Manual adjustment')`
  - Fetch updated card (with tier progress) and return 200 with updated card object
  - Handle errors: 404 if card not found, 400 if validation fails, 500 for server errors

**Redeem Reward Route** (`apps/web/app/api/v1/loyalty/rewards/[id]/redeem/route.ts`):

- **POST**: Redeem a reward using points from a loyalty card.
  - Validate `[id]` param as reward ID (integer or UUID)
  - Validate body with `redeemRewardSchema` (card_id: UUID)
  - Look up reward by ID, validate ownership via program -> companyId
  - Look up card by UUID from body.card_id, validate same program
  - Call `redeemReward(card.id, reward.id)` from rewards-engine
  - Return 200 with { message: 'Reward redeemed successfully', reward: { name, pointsCost }, newBalance }
  - Handle errors: 404 if reward/card not found, 400 if insufficient points or reward unavailable, 409 if reward maxed out

Both routes follow existing patterns from Phase 5 booking routes.

  <action>
Create `apps/web/lib/loyalty/booking-completed-consumer.ts` following the consumer pattern from `packages/events/src/consumer.ts`:

1. **handleBookingCompleted(event: CloudEvent<BookingCompletedPayload>): Promise<void>**
   - Extract bookingUuid and companyId from event.data
   - Call `awardPointsForBooking(bookingUuid, companyId)` from points-engine
   - Log success: `[Loyalty] Awarded points for booking ${bookingUuid}`
   - On error: log error and re-throw (consumer will NACK and requeue)

2. **startBookingCompletedConsumer(): Promise<void>**
   - Call `createConsumerConnection()` from `@schedulebox/events`
   - Call `consumeMessages(channel, { queueName: 'loyalty.booking-completed', routingKeys: ['booking.completed'], prefetch: 10, handler: handleBookingCompleted })`
   - Log: `[Loyalty Worker] Listening for booking.completed events...`
   - Set up SIGTERM handler for graceful shutdown: close channel, close connection

Import types:
- `CloudEvent, BookingCompletedPayload` from `@schedulebox/events`
- `createConsumerConnection, consumeMessages, gracefulShutdown` from `@schedulebox/events`
- `awardPointsForBooking` from `./points-engine`

The consumer is designed to be started as a standalone worker or imported and started within the Next.js server during development. The `handleBookingCompleted` function is exported separately for testing.

Add error handling for the idempotency case: if `awardPointsForBooking` returns without error (because points were already awarded), the message should still be ACKed (not requeued).
