  <action>
Follow the existing route pattern from `apps/web/app/api/v1/bookings/route.ts` for structure.

**Programs Route** (`apps/web/app/api/v1/loyalty/programs/route.ts`):

- **GET**: Fetch the company's loyalty program (SELECT from loyaltyPrograms WHERE companyId, include tiers via LEFT JOIN). Since UNIQUE(company_id), there's at most one program per company. Return the program with its tiers array. Return 404 if no program exists.

- **POST**: Create a loyalty program. Validate body with `loyaltyProgramCreateSchema`. INSERT into loyaltyPrograms. If creating with type 'tiers', optionally accept initial tiers. Return 201 with created program.

- **PUT**: Update the loyalty program. Validate body with `loyaltyProgramUpdateSchema`. UPDATE loyaltyPrograms WHERE companyId. Return 200 with updated program.

All endpoints must scope by companyId (tenant isolation). Use NextResponse for responses. Use standard error format: `{ error: string, code: string, message: string }`.

**Tiers Route** (`apps/web/app/api/v1/loyalty/tiers/route.ts`):

- **GET**: List tiers for the company's program. SELECT from loyaltyTiers JOIN loyaltyPrograms WHERE companyId. ORDER BY sortOrder ASC. Return array of tiers.

- **POST**: Create a tier. Validate body with `tierCreateSchema`. Look up program by companyId, INSERT into loyaltyTiers with programId. Return 201.

**Rewards Route** (`apps/web/app/api/v1/loyalty/rewards/route.ts`):

- **GET**: List rewards for the company's program. Accept query param `is_active` for filtering. SELECT from rewards JOIN loyaltyPrograms WHERE companyId. Return paginated array.

- **POST**: Create a reward. Validate body with `rewardCreateSchema`. Look up program by companyId, INSERT into rewards with programId. Return 201.

**Rewards Detail Route** (`apps/web/app/api/v1/loyalty/rewards/[id]/route.ts`):

- **GET**: Get reward detail by ID. Validate ownership via program -> companyId.
- **PUT**: Update reward. Validate body with `rewardUpdateSchema`.
- **DELETE**: Soft delete — set is_active = false (don't actually DELETE the row).

All routes must:
- Parse and validate request bodies with Zod schemas from `@schedulebox/shared`
- Scope all queries by companyId for tenant isolation
- Return UUID-based identifiers in responses (never expose SERIAL IDs in API)
- Use try/catch with appropriate HTTP status codes (400, 404, 500)

**Cards List + Create Route** (`apps/web/app/api/v1/loyalty/cards/route.ts`):

- **GET**: List loyalty cards. Accept query params from `loyaltyCardListQuerySchema` (page, limit, customer_id). SELECT from loyaltyCards JOIN loyaltyPrograms (for companyId scope) LEFT JOIN loyaltyTiers LEFT JOIN customers. Return paginated array with card details including customer name, current tier, points balance.

- **POST**: Issue a loyalty card. Validate body with `loyaltyCardCreateSchema`. Look up customer by UUID, look up program by companyId. Generate unique card_number (use `crypto.randomUUID()` or format like `SB-XXXX-XXXX-XXXX`). INSERT into loyaltyCards. Assign default tier (tier with minPoints = 0, or null). Publish LoyaltyCardCreatedEvent. Return 201.

**Card Detail Route** (`apps/web/app/api/v1/loyalty/cards/[id]/route.ts`):

- **GET**: Get card detail by UUID. JOIN loyaltyCards with loyaltyTiers, loyaltyPrograms, customers. Also call `calculateTierProgress()` to include nextTier and progressPercent in response. Validate ownership via program -> companyId. Return card with: pointsBalance, stampsBalance, currentTier (with name, color, minPoints), nextTier (with name, minPoints, pointsNeeded), customer info, applePassUrl, googlePassUrl.

**Transactions Route** (`apps/web/app/api/v1/loyalty/cards/[id]/transactions/route.ts`):

- **GET**: List transactions for a card. Accept query params from `transactionListQuerySchema` (page, limit, type filter). SELECT from loyaltyTransactions WHERE cardId (look up card by UUID first, validate ownership). ORDER BY createdAt DESC. Return paginated array with transaction details.

All routes must:
- Use `[id]` param as UUID (validate with z.string().uuid())
- JOIN through program to validate companyId ownership
- Generate card_number as formatted string (e.g., `SB-${randomHex(4)}-${randomHex(4)}-${randomHex(4)}`.toUpperCase())
- Return standard paginated response: `{ data: [...], pagination: { page, limit, total, totalPages } }`
