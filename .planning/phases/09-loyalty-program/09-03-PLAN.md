Create `apps/web/lib/loyalty/points-engine.ts` with the following functions:

1. **earnPoints(cardId: number, points: number, description: string, bookingId?: number): Promise<void>**
   - Open Drizzle transaction
   - SELECT FOR UPDATE on loyaltyCards row by cardId (`.for('update')`)
   - Validate card exists and is active
   - Calculate new balance: `card.pointsBalance + points`
   - UPDATE loyaltyCards set pointsBalance = newBalance, updatedAt = new Date()
   - INSERT loyaltyTransactions: { cardId, bookingId, type: 'earn', points: +points, balanceAfter: newBalance, description }
   - After transaction commits, call `checkAndUpgradeTier(cardId)` (imported from tier-engine)
   - Publish PointsEarnedEvent via publishEvent

2. **redeemPoints(cardId: number, pointsToRedeem: number, description: string): Promise<void>**
   - Open Drizzle transaction
   - SELECT FOR UPDATE on loyaltyCards row
   - Validate card exists and is active
   - Check `card.pointsBalance >= pointsToRedeem` (throw ValidationError if insufficient)
   - Calculate new balance: `card.pointsBalance - pointsToRedeem`
   - UPDATE loyaltyCards set pointsBalance = newBalance, updatedAt = new Date()
   - INSERT loyaltyTransactions: { cardId, type: 'redeem', points: -pointsToRedeem, balanceAfter: newBalance, description }

3. **adjustPoints(cardId: number, points: number, description: string): Promise<void>**
   - Open Drizzle transaction
   - SELECT FOR UPDATE on loyaltyCards row
   - Calculate new balance (points can be negative for deductions)
   - Validate new balance >= 0
   - UPDATE + INSERT transaction record with type: 'adjust'

4. **awardPointsForBooking(bookingUuid: string, companyId: number): Promise<void>**
   - This is the idempotent handler for booking.completed events
   - First check if loyaltyTransactions already has a record with this bookingId (via JOIN to bookings table by uuid) — if so, return early (idempotent)
   - Look up the booking by UUID to get price and customer_id
   - Look up loyalty program for company (loyaltyPrograms WHERE company_id = companyId AND is_active = true)
   - Look up loyalty card for customer + program (or create one if auto-enroll)
   - Calculate points: `Math.floor(bookingPrice * program.pointsPerCurrency)`
   - Call earnPoints(card.id, calculatedPoints, `Booking ${bookingUuid}`, booking.id)

Import db from `@schedulebox/database`, loyalty schema tables, publishEvent from `@schedulebox/events`, createPointsEarnedEvent. Use the same error handling pattern as `apps/web/lib/booking/booking-transitions.ts` for reference.

**Tier Engine** — Create `apps/web/lib/loyalty/tier-engine.ts`:

1. **checkAndUpgradeTier(cardId: number): Promise<boolean>**
   - SELECT card with current programId and pointsBalance
   - SELECT highest tier from loyaltyTiers WHERE programId = card.programId AND minPoints <= card.pointsBalance ORDER BY minPoints DESC LIMIT 1
   - If qualifiedTier.id !== card.currentTierId:
     - UPDATE loyaltyCards set tierId = qualifiedTier.id, updatedAt = new Date()
     - Get previous tier name for event
     - Publish TierUpgradedEvent via publishEvent + createTierUpgradedEvent
     - Return true
   - Return false

2. **calculateTierProgress(cardId: number): Promise<{ currentTier: LoyaltyTier | null; nextTier: { name: string; minPoints: number; pointsNeeded: number } | null; progressPercent: number }>**
   - SELECT card with pointsBalance and current tierId
   - SELECT current tier details
   - SELECT next tier (loyaltyTiers WHERE programId = card.programId AND minPoints > card.pointsBalance ORDER BY minPoints ASC LIMIT 1)
   - Calculate progress: `(card.pointsBalance - currentTier.minPoints) / (nextTier.minPoints - currentTier.minPoints) * 100`
   - Return { currentTier, nextTier: { name, minPoints, pointsNeeded: nextTier.minPoints - card.pointsBalance }, progressPercent }

**Rewards Engine** — Create `apps/web/lib/loyalty/rewards-engine.ts`:

1. **redeemReward(cardId: number, rewardId: number): Promise<void>**
   - Open Drizzle transaction
   - SELECT reward by id (validate exists, is_active)
   - Check max_redemptions: if maxRedemptions !== null AND currentRedemptions >= maxRedemptions, throw ValidationError('Reward no longer available')
   - SELECT FOR UPDATE on loyaltyCards row
   - Check card.pointsBalance >= reward.pointsCost (throw ValidationError if insufficient)
   - Calculate new balance
   - UPDATE loyaltyCards set pointsBalance = newBalance
   - INSERT loyaltyTransactions: { cardId, type: 'redeem', points: -reward.pointsCost, balanceAfter: newBalance, description: `Redeemed: ${reward.name}` }
   - UPDATE rewards set currentRedemptions = currentRedemptions + 1
   - After transaction: publish RewardRedeemedEvent

2. **checkRewardAvailability(rewardId: number, cardPointsBalance: number): { available: boolean; reason?: string }**
   - Check is_active
   - Check max_redemptions vs current_redemptions
   - Check cardPointsBalance >= pointsCost
   - Return availability status with reason if unavailable
