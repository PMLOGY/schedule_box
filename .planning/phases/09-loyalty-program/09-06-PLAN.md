**Apple Wallet Service** (`apps/web/lib/loyalty/wallet/apple-wallet.ts`):

Create `generateApplePass(cardId: number): Promise<Buffer>` using passkit-generator:

1. Fetch card data: SELECT loyaltyCards JOIN loyaltyPrograms JOIN loyaltyTiers JOIN customers WHERE loyaltyCards.id = cardId
2. Check environment variables exist (APPLE_WWDR_CERT_PATH, APPLE_PASS_CERT_PATH, APPLE_PASS_KEY_PATH). If missing, throw a descriptive error: `'Apple Wallet certificates not configured. Set APPLE_WWDR_CERT_PATH, APPLE_PASS_CERT_PATH, APPLE_PASS_KEY_PATH environment variables.'`
3. Create pass using PKPass.from():
   - model: Use a minimal pass.json template created inline (no template directory needed for MVP):
     ```
     {
       "formatVersion": 1,
       "passTypeIdentifier": "pass.com.schedulebox.loyalty",
       "teamIdentifier": process.env.APPLE_TEAM_ID,
       "organizationName": "ScheduleBox"
     }
     ```
   - certificates: { wwdr, signerCert, signerKey, signerKeyPassphrase } from env vars
4. Set pass properties:
   - serialNumber: card.cardNumber
   - description: `${program.name} - Loyalty Card`
   - foregroundColor: 'rgb(255, 255, 255)'
   - backgroundColor: tier.color converted from hex to rgb, or default 'rgb(59, 130, 246)'
   - storeCard with primaryFields: [{ key: 'points', label: 'Body', value: card.pointsBalance.toString() }]
   - auxiliaryFields: [{ key: 'tier', label: 'Úroveň', value: tier.name || 'Základní' }]
   - barcode: { format: 'PKBarcodeFormatQR', message: card.cardNumber, messageEncoding: 'iso-8859-1' }
5. Return `pass.getAsBuffer()`

**Apple Pass API Route** (`apps/web/app/api/v1/loyalty/cards/[id]/apple-pass/route.ts`):

- **GET**: Generate and download Apple Wallet pass.
  - Validate [id] param as UUID
  - Look up card by UUID, validate ownership via program -> companyId
  - Call `generateApplePass(card.id)`
  - Return Response with:
    - Body: pass buffer
    - Content-Type: 'application/vnd.apple.pkpass'
    - Content-Disposition: `attachment; filename="${card.cardNumber}.pkpass"`
  - On error: Return 500 with descriptive error (or 503 if certificates not configured)

Note: For MVP, passes are generated on-demand (not cached). Pass updates via APNs are deferred to a future phase.

**Google Wallet Service** (`apps/web/lib/loyalty/wallet/google-wallet.ts`):

Create `generateGooglePassUrl(cardId: number): Promise<string>` using google-wallet npm or direct Google Wallet REST API:

1. Fetch card data: SELECT loyaltyCards JOIN loyaltyPrograms JOIN loyaltyTiers JOIN customers WHERE loyaltyCards.id = cardId
2. Check environment variables (GOOGLE_WALLET_SERVICE_ACCOUNT_KEY_PATH, GOOGLE_WALLET_ISSUER_ID). If missing, throw descriptive error.
3. Authenticate with Google Cloud using service account credentials
4. Create or update LoyaltyClass (program template):
   - id: `${issuerId}.schedulebox_loyalty_${program.id}`
   - issuerName: program.name
   - programName: program.name
   - programLogo: placeholder (can be updated later with company logo)
5. Create or update LoyaltyObject (individual card):
   - id: `${issuerId}.card_${card.cardNumber}`
   - classId: (from step 4)
   - state: 'ACTIVE'
   - accountId: card.cardNumber
   - accountName: tier.name || 'Základní'
   - loyaltyPoints: { label: 'Body', balance: { int: card.pointsBalance } }
   - barcode: { type: 'QR_CODE', value: card.cardNumber }
6. Generate JWT-based save URL using google-auth-library to create a signed JWT
7. Return save URL: `https://pay.google.com/gp/v/save/{jwt}`

Handle the case where class/object already exists (catch 409 conflict, do update instead).

**Google Pass API Route** (`apps/web/app/api/v1/loyalty/cards/[id]/google-pass/route.ts`):

- **GET**: Generate Google Wallet save URL.
  - Validate [id] param as UUID
  - Look up card by UUID, validate ownership via program -> companyId
  - Call `generateGooglePassUrl(card.id)`
  - Store the Google pass URL in loyaltyCards.googlePassUrl for future updates
  - Return 200 with { saveUrl: string }
  - On error: Return 500 with descriptive error (or 503 if credentials not configured)

Note: For MVP, pass objects are created/updated on each URL generation. Background updates (updating points balance in existing passes) are deferred to a future phase.
