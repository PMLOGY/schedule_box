<task type="auto">
  <name>Task 1: Remove continue-on-error from E2E CI job and fix any test issues</name>
  <files>.github/workflows/ci.yml, apps/web/e2e/playwright.config.ts</files>
  <action>
1. In `.github/workflows/ci.yml`, find the `e2e` job (around line 260) and:
   - Remove `continue-on-error: true` — E2E failures must block the pipeline
   - Keep the `if:` condition for when E2E runs (main branch + PRs)
   - Ensure the `Upload Playwright report` and `Upload test results` steps still have `if: !cancelled()` and `continue-on-error: true` (artifacts should still upload on failure for debugging)

2. Review `apps/web/e2e/playwright.config.ts`:
   - `retries: process.env.CI ? 2 : 0` is fine (retries help with transient failures)
   - `workers: process.env.CI ? 1 : undefined` is fine (single worker prevents resource contention)
   - `forbidOnly: !!process.env.CI` is correct (prevents `.only` from sneaking in)
   - Verify `webServer` config works for CI: it uses `pnpm start` which requires a prior build step (CI does build first, then runs E2E — correct)

3. Read each E2E spec file and check for:
   - `test.skip()` calls — if any exist, either remove the skip and fix the test, or document why it's skipped
   - `.only` markers — must not exist (forbidOnly catches this in CI)
   - Hardcoded waits (`page.waitForTimeout`) — replace with proper locator waits where possible
   - Tests that depend on specific seed data — ensure they use the development seed (which CI runs via `db:setup`)

4. Run the E2E suite locally to identify any failures:
   `cd apps/web && pnpm exec playwright test --config e2e/playwright.config.ts --project=chromium`
   - If tests fail, fix the root cause (not by adding skips)
   - Common issues: stale selectors, timing issues, missing seed data, API route changes

5. If all 7 specs pass locally on chromium, that's sufficient validation. The CI will run all browsers.
