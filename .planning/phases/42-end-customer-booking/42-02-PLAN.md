  1. Add import at top of file:
     ```typescript
     import { awardPointsForBooking } from '@/lib/loyalty/points-engine';
     ```

  2. After the event publishing try/catch block in `completeBooking` (after line 281), add a synchronous loyalty points award call:
     ```typescript
     // Award loyalty points synchronously (fallback for when RabbitMQ is not running)
     // awardPointsForBooking is idempotent — safe to call even if consumer also processes the event
     try {
       await awardPointsForBooking(existing.id, companyId);
     } catch (error) {
       // Non-critical: log but don't fail the booking completion
       console.error('[Booking Transitions] Failed to award loyalty points:', error);
     }
     ```

  Note: `existing.id` here is the booking UUID (BookingWithRelations uses uuid as id field — verify by checking the getBooking return shape). The `awardPointsForBooking` function is idempotent (checks for existing transactions), so even if RabbitMQ IS running and the consumer also processes the event, points won't be double-awarded.

  **Part A: Create public loyalty lookup endpoint**

  Create `apps/web/app/api/v1/public/loyalty/route.ts` — a public (no auth) GET endpoint that allows checking loyalty status by email + company_slug. This lets the booking wizard show loyalty info to returning customers.

  ```
  GET /api/v1/public/loyalty?email=x@y.com&company_slug=salon-krasy
  ```

  Response:
  ```json
  {
    "data": {
      "has_card": true,
      "points_balance": 150,
      "available_rewards": [
        { "id": 1, "name": "10% sleva", "points_cost": 100, "reward_type": "discount_percentage", "reward_value": 10 }
      ]
    }
  }
  ```

  Implementation:
  1. Validate query params with Zod: `email` (required, email format), `company_slug` (required, string)
  2. Resolve company by slug
  3. Find customer by email + companyId
  4. If no customer, return `{ has_card: false, points_balance: 0, available_rewards: [] }`
  5. Find loyalty card for customer + company's program
  6. If no card, return `{ has_card: false, points_balance: 0, available_rewards: [] }`
  7. Fetch active rewards from program where `points_cost <= card.pointsBalance` and `(maxRedemptions IS NULL OR currentRedemptions < maxRedemptions)`
  8. Return card balance and available rewards

  Use `createRouteHandler` with `requiresAuth: false`. Import from `@schedulebox/database`: `db, companies, customers, loyaltyCards, loyaltyPrograms, rewards`.

  **Part B: Add optional coupon_code to booking creation**

  In `apps/web/app/api/v1/public/company/[slug]/bookings/route.ts`:

  1. Add `reward_id` (optional number) to `publicBookingCreateSchema`
  2. If `reward_id` is provided along with `customer_email`:
     a. Look up the customer by email + companyId
     b. Look up their loyalty card
     c. Look up the reward by ID, verify it belongs to the company's program
     d. Verify customer has enough points (`card.pointsBalance >= reward.pointsCost`)
     e. Calculate discount:
        - `discount_percentage`: `discountAmount = service.price * reward.rewardValue / 100`
        - `discount_fixed`: `discountAmount = min(reward.rewardValue, service.price)`
     f. Deduct points from card using `redeemPoints` from `@/lib/loyalty/points-engine`
     g. Increment reward's `currentRedemptions`
     h. Set `discountAmount` on the booking insert (instead of hardcoded '0')
  3. If reward_id is not provided, keep existing behavior (discountAmount: '0')

  Import `redeemPoints` from `@/lib/loyalty/points-engine` and `rewards` from `@schedulebox/database`.
