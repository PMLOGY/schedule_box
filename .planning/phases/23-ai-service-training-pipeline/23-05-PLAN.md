<task type="auto">
  <name>Task 1: Create weekly model retraining GitHub Actions workflow</name>
  <files>.github/workflows/train-models.yml</files>
  <action>
    Create `.github/workflows/train-models.yml`:

    ```yaml
    name: Weekly Model Retraining

    on:
      schedule:
        - cron: '0 3 * * 0'  # Sunday 03:00 UTC
      workflow_dispatch:       # Manual trigger option (AI-06 requirement)

    jobs:
      train-and-build:
        runs-on: ubuntu-latest
        defaults:
          run:
            working-directory: services/ai

        steps:
          - uses: actions/checkout@v4

          - uses: actions/setup-python@v5
            with:
              python-version: '3.12'
              cache: 'pip'
              cache-dependency-path: services/ai/requirements.txt

          - name: Install AI service dependencies
            run: pip install -r requirements.txt

          - name: Train no-show model
            run: python -m scripts.train_no_show
            env:
              SCHEDULEBOX_API_URL: ${{ secrets.SCHEDULEBOX_INTERNAL_URL }}
              AI_SERVICE_API_KEY: ${{ secrets.AI_SERVICE_API_KEY }}

          - name: Train CLV model
            run: python -m scripts.train_clv
            env:
              SCHEDULEBOX_API_URL: ${{ secrets.SCHEDULEBOX_INTERNAL_URL }}
              AI_SERVICE_API_KEY: ${{ secrets.AI_SERVICE_API_KEY }}

          - name: Train capacity model
            run: python -m scripts.train_capacity
            env:
              SCHEDULEBOX_API_URL: ${{ secrets.SCHEDULEBOX_INTERNAL_URL }}
              AI_SERVICE_API_KEY: ${{ secrets.AI_SERVICE_API_KEY }}

          - name: Verify model files produced
            run: |
              echo "Checking model files..."
              ls -la models/
              test -f models/no_show_v1.0.0.joblib || exit 1
              test -f models/no_show_v1.0.0.meta.json || exit 1
              test -f models/clv_v1.0.0.joblib || exit 1
              test -f models/clv_v1.0.0.meta.json || exit 1
              test -f models/capacity_v1.0.0.json || exit 1
              test -f models/capacity_v1.0.0.meta.json || exit 1
              echo "All model files verified"

          - name: Upload model artifacts
            uses: actions/upload-artifact@v4
            with:
              name: trained-models
              path: |
                services/ai/models/*.joblib
                services/ai/models/*.json
                services/ai/models/*.meta.json
              retention-days: 30
    ```

    Key design decisions:
    - Training runs BEFORE Docker build. The trained model files are in the `models/` directory, which `COPY ./models ./models` in the Dockerfile picks up.
    - Uses `actions/upload-artifact@v4` to save model files as CI artifacts for debugging and rollback.
    - Passes `SCHEDULEBOX_API_URL` and `AI_SERVICE_API_KEY` from GitHub secrets so training scripts can fetch real data from the live Next.js API. If secrets are not set, scripts fall back to synthetic data (graceful degradation).
    - The `verify model files` step ensures all expected files exist before proceeding.
    - The workflow does NOT include a Docker build step — that is handled by the existing `deploy-production.yml` workflow. The model files are committed to the repository (or handled via artifact + deploy trigger depending on deployment strategy).
    - `pip` caching via `actions/setup-python` cache parameter speeds up subsequent runs.
