<objective>
Enable business owners to create employee user accounts so employees can log in to the system with their own credentials.

Purpose: Currently the employees API only creates records in the `employees` table (scheduling/assignment data) but does NOT create login credentials in the `users` table. AUTH-04 requires employees to be able to log in. The `employees` table already has a `userId` column (FK to `users.id`) that is currently always null.

Output: A POST /api/v1/employees/invite endpoint that creates a user account (in `users` table with employee role) and links it to an existing employee record, plus a simple UI form on the employees page for owners to create employee accounts.
</objective>

**validations/employee.ts changes:**

1. Add a new `employeeInviteSchema` Zod schema:
```typescript
export const employeeInviteSchema = z.object({
  employee_uuid: z.string().uuid('Invalid employee UUID'),
  email: z.string().email('Invalid email'),
  password: z.string().min(8, 'Password must be at least 8 characters'),
  name: z.string().min(1, 'Name is required').optional(), // Falls back to employee.name
});
```

**New file: app/api/v1/employees/invite/route.ts:**

2. Create a POST handler using `createRouteHandler` with `requiresAuth: true`, `requiredPermissions: [PERMISSIONS.EMPLOYEES_MANAGE]`, and `bodySchema: employeeInviteSchema`.

3. Implementation logic:
   a. Get company scope via `findCompanyId(user.sub)`.
   b. Look up the employee by UUID, verify it belongs to the same company, verify `userId IS NULL` (not already linked to a user account). If already linked, return 409 Conflict with message "Employee already has a user account".
   c. Check that the email is not already registered in `users` table. If it is, return 409 Conflict.
   d. In a transaction:
      - Hash the password using `hashPassword`.
      - Find the `employee` role ID from `roles` table.
      - Insert a new `users` record with: `companyId`, `roleId` (employee), `email`, `passwordHash`, `name` (from body or fall back to `employee.name`), `isActive: true`.
      - Insert into `passwordHistory`.
      - Update the `employees` record to set `userId` to the new user's `id`, and also set `email` on the employee record to match.
   e. Return 201 with `{ uuid: newUser.uuid, email, name, role: 'employee' }`.

4. Use these imports (following existing patterns from register/route.ts and employees/route.ts):
   - `db`, `users`, `roles`, `employees`, `passwordHistory` from `@schedulebox/database`
   - `hashPassword` from `@/lib/auth/password`
   - `createRouteHandler` from `@/lib/middleware/route-handler`
   - `findCompanyId` from `@/lib/db/tenant-scope`
   - `PERMISSIONS` from `@/lib/middleware/rbac`
   - `createdResponse` from `@/lib/utils/response`
   - `ConflictError`, `NotFoundError` from `@schedulebox/shared`
