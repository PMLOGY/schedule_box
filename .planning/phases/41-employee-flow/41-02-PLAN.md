  <action>
    Audit the working hours flow for correctness. Known issues to check and fix:

    1. **PUT body wrapping**: The `useUpdateMyWorkingHours` hook sends `{ hours: data }`. The API expects `{ hours: [...] }` via `updateWorkingHoursSchema`. Verify this matches. If the hook sends raw array without wrapping, fix the hook to wrap in `{ hours: data }`. Currently it sends `apiClient.put('/employees/me/working-hours', { hours: data })` — this looks correct.

    2. **Save logic bug**: In `handleSave`, the code does `const activeHours = editedHours.filter(h => h.is_active); await updateMutation.mutateAsync(activeHours.length > 0 ? activeHours : editedHours)`. This means if ALL days are toggled off (no active hours), it sends ALL 7 days with `is_active: false`. But the API schema requires `min(1)` hours — so saving with zero active days would send all 7 (with is_active=false), which is fine. However, the intent is unclear — if all inactive hours are sent, the user will see them on reload as inactive days. Verify: after saving with some days active and reloading, the correct days show as active. Fix if needed.

    3. **Day labels**: Day names are hardcoded in Czech (`Nedele`, `Pondeli`, etc.) but without diacritics. These should use translations. Add translation keys to all 3 locale files:
       - `schedule.days.0` through `schedule.days.6` for full day names
       - en.json: Sunday, Monday, Tuesday, Wednesday, Thursday, Friday, Saturday
       - cs.json: Neděle, Pondělí, Úterý, Středa, Čtvrtek, Pátek, Sobota
       - sk.json: Nedeľa, Pondelok, Utorok, Streda, Štvrtok, Piatok, Sobota
       - Replace `DAY_NAMES_FULL[hour.day_of_week]` with `t(\`days.${hour.day_of_week}\`)` in the component

    4. **Short day labels**: Similarly replace `DAY_NAMES` array with translated short names (Ne/Po/Ut... → translation keys `schedule.daysShort.0` etc.) — though these may not be used in the current UI. Only add if referenced.

    5. **Verify API returns correct shape**: GET should return `[{ day_of_week, start_time, end_time, is_active }]`. The API currently selects `dayOfWeek` as `day_of_week` etc. — verify the column alias mapping is correct in Drizzle select.
