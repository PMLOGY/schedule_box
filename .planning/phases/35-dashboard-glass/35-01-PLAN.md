  <action>
In the `<header>` element, modify its className from:
```
"sticky top-0 z-10 flex h-16 items-center justify-between border-b bg-background px-6"
```
to:
```
"sticky top-0 z-10 flex h-16 items-center justify-between border-b border-glass px-6 glass-surface-subtle"
```

Specific changes:
1. REMOVE `bg-background` (replaced by glass-surface-subtle which provides its own background)
2. ADD `glass-surface-subtle` (provides 8px backdrop-blur, semi-transparent background, position:relative via @supports)
3. ADD `border-glass` AFTER `border-b` (overrides the default `border-border` color with `--glass-border-light` for a subtle transparent border matching the glass aesthetic)

Do NOT change:
- `sticky top-0 z-10` (header must remain sticky with z-10 for correct layering)
- `flex h-16 items-center justify-between px-6` (layout stays identical)
- Any child components (LocationSwitcher, ThemeToggle, UserMenu, Breadcrumbs, MobileNav)
- The MobileNav rendered below the header

Why this is safe: LocationSwitcher and ThemeToggle dropdowns use Radix UI Portals, which render outside the header DOM tree. The header's new stacking context (from sticky + z-10 + glass-surface-subtle's position:relative) cannot trap Portal-rendered content.
