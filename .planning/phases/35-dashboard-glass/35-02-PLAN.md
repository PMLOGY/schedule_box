**stat-card.tsx:**
Change the Card element from:
```tsx
<Card className={`shadow-sm hover:shadow transition-shadow ${className ?? ''}`}>
```
to:
```tsx
<Card variant="glass" className={className}>
```

Critical: REMOVE `shadow-sm hover:shadow transition-shadow` entirely. These shadow classes conflict with the glass variant's own `transition-shadow duration-200 hover:shadow-glass-hover`. The glass variant provides its own hover shadow effect. Pass only the optional `className` prop (without the empty string fallback or template literal — just pass `className` directly, it can be undefined which cn() handles fine).

**dashboard/page.tsx:**
Replace the PageHeader in the "Header row with quick actions" section. Currently:
```tsx
<PageHeader title={t('title')} />
```
Replace with an inline `<h1>` with gradient text classes:
```tsx
<h1 className="text-3xl font-bold tracking-tight bg-gradient-to-r from-blue-600 to-indigo-600 bg-clip-text text-transparent">
  {t('title')}
</h1>
```

Do NOT modify PageHeader component itself (it's used on 10+ other pages). Only replace its usage on the main dashboard page.

Do NOT add `variant="glass"` to the DemoDataCard or OnboardingChecklist Cards — these stay opaque as onboarding UI per research recommendation.

**kpi-comparison-cards.tsx:**
Change each `<Card key={kpi.title}>` to `<Card key={kpi.title} variant="glass">`. There are 4 cards mapped from the `kpis` array. Only the outer `<Card>` needs the variant prop. Also update the loading skeleton Cards in analytics/page.tsx (the `[...Array(4)].map` block) — those skeleton Cards should also use `variant="glass"` for visual consistency during loading.
