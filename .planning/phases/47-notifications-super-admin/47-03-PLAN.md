  <files>apps/web/lib/admin/impersonation.ts, apps/web/app/api/v1/admin/impersonate/route.ts, apps/web/app/api/v1/admin/companies/suspend/route.ts, apps/web/app/api/v1/auth/login/route.ts</files>
  <action>
1. Create `apps/web/lib/admin/impersonation.ts`:
   - `generateImpersonationToken(adminId, adminUuid, targetUser, targetCompanyId, ipAddress)`:
     - Generate session ID using `crypto.randomUUID()`.
     - Insert into `impersonationSessions` table: adminId, targetUserId, targetCompanyId, startedAt=now, expiresAt=now+15min, ipAddress.
     - Sign JWT with payload: `{ sub: targetUser.uuid, iss: 'schedulebox', aud: 'schedulebox-api', company_id: targetCompanyId, role: targetUser.role, permissions: [...], mfa_verified: false, token_type: 'impersonation', admin_uuid: adminUuid, session_id: sessionId }`. Use 15-minute expiry.
     - Return `{ token, sessionId, expiresAt }`.
   - `verifyImpersonationToken(token)`:
     - Verify JWT. Check `token_type === 'impersonation'`.
     - Query `impersonationSessions` for session_id — check revokedAt IS NULL.
     - If session revoked or expired, throw 401.
     - Return payload.
   - `endImpersonationSession(sessionId)`:
     - Update impersonationSessions: set revokedAt = now().
     - Return void.

2. Create `apps/web/app/api/v1/admin/impersonate/route.ts`:
   - **POST**: Start impersonation.
     - Auth: require admin role.
     - Body: `{ targetUserUuid: string }`.
     - Validate target user exists and is NOT an admin (cannot impersonate other admins).
     - Call `generateImpersonationToken()`.
     - Call `writeAuditLog({ actionType: 'impersonation_start', targetEntityType: 'user', targetEntityId: targetUserUuid })`.
     - Set `imp_token` HttpOnly cookie: `{ httpOnly: true, secure: true, sameSite: 'strict', path: '/', maxAge: 900 }`.
     - Return `{ sessionId, targetUser: { name, email, role }, expiresAt }`.
   - **DELETE**: End impersonation.
     - Read `imp_token` cookie, verify it, extract session_id.
     - Call `endImpersonationSession(sessionId)`.
     - Call `writeAuditLog({ actionType: 'impersonation_end' })`.
     - Clear `imp_token` cookie.
     - Return `{ success: true }`.

3. Create `apps/web/app/api/v1/admin/companies/suspend/route.ts`:
   - **POST**: Suspend or unsuspend a company.
     - Auth: require admin role.
     - Body: `{ companyUuid: string, action: 'suspend' | 'unsuspend', reason?: string }`.
     - For 'suspend': require `reason` field (non-empty). Update company: `suspendedAt = new Date(), suspendedReason = reason`.
     - For 'unsuspend': Update company: `suspendedAt = null, suspendedReason = null`.
     - Call `writeAuditLog()` with before/after values (include old/new suspendedAt and reason).
     - Return updated company status.

4. Modify `apps/web/app/api/v1/auth/login/route.ts`:
   - After password verification succeeds, BEFORE generating tokens:
   - Query the user's company for `suspendedAt`.
   - If `suspendedAt IS NOT NULL`, return `{ status: 403, body: { code: 'COMPANY_SUSPENDED', message: 'Ucet byl pozastaven: ' + company.suspendedReason } }`.
   - Exception: if the request includes a header or query param `billing_access=true`, allow login but with a limited permission set (billing-only). OR simpler: always allow login but return `{ suspended: true, reason: '...' }` in the response so the client can redirect to billing page. Choose the simpler approach.
