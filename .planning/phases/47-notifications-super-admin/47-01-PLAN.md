1. Install Twilio SDK: `pnpm --filter @schedulebox/web add twilio` (no @types/twilio needed — v5.x includes types).

2. Create `packages/database/src/schema/platform.ts` with ALL 6 new tables:

   a) `featureFlags` — columns: id (serial PK), name (varchar 100 unique), description (varchar 500), globalEnabled (boolean default false), createdAt, updatedAt. Indexes: name unique.

   b) `featureFlagOverrides` — columns: id (serial PK), flagId (int FK to featureFlags.id ON DELETE CASCADE), companyId (int FK to companies.id ON DELETE CASCADE), enabled (boolean), createdAt. Constraints: unique on (flagId, companyId). Indexes: flagIdx, companyIdx.

   c) `platformBroadcasts` — columns: id (serial PK), message (text), scheduledAt (timestamp tz), sentAt (timestamp tz nullable), audience (varchar 50 — 'all' | 'free' | 'essential' | 'growth' | 'ai_powered'), createdBy (int FK to users.id), createdAt. Index: scheduledIdx on scheduledAt WHERE sentAt IS NULL.

   d) `platformDailyMetrics` — columns: id (serial PK), date (timestamp tz date), metricName (varchar 100), metricValue (jsonb default {}), createdAt. Constraints: unique on (date, metricName). Index: dateIdx.

   e) `platformAuditLogs` — columns: id (serial PK), timestamp (timestamp tz defaultNow), adminId (int FK users.id), adminUuid (uuid), actionType (varchar 100), targetEntityType (varchar 50 nullable), targetEntityId (varchar 255 nullable), ipAddress (varchar 45), requestId (varchar 64), beforeValue (jsonb nullable), afterValue (jsonb nullable), metadata (jsonb default {}). Indexes: adminIdx, timestampIdx, actionIdx.

   f) `impersonationSessions` — columns: id (varchar 64 PK — nanoid), adminId (int FK users.id), targetUserId (int FK users.id), targetCompanyId (int FK companies.id nullable), startedAt (timestamp tz defaultNow), expiresAt (timestamp tz), revokedAt (timestamp tz nullable), ipAddress (varchar 45). Indexes: adminIdx, targetIdx, activeIdx (partial WHERE revoked_at IS NULL).

3. Add `suspendedReason: text('suspended_reason')` column to the companies table in `packages/database/src/schema/auth.ts` — place it right after the existing `suspendedAt` column (line ~64).

4. Add `export * from './platform';` to `packages/database/src/schema/index.ts`.

5. Run `pnpm --filter @schedulebox/database db:push` to push schema to DB (if available), otherwise just ensure the schema compiles with `pnpm --filter @schedulebox/database build` or `npx tsc --noEmit` in the database package.

Import references: Import `companies` and `users` from `./auth` in platform.ts. Use the same drizzle-orm/pg-core imports pattern as other schema files.

<task type="auto">
  <name>Task 2: Create audit helper + Twilio client + booking email functions</name>
  <files>apps/web/lib/admin/audit.ts, apps/web/lib/sms/twilio-client.ts, apps/web/lib/email/booking-emails.ts</files>
  <action>
1. Create `apps/web/lib/admin/audit.ts`:
   - Export `writeAuditLog(entry)` function matching the signature from RESEARCH.md Pattern.
   - Accepts: `{ req: NextRequest, adminUuid: string, adminId: number, actionType: string, targetEntityType?: string, targetEntityId?: string, beforeValue?: Record<string, unknown>, afterValue?: Record<string, unknown> }`.
   - Extracts IP from `x-forwarded-for` (first value) or `x-real-ip` or 'unknown'.
   - Extracts requestId from `x-vercel-id` or `x-request-id` or generates nanoid(16).
   - Inserts into `platformAuditLogs` table.
   - Does NOT try/catch — audit log failure must propagate (security requirement).
   - Use `nanoid` for request ID fallback — if nanoid not installed, use `crypto.randomUUID().slice(0,16)`.

2. Create `apps/web/lib/sms/twilio-client.ts`:
   - Import twilio SDK.
   - Create lazy-initialized client using `TWILIO_ACCOUNT_SID` and `TWILIO_AUTH_TOKEN` env vars.
   - Export `sendSMS(to: string, body: string): Promise<string>` — returns message SID.
   - Normalize phone: if `to` does not start with '+', prepend '+420' (Czech default).
   - Use `client.messages.create({ body, from: process.env.TWILIO_FROM_NUMBER, to: normalizedTo })`.
   - Guard: if env vars not set, log warning and return empty string (dev-safe).

3. Create `apps/web/lib/email/booking-emails.ts`:
   - Reuse the nodemailer transporter pattern from `lib/email/auth-emails.ts` (import the same transporter or create a shared one).
   - Read `auth-emails.ts` first to understand the exact transporter setup.
   - Export three functions:
     a) `sendBookingConfirmationEmail(data: BookingEmailData): Promise<void>` — sends branded HTML confirmation with service name, date/time, company name, booking UUID link. Plain text fallback.
     b) `sendBookingStatusChangeEmail(data: BookingEmailData & { newStatus: 'confirmed' | 'cancelled' | 'completed' }): Promise<void>` — sends status update email with Czech subject lines: "Rezervace potvrzena" / "Rezervace zrusena" / "Rezervace dokoncena".
     c) `sendBookingReminderEmail(data: BookingEmailData): Promise<void>` — sends reminder 24h before.
   - Define `BookingEmailData` interface: `{ to: string, customerName: string, serviceName: string, employeeName: string | null, startTime: Date, companyName: string, companyPhone: string | null, bookingUuid: string, locale?: 'cs' | 'sk' | 'en' }`.
   - Email HTML: simple branded template with ScheduleBox blue (#0057FF) header, white body, booking details table. Use inline CSS (no external stylesheet).
   - All functions are fire-and-forget safe — callers wrap in `.catch()`.
