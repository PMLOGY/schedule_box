  <files>apps/web/app/api/v1/admin/broadcast/route.ts, apps/web/app/api/v1/cron/broadcast-dispatch/route.ts, apps/web/components/shared/broadcast-banner.tsx, apps/web/app/[locale]/(admin)/admin/broadcast/page.tsx, apps/web/components/dashboard/dashboard-grid.tsx, vercel.json</files>
  <action>
1. Create `apps/web/app/api/v1/admin/broadcast/route.ts`:
   - **GET**: List all broadcasts (newest first). Query params: `active=true` to get only unsent/scheduled broadcasts. Also support `?current=true` to get broadcasts that have been sent and are within 7 days (for in-app banner display).
   - **POST**: Create broadcast. Auth: admin only.
     - Body: `{ message: string, scheduledAt: string (ISO date), audience: 'all' | 'free' | 'essential' | 'growth' | 'ai_powered' }`.
     - Validation: message non-empty max 1000 chars, scheduledAt must be in the future, audience must be valid enum.
     - Rate limit: check if any broadcast was created in last 10 minutes. If yes, return 429.
     - Confirmation: require `confirmCount` in body matching the number of target companies (query DB: active, non-suspended, matching plan tier). This prevents accidental mass email.
     - Insert into platformBroadcasts.
     - Call writeAuditLog.
     - Return created broadcast.

2. Create `apps/web/app/api/v1/cron/broadcast-dispatch/route.ts`:
   - Vercel Cron endpoint (GET). Validate CRON_SECRET header.
   - Query platformBroadcasts: `scheduledAt <= NOW() AND sentAt IS NULL`.
   - For each pending broadcast:
     a) Query companies matching audience filter: `isActive = true AND suspendedAt IS NULL AND (audience = 'all' OR subscriptionPlan = audience)`.
     b) For each company: send email using nodemailer transporter (reuse from auth-emails.ts). Subject: "ScheduleBox: " + first 50 chars of message. Body: full message as HTML in a branded template.
     c) Update broadcast: `sentAt = new Date()`.
   - Process max 5 broadcasts per run. Rate limit: max 100 emails per cron invocation.
   - `export const runtime = 'nodejs'`.
   - `export const maxDuration = 30`.
   - Return `{ processed: N, emailsSent: N }`.

3. Update `vercel.json` — add broadcast dispatch cron:
   ```json
   {
     "path": "/api/v1/cron/broadcast-dispatch",
     "schedule": "*/5 * * * *"
   }
   ```
   (Every 5 minutes to dispatch broadcasts close to scheduled time.)

4. Create `apps/web/components/shared/broadcast-banner.tsx`:
   - Client component that fetches active broadcasts from `GET /api/v1/admin/broadcast?current=true`.
   - Renders a yellow/blue gradient dismissible banner at the top of the dashboard.
   - Dismiss stores broadcast ID in localStorage: `dismissed_broadcast_{id}`.
   - Auto-hides broadcasts older than 7 days from sentAt.
   - Only renders on authenticated dashboard pages (not public pages).

5. Add BroadcastBanner to `apps/web/components/dashboard/dashboard-grid.tsx` (or the dashboard layout) — render above the main dashboard content for owner view.

6. Create `apps/web/app/[locale]/(admin)/admin/broadcast/page.tsx`:
   - List of all broadcasts with status (Scheduled, Sent, Draft).
   - "Create Broadcast" button opens dialog:
     - Message textarea.
     - Schedule date/time picker.
     - Audience dropdown (All, Free, Essential, Growth, AI-Powered).
     - Preview showing target company count (fetched live from API).
     - Confirmation: type the target count number to confirm.
     - Send button.
   - Glass card container. Use shadcn Dialog, Textarea, Select, Button.
   - Add translations: `admin.broadcast.*` to en.json, cs.json, sk.json.

   - **GET**: Return combined business KPIs and operational health. Auth: admin only.
   - Business KPIs (computed live from DB aggregation):
     - `newSignupsToday`: COUNT companies WHERE createdAt >= today
     - `newSignupsThisWeek`: COUNT companies WHERE createdAt >= 7 days ago
     - `totalActiveCompanies`: COUNT companies WHERE isActive = true AND suspendedAt IS NULL
     - `totalBookingsThisWeek`: COUNT bookings WHERE createdAt >= 7 days ago
     - `mrr`: SUM of monthly subscription values from subscriptions table (or compute from company plan tiers * price)
     - `churnRate`: companies deactivated this month / total companies at start of month (approximate)
   - Operational health (computed from DB):
     - `notificationDeliveryRate`: COUNT notifications WHERE status='sent' / total notifications (last 24h)
     - `smsDeliveryRate`: COUNT notifications WHERE channel='sms' AND status='sent' / total SMS notifications (last 24h)
     - `failedPaymentsToday`: COUNT payments WHERE status='failed' AND createdAt >= today
