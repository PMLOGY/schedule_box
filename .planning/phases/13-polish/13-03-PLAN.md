```bash
cd apps/web && pnpm add @react-pdf/renderer
```

**Create csv-exporter.ts:**
A utility module with a `downloadCSV` function. Uses `jsonToCSV` from `react-papaparse` (already installed). Accepts `{ data: Record<string, unknown>[], filename: string }`. Generates CSV with BOM prefix (`\uFEFF`) for Excel Czech diacritic support. Creates a Blob, generates object URL, triggers download via hidden anchor click, and revokes the URL. Export a helper `formatCSVDate(date: string): string` that formats ISO dates to `dd.MM.yyyy` (Czech date format).

Also export a `downloadBlob(blob: Blob, filename: string): void` helper for PDF downloads.

**Create revenue-report.tsx:**
A @react-pdf/renderer Document component. Props: `{ data: Array<{ date: string, revenue: number, bookings: number }>, period: string, totals: { totalRevenue: number, totalBookings: number, avgRevenue: number } }`.

Layout:
- Page size A4 with 30pt padding
- Header: "ScheduleBox - Report trzeb" + period string + generation date
- Summary section: Total revenue, total bookings, average revenue/day in a 3-column layout
- Table: Date | Revenue (CZK) | Bookings columns with zebra striping
- Footer: "Vygenerovano automaticky systemem ScheduleBox" + page number

Use StyleSheet.create for all styles. Format revenue with Czech locale (space thousands separator, comma decimal).

**Create booking-report.tsx:**
Similar structure to revenue-report but for booking stats. Props: `{ data: Array<{ date: string, completed: number, cancelled: number, noShows: number, total: number }>, period: string, totals: { completed: number, cancelled: number, noShows: number, total: number } }`.

Table columns: Date | Completed | Cancelled | No-Shows | Total.

**Create GET /api/v1/reports/revenue/pdf/route.ts:**
Protected route with BOOKINGS_READ permission. Accepts `days` query param (default 30). Queries v_daily_booking_summary for the period. Computes totals. Uses `renderToBuffer` from `@react-pdf/renderer` (NOT renderToStream -- Buffer is simpler for Next.js Response). Returns Response with:
- `Content-Type: application/pdf`
- `Content-Disposition: attachment; filename="revenue-report-{date}.pdf"`

If @react-pdf/renderer doesn't export renderToBuffer, use renderToStream and collect chunks into a Buffer.

**Create GET /api/v1/reports/bookings/pdf/route.ts:**
Same pattern as revenue PDF but uses booking-report template. Returns booking stats PDF.

IMPORTANT: Add `@react-pdf/renderer` to `serverExternalPackages` in next.config.mjs if PDF generation fails due to Node.js native module issues. This is a known issue with @react-pdf/renderer in Next.js edge runtime.

**Create export-toolbar.tsx:**
Client component with CSV and PDF export buttons for both revenue and booking data.

Props: `{ revenueData: Array<{date, revenue, bookings}> | undefined, bookingData: Array<{date, completed, cancelled, noShows, total}> | undefined, days: number, isLoading: boolean }`.

Renders a flex row with 4 buttons using shadcn Button (variant="outline", size="sm"):
1. "Export Revenue CSV" - calls `downloadCSV` with revenue data formatted for export (Date in dd.MM.yyyy, Revenue as number, Bookings as number). Filename: `revenue-{yyyy-MM-dd}.csv`.
2. "Export Bookings CSV" - calls `downloadCSV` with booking data. Filename: `bookings-{yyyy-MM-dd}.csv`.
3. "Export Revenue PDF" - fetches `/api/v1/reports/revenue/pdf?days={days}`, gets blob, calls `downloadBlob`. Shows loading state.
4. "Export Bookings PDF" - fetches `/api/v1/reports/bookings/pdf?days={days}`, gets blob, calls `downloadBlob`. Shows loading state.

Use Download icon from lucide-react. Use FileText icon for PDF buttons. Disable buttons when isLoading or when data is undefined. Use `useTranslations('analytics.export')` for button labels.

Each button has descriptive aria-label (e.g., "Export revenue data as CSV file").

On export error, show toast via sonner: `toast.error(t('error'))`. On success: `toast.success(t('success'))`.

Wrap both in a `flex items-center justify-between` container below the PageHeader.
