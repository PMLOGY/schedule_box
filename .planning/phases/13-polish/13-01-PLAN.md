  <action>
Install recharts and react-papaparse as dependencies:
```bash
cd apps/web && pnpm add recharts react-papaparse
```

Create three protected API routes using the existing createRouteHandler pattern with BOOKINGS_READ permission:

1. **GET /api/v1/analytics/revenue** - Accepts `days` query param (7, 30, 90; default 30). Queries v_daily_booking_summary view filtered by company_id (from session) and date >= now - days. Returns array of `{ date: string, revenue: number, bookings: number }` ordered by date ASC. Use `successResponse()` wrapper.

2. **GET /api/v1/analytics/bookings** - Same parameters. Queries v_daily_booking_summary. Returns array of `{ date: string, completed: number, cancelled: number, noShows: number, total: number }` ordered by date ASC.

3. **GET /api/v1/analytics/overview** - Accepts `days` query param (default 30). Computes aggregated KPIs for the requested period AND the previous period of equal length. Returns:
```typescript
{
  currentPeriod: {
    totalBookings: number,
    totalRevenue: number,
    completedBookings: number,
    cancelledBookings: number,
    noShows: number,
    avgRevenuePerDay: number
  },
  previousPeriod: { /* same shape */ },
  comparison: {
    revenueChange: number,     // percentage
    bookingsChange: number,    // percentage
    noShowChange: number       // percentage
  }
}
```

For the overview endpoint, run two queries against v_daily_booking_summary: one for current period, one for previous period. Compute percentages in the route handler. Use `COALESCE` for NULL revenue sums (no bookings = 0 revenue).

Use the existing findCompanyId helper to resolve companyId from the authenticated user. Follow the pattern from existing booking routes: import createRouteHandler from `@/lib/auth/route-handler`, use Zod for query param validation with `z.coerce.number()`.

Import the `dailyBookingSummary` view from `@schedulebox/database` and query it with Drizzle's `db.select().from(dailyBookingSummary).where(...)`.

Create the following components:

1. **chart-container.tsx** - Accessible wrapper for all Recharts charts. Props: `{ children, title, description, id }`. Renders `<figure role="img" aria-labelledby aria-describedby>` with `<ResponsiveContainer width="100%" height={350}>`, sr-only title/description divs, and visible figcaption. This ensures WCAG 2.1 AA compliance for chart accessibility.

2. **revenue-chart.tsx** - Client component (`'use client'`). Renders Recharts `<LineChart>` with `accessibilityLayer` prop inside `<AccessibleChart>`. Props: `{ data: Array<{ date: string, revenue: number }> }`. XAxis formats dates with date-fns `format(new Date(date), 'MMM d')`. YAxis formats as `Xk CZK`. Tooltip formats with Czech locale `toLocaleString('cs')`. Use ScheduleBox primary color `#3B82F6` for the line. Add `strokeWidth={2}` and `activeDot={{ r: 8 }}`.

3. **booking-stats-chart.tsx** - Client component. Renders Recharts `<BarChart>` with `accessibilityLayer` inside `<AccessibleChart>`. Props: `{ data: Array<{ date: string, completed: number, cancelled: number, noShows: number }> }`. Three stacked bars: completed (green #22C55E), cancelled (red #EF4444, strokeDasharray for color-blind differentiation), no-shows (amber #F59E0B). Include `<Legend />` and `<Tooltip />`.

4. **kpi-comparison-cards.tsx** - Displays 4 KPI cards using existing Card component. Props: `{ overview: OverviewData }`. Cards: Total Bookings, Total Revenue, Avg Revenue/Day, No-Show Rate. Each card shows current value and a percentage change badge (green for positive, red for negative). Use `ArrowUp` / `ArrowDown` icons from lucide-react.

5. **period-selector.tsx** - Client component with shadcn Select. Options: 7 days, 30 days, 90 days. Props: `{ value: number, onChange: (days: number) => void }`. Uses useTranslations('analytics.period') for labels.

6. **use-revenue-analytics.ts** - TanStack Query hook. Accepts `days: number`. Fetches `/analytics/revenue?days={days}`. Returns typed revenue data array. staleTime: 5 minutes.

7. **use-booking-analytics.ts** - TanStack Query hook. Accepts `days: number`. Fetches `/analytics/bookings?days={days}`. Returns typed booking stats array. Also fetches `/analytics/overview?days={days}` for KPI data. staleTime: 5 minutes.

8. **Update use-analytics-query.ts** - Refactor to use the new `/analytics/overview` endpoint instead of aggregating from /customers and /bookings. Accept `days` param. This simplifies the hook and provides accurate data from DB views.

9. **Update analytics/page.tsx** - Complete rewrite. Add `days` state (default 30). Render PeriodSelector, KpiComparisonCards, and two chart sections. Import charts with `dynamic(() => import(...), { ssr: false, loading: () => <Skeleton /> })` to prevent SSR hydration issues. Use the refactored hooks. Layout: period selector at top-right, 4 KPI cards in a grid, revenue chart in full-width card, booking stats chart in full-width card. Use existing PageHeader, Card, CardHeader, CardContent components.

All chart components use `useTranslations('analytics')` for labels. All text must reference translation keys (the existing cs.json already has `analytics.*` keys; use them and note any missing keys for Plan 02 to add).
