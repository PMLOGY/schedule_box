  <action>
Create `packages/events/src/types.ts` with CloudEvents v1.0 spec types:

```typescript
interface CloudEvent<T = unknown> {
  specversion: '1.0';
  type: string;           // e.g., 'com.schedulebox.booking.created'
  source: string;         // e.g., 'booking-service'
  id: string;             // UUID
  time: string;           // ISO 8601
  subject?: string;       // Entity UUID
  datacontenttype: string; // 'application/json'
  data: T;
}

interface EventMetadata {
  companyId: number;
  userId?: number;
  correlationId?: string;
}
```

Export a `DomainEvent<T>` type that combines `CloudEvent<T>` with `EventMetadata`.

Create `packages/events/src/publisher.ts`:

1. Use `amqplib` (check if installed in packages/events/package.json; if not, install it)
2. Create a singleton connection pattern: `let connection: Connection | null = null; let channel: Channel | null = null;`
3. `getChannel()`: lazily creates connection to `process.env.RABBITMQ_URL || 'amqp://schedulebox:schedulebox@localhost:5672'`
4. `publishEvent<T>(event: CloudEvent<T>)`:
   - Get channel
   - Assert topic exchange: `schedulebox.events` (durable: true)
   - Publish with routing key derived from event type: `com.schedulebox.booking.created` -> `booking.created`
   - Message properties: `{ contentType: 'application/json', persistent: true, messageId: event.id, timestamp: Date.now(), headers: { specversion: '1.0', type: event.type, source: event.source } }`
5. `closeConnection()`: graceful shutdown
6. Include try/catch with console.error logging. If RabbitMQ is unavailable, log error but DO NOT throw (fire-and-forget for MVP, reliable delivery in Phase 7).

Add a helper `createCloudEvent<T>(type: string, source: string, data: T, subject?: string): CloudEvent<T>` that generates UUID id, ISO time, and sets specversion.

Run `cd D:/Project/ScheduleBox && pnpm exec tsc --noEmit -p packages/events/tsconfig.json` to verify TypeScript compiles. Verify amqplib is in packages/events/package.json.
  </verify>
  <done>CloudEvent types defined, RabbitMQ publisher connects and publishes to schedulebox.events exchange with fire-and-forget semantics.</done>
