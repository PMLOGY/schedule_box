<objective>
Build the 4-step booking wizard form with per-step Zod validation, availability grid, and booking submission.

Purpose: This is the primary booking flow for admin users creating bookings on behalf of customers. It follows the pattern: select service -> pick date/time -> enter customer info -> review and confirm. Uses Zustand for cross-step state persistence and React Hook Form + Zod for per-step validation per research recommendations.

Output: Complete booking wizard accessible at /bookings/new with service selection, date/time picker with real availability data, customer form, and confirmation step.
</objective>

```typescript
interface BookingWizardData {
  // Step 1
  serviceId?: number;
  serviceName?: string;
  serviceDuration?: number;
  servicePrice?: string;
  // Step 2
  employeeId?: number;
  employeeName?: string;
  selectedDate?: string;    // YYYY-MM-DD
  startTime?: string;       // ISO 8601 datetime
  endTime?: string;         // ISO 8601 datetime
  displayTime?: string;     // HH:mm for display
  // Step 3
  customerId?: number;      // Existing customer
  customerName?: string;
  customerEmail?: string;
  customerPhone?: string;
  notes?: string;
}

interface BookingWizardState {
  step: 1 | 2 | 3 | 4;
  data: BookingWizardData;
  isSubmitting: boolean;
  error: string | null;
  setStep: (step: 1 | 2 | 3 | 4) => void;
  nextStep: () => void;
  prevStep: () => void;
  updateData: (data: Partial<BookingWizardData>) => void;
  setSubmitting: (isSubmitting: boolean) => void;
  setError: (error: string | null) => void;
  reset: () => void;
}
```

**Step1ServiceSelect.tsx**:
- Fetch active services from GET /api/v1/services using TanStack Query (useQuery)
- Group services by category (categoryId)
- Display as cards/radio buttons with: name, duration, price
- On select: `updateData({ serviceId, serviceName, serviceDuration, servicePrice })` then `nextStep()`
- Optional employee preference select (dropdown of employees assigned to selected service via GET /api/v1/employees?service_id=X)

**Step2DateTimeSelect.tsx**:
- Date picker (use shadcn/ui Calendar component) defaulting to tomorrow
- When date selected, fetch availability from GET /api/v1/availability:
  - `company_slug` from auth store or settings
  - `service_id` from wizard data
  - `employee_id` from wizard data (optional)
  - `date_from` and `date_to` = selected date (single day)
- Render `AvailabilityGrid` with fetched slots
- Loading skeleton while fetching

**AvailabilityGrid.tsx**:
- Receives `slots: AvailabilitySlot[]` and `onSelect: (slot) => void`
- Groups slots by employee (show employee name header if multiple)
- Renders time buttons in a grid (4 columns on desktop, 2 on mobile)
- Each button shows HH:mm, green for available
- On click: `updateData({ employeeId, employeeName, startTime, endTime, displayTime, selectedDate })` then `nextStep()`
- Empty state message if no slots available: "No available times for this date"

**Step3CustomerInfo.tsx**:
- Two modes: "Select existing customer" (searchable dropdown from GET /api/v1/customers?search=X) OR "New customer" (form fields)
- Use React Hook Form with zodResolver
- Per-step Zod schema:
  ```
  customerName: z.string().min(2)
  customerEmail: z.string().email().optional()
  customerPhone: z.string().regex(/^\+?[0-9]{9,15}$/).optional()
  notes: z.string().max(1000).optional()
  ```
- At least name is required. If selecting existing customer, auto-fill fields.
- On submit: `updateData(values)` then `nextStep()`
- Back button calls `prevStep()`

**Step4Confirmation.tsx**:
- Summary card showing all selected data:
  - Service: name, duration, price
  - Date & Time: formatted date + time
  - Employee: name
  - Customer: name, email, phone
  - Notes (if any)
- "Confirm Booking" button:
  - Uses TanStack Query `useMutation` to POST /api/v1/bookings
  - Build request body from wizard data
  - On success: show success toast (sonner), reset wizard, redirect to /bookings or /bookings/:id
  - On error: if 409 (SLOT_TAKEN), show alert "This slot is no longer available, please select a different time" and go back to step 2. For other errors, show generic error message.
  - Disable button while submitting (isSubmitting state)
- "Back" button to go to Step 3
- "Cancel" button to reset wizard and go to /bookings

Use `apiClient` from `apps/web/lib/api-client.ts` for all API calls. Use `useTranslations` from next-intl for all user-facing strings.
