Create `apps/web/lib/booking/buffer-time.ts`:

1. `calculateBookingTimeBlock(service: { durationMinutes: number; bufferBeforeMinutes: number; bufferAfterMinutes: number }, appointmentStart: Date)`:
   - Returns `{ appointmentStart, appointmentEnd, blockStart, blockEnd, totalMinutes }`
   - `appointmentEnd = appointmentStart + durationMinutes`
   - `blockStart = appointmentStart - bufferBeforeMinutes` (for conflict detection)
   - `blockEnd = appointmentEnd + bufferAfterMinutes` (for conflict detection)
   - Use `addMinutes` from `date-fns`

2. `isSlotConflicting(slotStart: Date, slotEnd: Date, blockedPeriods: Array<{ start: Date; end: Date }>): boolean`:
   - Returns true if slot overlaps with any blocked period
   - Overlap: `slotStart < blockedEnd && slotEnd > blockedStart`

Create `apps/web/lib/booking/availability-engine.ts`:

Import from `@schedulebox/database` (db client), Drizzle ORM operators, and date-fns.

`calculateAvailability(params: AvailabilityParams): Promise<AvailabilitySlot[]>`:

Parameters:
```typescript
interface AvailabilityParams {
  companyId: number;
  serviceId: number;
  employeeId?: number;
  dateFrom: string; // YYYY-MM-DD
  dateTo: string;   // YYYY-MM-DD
  timezone: string; // e.g., 'Europe/Prague'
}
```

Implementation approach (single SQL query per research recommendation):

Step 1: Fetch service details (duration, bufferBefore, bufferAfter) from services table. Throw AppError 404 if not found or not active.

Step 2: Determine which employees can provide this service. Query `employee_services` JOIN `employees` where `serviceId` matches and employee is active. If `employeeId` provided, filter to that employee. If no matching employees, return empty array.

Step 3: Execute a raw SQL query via `db.execute(sql`...`)` that:
- Generates date series from dateFrom to dateTo
- For each date + employee, gets working hours:
  - FIRST check working_hours_overrides (if override exists for date+employee: use override hours OR skip if is_day_off=true)
  - ELSE use regular working_hours for that dayOfWeek
- LEFT JOINs existing bookings for those employees in the date range (status NOT IN cancelled, expired) along with their service buffer times
- Returns rows: `{ date, employee_id, employee_name, start_time, end_time, blocked_periods_json }`

Step 4: In application code, iterate periods and generate 15-minute interval slots:
- For each working period, start at period start_time
- While (currentSlotStart + totalDuration) <= period end_time:
  - Check if slot conflicts with any blocked period using `isSlotConflicting`
  - If not blocked, add to results: `{ date, startTime (HH:mm), endTime (HH:mm), employeeId, employeeName, isAvailable: true }`
  - Advance by 15 minutes

CRITICAL per research pitfall #2: Buffer times are applied to EXISTING bookings (expand their blocked range), NOT to the new slot. A booking from 10:00-10:30 with 15min buffer_after blocks 10:00-10:45. The new slot of 10:45-11:15 is NOT additionally buffered.

Handle the company timezone correctly: working hours TIME values are in local time. Convert slot times to the company's timezone for display. All comparisons with existing bookings (TIMESTAMPTZ) must account for timezone.

For the date-fns timezone handling, use `@date-fns/tz` functions: `TZDate` for timezone-aware date construction.

Create `apps/web/app/api/v1/availability/route.ts`:

This is a PUBLIC endpoint (no auth required per API spec line 2668: `security: []`).

**GET handler:**

1. Parse query params from URL: `company_slug`, `service_id`, `employee_id`, `date_from`, `date_to`
2. Validate with `availabilityRequestSchema` from `@schedulebox/shared`. If invalid, return 400 with error details.
3. Look up company by slug: `SELECT id, timezone FROM companies WHERE slug = $1`. If not found, return 404.
4. Call `calculateAvailability({ companyId: company.id, serviceId, employeeId, dateFrom, dateTo, timezone: company.timezone || 'Europe/Prague' })`
5. Return `{ data: { slots } }` with 200 status

Do NOT use `createRouteHandler` factory since this is public (no auth). Use a simple try/catch with `handleRouteError` for error formatting.

Handle edge cases:
- Invalid date range (date_to before date_from) -> 400
- Date range > 31 days -> 400 (already validated by schema refine)
- Company not found -> 404
- Service not found or not active -> 404
- No employees for service -> return empty slots array (not an error)
