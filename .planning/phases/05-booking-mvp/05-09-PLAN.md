    Complete Booking MVP with:
    - Availability engine (working hours + overrides + buffer times + blocked slots)
    - Booking CRUD with double-booking prevention (SELECT FOR UPDATE + btree_gist)
    - 4-step booking wizard (service -> date/time -> customer -> confirm)
    - Status transitions (confirm, cancel, complete, no-show, reschedule)

    - Booking list with filters and pagination
    - Admin time blocking
    - RabbitMQ domain events
    - Booking expiration for stale pending bookings

  <how-to-verify>
    Prerequisites: Docker running (`docker compose up -d`), dev server running (`pnpm dev`)

    1. **Booking Creation Flow**
       - Navigate to /bookings/new
       - Select a service from the list (verify services load from API)
       - Pick a date and verify available time slots appear
       - Select a time slot
       - Enter customer information (test validation — submit with empty name)
       - Review confirmation page (verify all details correct)
       - Click "Confirm Booking" — should see success toast and redirect

    2. **Calendar View**
       - Navigate to /calendar
       - Verify the new booking appears as a colored event
       - Switch between Day/Week/Month views
       - Drag the booking event to a different time — verify reschedule confirmation
       - Click the booking event — verify detail panel opens with correct info

    3. **Status Transitions**
       - From the detail panel, click "Confirm" on a pending booking
       - Verify status badge changes to "Confirmed"
       - Click "Complete" — verify status changes
       - Create another booking and cancel it — verify cancellation with reason

    4. **Double-Booking Prevention**
       - Note the time of an existing confirmed booking
       - Try to create a new booking at the exact same time + employee
       - Verify you get a "slot not available" error, not a successful booking

    5. **Booking List**
       - Navigate to /bookings
       - Verify table shows bookings with correct data
       - Test status filter — select "Confirmed" only
       - Test date range filter
       - Verify pagination if enough bookings exist

    6. **Time Blocking**
       - Create a schedule override (block a time period) for an employee
       - Verify the availability endpoint no longer shows slots during blocked time
       - Verify the booking wizard step 2 does not offer blocked slots
