Create `apps/web/hooks/use-bookings-query.ts`:

1. `useBookingsQuery(params: BookingListQuery)`:
   - TanStack Query `useQuery` wrapping GET /api/v1/bookings with query params
   - Query key: `['bookings', params]`
   - Returns `{ data, meta, isLoading, error }`
   - `staleTime: 30_000` (30 seconds, bookings change frequently)

2. `useBookingsForCalendar(dateFrom: string, dateTo: string, employeeIds?: number[])`:
   - Specialized hook for calendar view, fetches bookings in date range
   - Query key: `['bookings', 'calendar', dateFrom, dateTo, employeeIds]`
   - Limit: 500 (calendar shows limited range)
   - Transforms response to FullCalendar EventInput format:
     ```typescript
     events.map(b => ({
       id: String(b.id),
       title: `${b.customer.name} - ${b.service.name}`,
       start: b.startTime,
       end: b.endTime,
       backgroundColor: b.service.color || b.employee?.color || '#3B82F6',
       borderColor: 'transparent',
       extendedProps: { booking: b },
       resourceId: b.employee ? String(b.employee.id) : undefined,
     }))
     ```

3. `useBookingDetail(bookingId: number | null)`:
   - Fetches single booking detail when id is provided
   - Disabled when bookingId is null
   - Query key: `['bookings', bookingId]`

Create `apps/web/hooks/use-reschedule-booking.ts`:

Per research Pattern 4 (optimistic updates):
- `useMutation` calling POST /api/v1/bookings/:id/reschedule
- `onMutate`: Cancel outgoing queries, snapshot calendar data, optimistically update event position
- `onError`: Rollback to snapshot, show toast "Failed to reschedule. Slot may be taken."
- `onSettled`: Invalidate bookings queries to refetch
- Accept `revertFn` parameter for FullCalendar's `info.revert()` on failure
