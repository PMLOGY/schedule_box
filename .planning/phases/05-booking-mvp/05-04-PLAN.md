  <name>Task 1: Booking Service Layer</name>
  <files>apps/web/lib/booking/booking-service.ts</files>
  <action>
Create `apps/web/lib/booking/booking-service.ts` with these functions:

1. **createBooking(input: BookingCreate, context: { companyId: number; userId: number }): Promise<Booking>**

   Implementation inside `db.transaction(async (tx) => { ... })`:
   a. Fetch service: verify it exists, is active, belongs to company. Get duration, price, bufferBefore, bufferAfter.
   b. Calculate endTime from startTime + durationMinutes.
   c. If employeeId provided, verify employee exists, is active, belongs to company, and is assigned to this service (employee_services junction).
   d. If no employeeId, pick first available employee for this service (query employee_services, check availability).
   e. **SELECT FOR UPDATE on employee row**: `tx.select().from(employees).where(eq(employees.id, employeeId)).for('update')` to lock the row.
   f. **Re-check availability within transaction**: query bookings table for overlapping time slots on this employee. Include buffer time expansion: check if `tstzrange(startTime - bufferBefore, endTime + bufferAfter)` overlaps with any existing non-cancelled/expired booking's buffered range.
   g. If conflict found, throw AppError with code `'SLOT_TAKEN'`, status 409.
   h. Insert booking with pricing snapshot from service (price, currency).
   i. If resource_ids provided, insert into booking_resources junction table.
   j. Return inserted booking.

   After transaction commits:
   k. Publish `booking.created` event via `publishEvent(createBookingCreatedEvent(...))`.
   l. Return booking with joined customer/service/employee data.

2. **listBookings(query: BookingListQuery, companyId: number): Promise<{ data: Booking[]; meta: PaginationMeta }>**
   - Query bookings table with Drizzle, filtered by companyId
   - Apply optional filters: status, customer_id, employee_id, service_id, date_from, date_to, source
   - JOIN customer (name, email), service (name), employee (name) for response enrichment
   - Apply pagination (offset = (page-1) * limit)
   - Return total count for pagination meta
   - Exclude soft-deleted bookings (deletedAt IS NULL)

3. **getBooking(bookingId: number, companyId: number): Promise<Booking | null>**
   - Query with JOINs for customer, service, employee data
   - Filter by companyId for tenant isolation
   - Return null if not found

4. **updateBooking(bookingId: number, input: BookingUpdate, companyId: number): Promise<Booking>**
   - Get existing booking (404 if not found)
   - If start_time changed, re-validate availability (same SELECT FOR UPDATE pattern)
   - Update fields, set updatedAt
   - Return updated booking with joins

5. **deleteBooking(bookingId: number, companyId: number): Promise<void>**
   - Soft delete: SET deletedAt = NOW()
   - 404 if not found

Use UUID in API responses, SERIAL internally. Map `id` -> `uuid` when returning to client. Accept both UUID and SERIAL id params (check if param is UUID format, look up accordingly).

Follow existing patterns from customer/service CRUD in Phase 3:
- Use `createRouteHandler` factory for protected endpoints
- Use `successResponse`, `errorResponse`, `paginatedResponse` from `lib/utils/response.ts`
- Use `AppError` from `lib/utils/errors.ts`
