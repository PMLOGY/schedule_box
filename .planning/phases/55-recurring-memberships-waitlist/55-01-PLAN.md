Create three new Drizzle schema files following existing patterns (serial PK, uuid for public, company_id FK with cascade, timestamps, CHECK constraints, indexes).

**packages/database/src/schema/recurring.ts** — `recurringSeries` table:
- id: serial PK
- uuid: uuid defaultRandom unique
- companyId: integer FK -> companies.id (cascade)
- serviceId: integer FK -> services.id (restrict)
- employeeId: integer FK -> employees.id (set null), nullable
- customerId: integer FK -> customers.id (restrict)
- repeatPattern: varchar(20) — 'weekly' | 'biweekly' | 'monthly'
- startDate: date — first occurrence date
- endDate: date — nullable, when series ends (null = indefinite)
- startTime: time — time of day for occurrences
- durationMinutes: integer — duration of each occurrence
- maxOccurrences: integer — nullable, alternative to endDate
- notes: text — nullable
- isActive: boolean default true
- createdAt, updatedAt timestamps
- CHECK on repeatPattern IN ('weekly', 'biweekly', 'monthly')
- Index on companyId, index on (companyId, isActive)

**packages/database/src/schema/memberships.ts** — two tables:

`membershipTypes` table:
- id: serial PK
- uuid: uuid defaultRandom unique
- companyId: integer FK -> companies.id (cascade)
- name: varchar(255)
- description: text nullable
- type: varchar(20) — 'monthly' | 'annual' | 'punch_card'
- price: numeric(10,2)
- currency: varchar(3) default 'CZK'
- punchesIncluded: integer nullable — only for punch_card type
- durationDays: integer nullable — validity period (e.g., 30 for monthly, 365 for annual)
- serviceIds: jsonb nullable — array of service IDs this membership applies to (null = all)
- isActive: boolean default true
- createdAt, updatedAt timestamps
- CHECK on type IN ('monthly', 'annual', 'punch_card')
- Index on companyId

`customerMemberships` table:
- id: serial PK
- uuid: uuid defaultRandom unique
- companyId: integer FK -> companies.id (cascade)
- customerId: integer FK -> customers.id (cascade)
- membershipTypeId: integer FK -> membershipTypes.id (restrict)
- status: varchar(20) default 'active' — 'active' | 'expired' | 'cancelled' | 'suspended'
- startDate: date
- endDate: date nullable
- remainingUses: integer nullable — for punch cards, decremented on booking
- createdAt, updatedAt timestamps
- CHECK on status IN ('active', 'expired', 'cancelled', 'suspended')
- Index on (companyId, customerId), index on (companyId, status)

**packages/database/src/schema/waitlist.ts** — `bookingWaitlist` table:
- id: serial PK
- uuid: uuid defaultRandom unique
- companyId: integer FK -> companies.id (cascade)
- customerId: integer FK -> customers.id (cascade)
- serviceId: integer FK -> services.id (restrict)
- employeeId: integer FK -> employees.id (set null), nullable
- preferredTime: timestamp with timezone — the slot they wanted
- position: integer — queue position (1-based)
- status: varchar(20) default 'waiting' — 'waiting' | 'promoted' | 'expired' | 'cancelled'
- promotedAt: timestamp nullable
- promotedBookingId: integer nullable — FK to bookings.id if promoted
- notifiedAt: timestamp nullable
- createdAt, updatedAt timestamps
- CHECK on status IN ('waiting', 'promoted', 'expired', 'cancelled')
- Index on (companyId, serviceId, preferredTime), index on (companyId, status)
- UNIQUE on (companyId, customerId, serviceId, preferredTime, status) with partial WHERE status='waiting' to prevent duplicate waitlist entries
