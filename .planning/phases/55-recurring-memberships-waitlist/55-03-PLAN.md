**apps/web/validations/membership.ts** — Zod schemas:
- `membershipTypeCreateSchema`: name (string 1-255), description (string optional), type ('monthly' | 'annual' | 'punch_card'), price (string numeric, positive), currency (string default 'CZK'), punchesIncluded (number optional — required if type='punch_card', refine), durationDays (number optional — required if type='monthly' or 'annual', refine: monthly defaults 30, annual defaults 365), serviceIds (array of string uuids, optional — null means all services), isActive (boolean default true)
- `membershipTypeUpdateSchema`: partial of create
- `customerMembershipAssignSchema`: membershipTypeId (string uuid), startDate (string ISO date), endDate (string ISO date optional — auto-calculated from durationDays if not given)

**apps/web/lib/membership/membership-service.ts** — Service functions:

`createMembershipType(data, companyId)` — Insert into membershipTypes, return created record with uuid
`updateMembershipType(typeId, data, companyId)` — Update by uuid, tenant-scoped
`deleteMembershipType(typeId, companyId)` — Soft delete via isActive=false
`listMembershipTypes(companyId)` — List all active types for company
`getMembershipType(typeId, companyId)` — Get single by uuid

`assignMembership(customerId, data, companyId)`:
1. Resolve customer UUID to internal ID
2. Resolve membershipType UUID to internal ID
3. Calculate endDate from startDate + durationDays if not explicitly provided
4. For punch_card type, set remainingUses = membershipType.punchesIncluded
5. Insert into customerMemberships with status='active'
6. Return created record

`listCustomerMemberships(customerId, companyId)` — List all memberships for a customer (include membershipType name/type in response via join)

`validateMembershipForBooking(customerId, serviceId, companyId)`:
1. Find active customer_memberships for this customer WHERE status='active' AND (endDate IS NULL OR endDate >= today)
2. Check if any membership's serviceIds includes this serviceId (or serviceIds is null = all services)
3. For punch_card type: check remainingUses > 0
4. Return { valid: boolean, membership: CustomerMembership | null, reason?: string }
5. Reasons: 'no_active_membership', 'membership_expired', 'service_not_covered', 'no_remaining_uses'

`decrementPunchCard(customerMembershipId, companyId)`:
1. Decrement remainingUses by 1
2. If remainingUses reaches 0, set status='expired'
3. Use dbTx for atomicity

**API Routes** — follow createRouteHandler pattern:

`/api/v1/memberships` (route.ts):
- GET: BOOKINGS_READ permission. List membership types for company.
- POST: BOOKINGS_CREATE permission (or add MEMBERSHIPS_CREATE if granular RBAC exists). Create membership type.

`/api/v1/memberships/[id]` (route.ts):
- GET, PUT, DELETE for individual membership type (by UUID).

`/api/v1/customers/[id]/memberships` (route.ts):
- GET: List customer's memberships. The [id] is customer UUID.
- POST: Assign membership to customer. Validate with customerMembershipAssignSchema.

**Booking integration** — In `apps/web/lib/booking/booking-service.ts`, add a membership validation call inside createBooking:
1. After resolving the customer, call validateMembershipForBooking
2. If the customer HAS an active membership covering this service, proceed (optionally apply a discount or mark the booking as membership-covered)
3. If punch_card and valid, call decrementPunchCard after booking creation
4. Do NOT block booking creation if customer has NO membership — memberships are optional perks. Only block if customer explicitly tries to USE an expired/exhausted membership (this would be a future UI concern; for now, just validate and decrement if applicable)
