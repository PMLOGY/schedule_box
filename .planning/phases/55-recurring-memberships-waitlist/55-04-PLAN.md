**apps/web/validations/waitlist.ts** — Zod schemas:
- `waitlistJoinSchema`: serviceId (string uuid), employeeId (string uuid, optional), customerId (string uuid), preferredTime (string ISO datetime)

**apps/web/lib/waitlist/waitlist-service.ts** — Service functions:

`joinWaitlist(data, companyId)`:
1. Resolve UUIDs to internal IDs
2. Verify the service exists and is a group class (maxCapacity > 1) — return error if service is 1:1
3. Count existing confirmed/pending bookings for this service + time slot — verify it IS actually full (count >= maxCapacity). If not full, return error telling client to just book normally.
4. Check no duplicate waitlist entry (same customer, service, time, status='waiting')
5. Calculate position: count existing 'waiting' entries for same service+time + 1
6. Insert into bookingWaitlist with status='waiting'
7. Return the waitlist entry with position

`promoteFromWaitlist(serviceId, employeeId, preferredTime, companyId)`:
1. Find first 'waiting' entry for this service+time ordered by position ASC
2. If none found, return null (no one waiting)
3. Create a booking for the promoted customer using createBooking from booking-service.ts
4. Update waitlist entry: status='promoted', promotedAt=now, promotedBookingId=new booking id
5. Send notifications (fire-and-forget, never throw):
   a. Push: find customer's userId, call sendPushToUser with title "Místo se uvolnilo!" body "Byli jste automaticky přesunuti z čekací listiny na {serviceName}."
   b. Email: call sendBookingConfirmationEmail with the new booking details (reuse the confirmation template — the customer effectively gets a new booking confirmation)
   c. Create notification record in notifications table with channel='push' and 'email' for audit trail
6. Update waitlist entry: notifiedAt=now
7. Return the promoted entry + new booking

`getWaitlistForSlot(serviceId, preferredTime, companyId)` — List all waiting entries for a specific slot, ordered by position
`getWaitlistEntry(entryId, companyId)` — Get single entry by UUID
`cancelWaitlistEntry(entryId, companyId)` — Set status='cancelled', reposition remaining entries (decrement position for entries after this one)
`getCustomerWaitlistEntries(customerId, companyId)` — List customer's active waitlist entries

**API Routes** — follow createRouteHandler pattern:

`/api/v1/bookings/waitlist` (route.ts):
- POST: BOOKINGS_CREATE permission. Validate with waitlistJoinSchema. Call joinWaitlist. Return 201 with entry + position.
- GET: BOOKINGS_READ permission. Query params: serviceId (optional), status (optional, default 'waiting'). List waitlist entries for company.

`/api/v1/bookings/waitlist/[id]` (route.ts):
- GET: Get single waitlist entry by UUID.
- DELETE: Cancel waitlist entry. Call cancelWaitlistEntry. Return 200.
