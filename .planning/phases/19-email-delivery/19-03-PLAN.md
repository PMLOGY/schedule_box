<objective>
Fix Helm chart SMTP configuration and update developer documentation so Brevo credentials are properly injected into the notification worker at runtime.

Purpose: The worker-deployment.yaml currently has hardcoded `value: ""` env entries for SMTP_HOST, SMTP_USER, and SMTP_PASS. In Kubernetes, explicit `env` entries take precedence over `envFrom` — so these empty strings override the Kubernetes secret, preventing SMTP from working even if credentials are set in the secret. The .env.example still documents SendGrid credentials. Both must be fixed before email delivery can work in production.

Output: Updated `secrets.yaml` with SMTP fields, updated `worker-deployment.yaml` with empty overrides removed, updated `.env.example` with Brevo settings.
</objective>

    ```yaml
    SMTP_HOST: {{ .Values.secrets.smtpHost | default "smtp-relay.brevo.com" | quote }}
    SMTP_PORT: {{ .Values.secrets.smtpPort | default "587" | quote }}

    ```

    **ExternalSecret block** (inside `data:` array, after existing entries):
    ```yaml
    - secretKey: SMTP_HOST
      remoteRef:
        key: {{ .Values.externalSecrets.remoteKeyPrefix }}/smtp-host
    - secretKey: SMTP_USER
      remoteRef:
        key: {{ .Values.externalSecrets.remoteKeyPrefix }}/smtp-user
    - secretKey: SMTP_PASS
      remoteRef:
        key: {{ .Values.externalSecrets.remoteKeyPrefix }}/smtp-pass
    - secretKey: SMTP_FROM
      remoteRef:
        key: {{ .Values.externalSecrets.remoteKeyPrefix }}/smtp-from
    ```

    Do NOT add SMTP_PORT to ExternalSecret (it has a safe default in the native block). Both blocks are controlled by `{{- if not .Values.externalSecrets.enabled }}` and `{{- if .Values.externalSecrets.enabled }}` conditionals already in the file — add within the correct conditional.

    The `smtpUser` and `smtpPass` fields intentionally have NO default — they must be explicitly provided at deploy time. If `.Values.secrets.smtpUser` is empty, Helm will render an empty string (Kubernetes will have an empty SMTP_USER, and the worker will log "SMTP not configured"). This is acceptable — the secret must be set to enable email.
