<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure Brevo account and DNS authentication records</name>
  <action>
    This is a human-only step. Claude cannot access the Brevo dashboard or your DNS provider.
    Complete the following steps in order. Do not rush DNS changes — mistakes take 24-48h to correct.
  </action>
  <what-to-do>
    **Step 1: Create Brevo account (if not already done)**
    - Go to https://app.brevo.com and create a free account
    - Free tier: 300 emails/day — sufficient for launch
    - Verify your Brevo account email

    **Step 2: Get Brevo SMTP credentials**
    - In Brevo: SMTP & API → SMTP tab
    - Note your SMTP_USER = your Brevo account login email address
    - Click "Generate a new SMTP key" → copy the key (this is SMTP_PASS)
    - WARNING: Do NOT use the API key — it is different from the SMTP key. The SMTP key starts with "xsmtpsib-"

    **Step 3: Add schedulebox.cz domain in Brevo**
    - In Brevo: Senders, Domains, and Dedicated IPs → Domains → Add domain
    - Enter: schedulebox.cz
    - Choose "Authenticate with CNAME" (2048-bit keys — preferred over TXT method)
    - Brevo will show you 2 CNAME records — COPY BOTH RECORD NAMES AND VALUES

    **Step 4: Set DNS TTL to 300 seconds BEFORE adding records**
    - In your DNS provider (Cloudflare / Wedos / Route53 / other):
    - Find the schedulebox.cz zone
    - Temporarily set TTL to 300 seconds (5 minutes) for fast propagation
    - This allows fast correction if you make a mistake

    **Step 5: Add DKIM CNAME records to DNS**
    - Add both CNAME records shown in Brevo dashboard
    - They will look like: `brevo-code._domainkey.schedulebox.cz → <something>.dkim.brevo.com`
    - After adding, verify propagation: https://mxtoolbox.com/dkim.aspx
      - Enter selector (from Brevo, e.g. "brevo-code") and domain "schedulebox.cz"
      - Wait until it shows "DKIM record found" before proceeding

    **Step 6: Add DMARC TXT record to DNS**
    - Type: TXT
    - Name: _dmarc.schedulebox.cz (or just _dmarc if your DNS provider adds the zone automatically)
    - Value: `v=DMARC1; p=none; rua=mailto:dmarc@schedulebox.cz; ruf=mailto:dmarc@schedulebox.cz; fo=1`
    - After adding, verify: https://mxtoolbox.com/dmarc.aspx → enter schedulebox.cz

    **Step 7: DO NOT add SPF record for Brevo**
    - Brevo shared IPs do NOT require you to add `include:spf.brevo.com` to your SPF record
    - If you have an existing SPF record for Google Workspace or other services, leave it unchanged
    - Adding the Brevo SPF include will BREAK email authentication

    **Step 8: Verify in Brevo dashboard**
    - Return to Brevo: Senders, Domains, and Dedicated IPs → Domains
    - Click "Verify" next to schedulebox.cz
    - Both DKIM entries should show green checkmarks

    **Step 9: Configure SMTP credentials in your deployment**
    - For local development: add to `.env.local`:
      ```
      SMTP_HOST=smtp-relay.brevo.com
      SMTP_PORT=587
      SMTP_USER=your-brevo-email@example.com
      SMTP_PASS=xsmtpsib-your-smtp-key-here
      SMTP_FROM=info@schedulebox.cz
      ```
    - For Railway (production): set these as environment variables in Railway dashboard
    - For Kubernetes: `helm upgrade schedulebox . --set secrets.smtpHost=smtp-relay.brevo.com --set secrets.smtpUser=... --set secrets.smtpPass=...`
