    FIX-07 (Cookie consent with categories):

    Create `apps/web/lib/cookies/consent-store.ts`:
    - Define type `CookiePreferences = { necessary: true; analytics: boolean; marketing: boolean }`
    - `necessary` is always `true` (cannot be toggled off)
    - Export `getConsentPreferences(): CookiePreferences | null` — reads from localStorage key `sb_cookie_preferences` (JSON)
    - Export `setConsentPreferences(prefs: CookiePreferences): void` — saves to localStorage
    - Export `hasConsented(): boolean` — returns whether any preferences are stored
    - Export `isAnalyticsAllowed(): boolean` and `isMarketingAllowed(): boolean` — convenience helpers

    Rewrite `apps/web/app/[locale]/(marketing)/_components/cookie-consent-banner.tsx`:
    - Replace the simple accept/reject with a category-based consent UI
    - Show on first visit (no stored preferences)
    - Layout: description text at top, then three toggle rows:
      1. "Nezbytne cookies" / necessary — always on, disabled toggle (grayed out, checked)
      2. "Analyticke cookies" / analytics — toggle, default off
      3. "Marketingove cookies" / marketing — toggle, default off
    - Two buttons: "Prijmout vse" (sets all to true), "Ulozit vyber" (saves current toggle state)
    - Also keep "Odmitnout vse" as text link (sets analytics and marketing to false)
    - Use shadcn `Switch` component for toggles
    - After saving, banner does not reappear (check `hasConsented()`)
    - Add translations in `landing.json` (cs/en) under `landing.cookie`: `necessary`, `analytics`, `marketing`, `acceptAll`, `saveSelection`, `rejectAll`, `categoryDescription`

    FIX-11 (Offline banner):

    Create `apps/web/components/ui/offline-banner.tsx`:
    - Client component that uses `navigator.onLine` state + `online`/`offline` window events
    - When offline: show a fixed top banner (z-50, yellow/amber background) with text "Jste offline. Nektera funkcionalita nemusi byt dostupna." / "You are offline. Some features may be unavailable."
    - When back online: hide the banner (with 1s delay to avoid flicker)
    - Use `useEffect` for event listeners, clean up on unmount

    - Import and render `<OfflineBanner />` at the top of the providers tree (above other content)
    - This ensures it appears on ALL pages (dashboard, marketing, admin, etc.)

    FIX-10 (Web Component widget):

    Create `apps/web/public/schedulebox-widget.js` (vanilla JS, no build step needed):
    - Define a custom element `<schedulebox-widget>` using `class ScheduleBoxWidget extends HTMLElement`
    - Observed attributes: `slug` (required, company slug), `theme` (optional, "light"/"dark"), `locale` (optional, "cs"/"en"), `height` (optional, default "600px")
    - In `connectedCallback()`:
      1. Create a shadow DOM (`this.attachShadow({ mode: 'open' })`)
      2. Create an iframe inside the shadow DOM pointing to `${this.getAttribute('base-url') || 'https://app.schedulebox.cz'}/embed/${slug}?theme=${theme}&locale=${locale}&parent_origin=${encodeURIComponent(window.location.origin)}`
      3. Style the iframe: width 100%, height from attribute, border none, border-radius 12px
      4. Add a loading skeleton in the shadow DOM that hides when iframe loads
    - In `attributeChangedCallback()`: update iframe src when slug changes
    - Register: `customElements.define('schedulebox-widget', ScheduleBoxWidget)`
    - File must be self-contained (no imports, no bundling), servable as a static JS file
    - Add CORS header for the embed layout: update `apps/web/app/embed/[company_slug]/layout.tsx` to add `X-Frame-Options: ALLOWALL` or remove X-Frame-Options header for embed routes (they need to be iframeable)

    Usage example (for documentation):
    ```html
    <script src="https://app.schedulebox.cz/schedulebox-widget.js"></script>
    <schedulebox-widget slug="demo-salon" locale="cs" height="700px"></schedulebox-widget>
    ```
