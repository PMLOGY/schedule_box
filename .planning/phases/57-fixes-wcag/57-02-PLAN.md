    FIX-04 (Calendar resize):

    Create `apps/web/hooks/use-resize-booking.ts`:
    - Follow exact pattern of `use-reschedule-booking.ts`
    - `useResizeBooking()` returns a mutation that calls `apiClient.patch('/bookings/{id}/resize', { endTime })`
    - On success: invalidate `['bookings']` and calendar queries, show success toast
    - On error: show error toast

    Create `apps/web/app/api/v1/bookings/[id]/resize/route.ts`:
    - PATCH handler, requiresAuth, Zod body schema: `{ endTime: z.string().datetime() }`
    - Validate booking belongs to user's company (tenant isolation via findCompanyId)
    - Validate booking status is in ['pending', 'confirmed'] (same as draggable statuses)
    - Update booking's `endTime` in DB
    - Return updated booking

    Update `apps/web/components/booking/BookingCalendar.tsx`:
    - Import `useResizeBooking` hook
    - Add `resizeMutation` from the hook
    - Add `handleEventResize` callback (same shape as `handleEventDrop` but uses `end` date and calls resizeMutation)
    - Add `onEventResize={handleEventResize}` and `resizable` prop to `DnDCalendar`

    FIX-05 (Service reorder):

    Install @dnd-kit packages:
    ```
    pnpm --filter @schedulebox/web add @dnd-kit/core @dnd-kit/sortable @dnd-kit/utilities
    ```

    Create `apps/web/app/api/v1/services/reorder/route.ts`:
    - PATCH handler, requiresAuth
    - Body schema: `{ orderedIds: z.array(z.string().uuid()) }` — array of service UUIDs in desired order
    - For each UUID at index i, update `sort_order = i` in the services table
    - Validate all services belong to user's company
    - Wrap in a transaction for atomicity

    Create `apps/web/components/services/service-list-sortable.tsx`:
    - Use @dnd-kit `DndContext`, `SortableContext`, `useSortable`, `verticalListSortingStrategy`
    - Each service row is a `SortableItem` with a drag handle (GripVertical icon)
    - On `onDragEnd`: reorder the local array, then call `apiClient.patch('/services/reorder', { orderedIds })`
    - Show toast on success/error
    - Accept same props as the existing service list in the services page

    Update `apps/web/app/[locale]/(dashboard)/services/page.tsx`:
    - Replace the existing service list rendering with `ServiceListSortable`
    - Pass services data (already fetched) to the sortable component

    Add translation keys in `services.json` (cs/en): `reorder.success` / `reorder.error`
