    FIX-03 (Notification tracking): The notifications table already has `opened_at` and `clicked_at` timestamp columns.

    Create `apps/web/lib/notifications/tracking-utils.ts`:
    - Export `generateTrackingId(notificationId: number): string` — creates a signed token (HMAC-SHA256 with process.env.NEXTAUTH_SECRET) encoding the notification ID so tracking URLs cannot be forged
    - Export `verifyTrackingId(token: string): number | null` — verifies and returns notification ID or null
    - Export `buildTrackingPixelUrl(notificationId: number): string` — returns `/api/v1/notifications/track/pixel?t={token}`
    - Export `buildTrackingClickUrl(notificationId: number, targetUrl: string): string` — returns `/api/v1/notifications/{id}/track?t={token}&url={encodedUrl}`

    Create `apps/web/app/api/v1/notifications/track/pixel/route.ts` (public, no auth):
    - GET handler: extracts `t` query param, verifies tracking token, updates `opened_at = NOW()` on the notification record (only if `opened_at` is null — don't overwrite first open)
    - Returns a 1x1 transparent GIF (Buffer from hex: `47494638396101000100800000ffffff00000021f90401000000002c00000000010001000002024401003b`)
    - Content-Type: `image/gif`, Cache-Control: `no-store, no-cache`
    - On invalid token: still return the GIF (don't leak info)

    Create `apps/web/app/api/v1/notifications/[id]/track/route.ts` (public, no auth):
    - GET handler: extracts `t` and `url` query params
    - Verifies tracking token matches the `[id]` param
    - Updates `clicked_at = NOW()` on the notification record (only if `clicked_at` is null)
    - Redirects (302) to the `url` param (validate it starts with `http://` or `https://` to prevent open redirect)
    - On invalid token or missing url: redirect to homepage `/`

    These endpoints are intentionally public (no auth) because they are embedded in emails as tracking pixels and links.
