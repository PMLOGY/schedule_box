    A11Y-03 (Focus-visible indicators):

    Create `apps/web/styles/accessibility.css`:
    ```css
    /* WCAG 2.1 AA Focus Indicators */

    /* Global focus-visible ring for all focusable elements */
    *:focus-visible {
      outline: 2px solid hsl(var(--ring));
      outline-offset: 2px;
      border-radius: 2px;
    }

    /* Remove default outline for mouse users (focus without focus-visible) */
    *:focus:not(:focus-visible) {
      outline: none;
    }

    /* Skip-to-content link */
    .skip-to-content {
      position: absolute;
      left: -9999px;
      top: auto;
      width: 1px;
      height: 1px;
      overflow: hidden;
      z-index: 9999;
    }

    .skip-to-content:focus {
      position: fixed;
      top: 0;
      left: 0;
      width: auto;
      height: auto;
      padding: 1rem 2rem;
      background: hsl(221.2 83.2% 53.3%);
      color: white;
      font-size: 1rem;
      font-weight: 600;
      text-decoration: none;
      z-index: 9999;
    }

    /* Ensure buttons and links in shadcn components get visible focus */
    button:focus-visible,
    a:focus-visible,
    input:focus-visible,
    select:focus-visible,
    textarea:focus-visible,
    [role="button"]:focus-visible,
    [role="link"]:focus-visible,
    [role="tab"]:focus-visible,
    [role="menuitem"]:focus-visible,
    [tabindex]:focus-visible {
      outline: 2px solid hsl(var(--ring));
      outline-offset: 2px;
    }
    ```

    Import accessibility.css in `apps/web/app/globals.css` — add `@import '../styles/accessibility.css';` after the Tailwind imports.

    A11Y-04 (Color contrast):

    Audit the CSS custom properties in globals.css. The glassmorphism design tokens need checking. Fix any that fail WCAG AA:
    - `--muted-foreground` must have 4.5:1 against `--background` for normal text
    - `--card-foreground` must have 4.5:1 against `--card` background
    - If muted-foreground is too light (common in glass themes), darken it to at least `hsl(215 16% 40%)` or similar

    Check these specific elements that commonly fail contrast in glass themes:
    - Placeholder text in inputs (often too light)
    - Muted/secondary text colors
    - Disabled button text

    Add Tailwind utility overrides in accessibility.css if needed:
    ```css
    /* Contrast fixes for glass theme */
    .text-muted-foreground {
      color: hsl(var(--muted-foreground)) !important;
    }
    ```

    The goal is NOT to redesign the theme — just ensure the minimum contrast ratios are met by slightly darkening any failing text colors.

    Verify and fix keyboard navigation patterns:
