<objective>
Create the frontend pages for notification history, template management, and the visual automation rule builder using React Flow.

Purpose: Business owners need a UI to view sent notifications (delivery status tracking), manage email/SMS templates (customize content), and create automation rules visually (drag-drop trigger -> delay -> action). The visual builder is the key differentiator - it turns complex event-driven automation into an intuitive visual workflow.

Output: Six pages (notifications, templates list, template editor, automation rules list, visual builder, execution logs) plus four custom React Flow node components for the visual builder.
</objective>

**Custom React Flow Nodes:**

`apps/web/src/components/automation/TriggerNode.tsx`:
- Custom React Flow node component
- Displays trigger type selector (dropdown/select)
- Options: booking_created, booking_confirmed, booking_completed, booking_cancelled, booking_no_show, payment_received, customer_created, review_received
- Czech labels for each trigger type
- Visual: rounded rectangle with lightning bolt icon, blue-500 border, one output handle (bottom)
- Uses shadcn Select component inside the node

`apps/web/src/components/automation/DelayNode.tsx`:
- Custom React Flow node component
- Displays delay configuration: number input + unit selector (minutes/hours/days)
- Converts to total minutes internally
- Visual: rounded rectangle with clock icon, amber-500 border, input handle (top) + output handle (bottom)
- Default: 0 minutes (immediate)

`apps/web/src/components/automation/ActionNode.tsx`:
- Custom React Flow node component
- Displays action type selector: send_email, send_sms, send_push
- When action type selected, shows template picker (dropdown of templates fetched from /api/v1/notification-templates filtered by channel matching action type)
- Visual: rounded rectangle with mail/message icon, green-500 border, one input handle (top)
- Czech labels for action types

`apps/web/src/components/automation/NodePalette.tsx`:
- Side panel showing available node types to drag onto canvas
- Three items: "Trigger" (blue), "Delay" (amber), "Action" (green)
- Each item is draggable (uses React Flow's drag-to-create pattern)

`apps/web/app/[locale]/(dashboard)/automation/builder/page.tsx`:
- 'use client' directive
- Full-screen React Flow canvas with Controls and Background
- Top bar: rule name input, save button, back button
- Node palette on left side (NodePalette component)
- Register custom nodeTypes: { trigger: TriggerNode, delay: DelayNode, action: ActionNode }
- Default new rule starts with one trigger node and one action node connected by edge
- If editing (ruleId in query params): load existing rule from API, convert rule data to nodes/edges:
  - triggerType -> TriggerNode position {x:250, y:50}
  - If delayMinutes > 0 -> DelayNode position {x:250, y:200}
  - actionType + actionConfig -> ActionNode position {x:250, y:350 or y:200 if no delay}
  - Connect with animated edges
- Save handler: extract triggerType from trigger node, delayMinutes from delay node (or 0), actionType + actionConfig from action node. Validate: must have at least trigger + action. POST (create) or PUT (update) to `/api/v1/automation/rules`.
- Use ReactFlow from '@xyflow/react' with import of styles: `import '@xyflow/react/dist/style.css'`
