<objective>
Create RabbitMQ event consumers that listen to booking, payment, and review lifecycle events and enqueue notification delivery jobs to BullMQ across all three channels (email, SMS, push) with idempotent processing.

Purpose: This is the bridge between domain events and notification delivery. When a booking is created, the consumer fetches booking details, renders the confirmation template, and enqueues email, SMS, and push jobs as appropriate. When a booking is completed, it schedules a review request with a 2-hour delay. When a review comes in, it routes to external platforms (high rating) or internal feedback (low rating). Idempotency via BullMQ jobId prevents duplicate sends.

Output: Three RabbitMQ consumers (booking, payment, review) that process domain events and enqueue notification jobs to all three BullMQ queues (email, SMS, push). Worker entrypoint updated to start consumers alongside BullMQ workers.
</objective>

Create `services/notification-worker/src/consumers/booking-consumer.ts`:

1. `setupBookingConsumer(channel: amqp.Channel, queues: { emailQueue: Queue, smsQueue: Queue, pushQueue: Queue })`:
   - Assert durable queue `notification-worker.booking`
   - Bind to routing keys: `booking.created`, `booking.confirmed`, `booking.completed`, `booking.cancelled`, `booking.rescheduled`
   - Set prefetch to 10
   - Consume messages with try/catch, ACK on success, NACK+requeue on error

2. Event handler `handleBookingCreated(event, queues)`:
   - Parse CloudEvent<BookingCreatedPayload> from message
   - Fetch booking from DB with relations: `db.query.bookings.findFirst({ where: eq(bookings.uuid, event.data.bookingUuid), with: { customer: true, service: true, employee: true } })`
   - If no booking or no customer email, log warning and return (don't enqueue)
   - Fetch active notification template from DB: `notificationTemplates` where companyId + type='booking_confirmation' + channel='email' + isActive=true
   - If DB template exists, use it. If not, use default file template via renderTemplateFile('booking-confirmation', 'email', data)
   - Template data: `{ customer_name, service_name, booking_date: booking.startTime, booking_time: booking.startTime, employee_name (or 'nas tym'), price, currency, company_name, cancel_url: '{appUrl}/bookings/{uuid}/cancel' }`
   - Render template with renderTemplate() (template string from DB or file)
   - Render subject similarly (from DB template.subject or default "Potvrzeni rezervace - {{service_name}}")
   - Enqueue to emailQueue with: jobId = `confirm-${event.id}` (CRITICAL for idempotency), attempts: 3, backoff: { type: 'exponential', delay: 5000 }
   - Also check for SMS template: if customer has phone and SMS template exists, enqueue to smsQueue with jobId = `confirm-sms-${event.id}`
   - **PUSH NOTIFICATION:** Check if customer has a push subscription stored in user metadata (query users table joined via customer.userId, check metadata JSONB for pushSubscription field). If subscription exists, enqueue to pushQueue with: jobId = `confirm-push-${event.id}`, data: `{ companyId, customerId, subscription: user.metadata.pushSubscription, title: 'Rezervace potvrzena', body: '${service_name} - ${formatDate(booking.startTime)}', url: '{appUrl}/bookings/{booking.uuid}' }`

3. Event handler `handleBookingCompleted(event, queues)`:
   - Similar booking fetch
   - Enqueue review request email with 2-HOUR DELAY: `delay: 2 * 60 * 60 * 1000`
   - jobId = `review-request-${event.id}` for deduplication
   - Template: 'review-request' with review_url = `{appUrl}/reviews/submit/{booking.uuid}`

4. Event handler `handleBookingCancelled(event, queues)`:
   - Fetch booking + customer
   - Send cancellation notification email
   - jobId = `cancel-${event.id}`
   - Also enqueue push notification if subscription exists: jobId = `cancel-push-${event.id}`, title: 'Rezervace zrusena', body with service name

**Push subscription lookup helper** (define at top of each consumer file or extract to a shared utility):
```typescript
async function getCustomerPushSubscription(customerId: number): Promise<PushSubscription | null> {
  // Query: users table via customers.userId, check metadata JSONB for pushSubscription
  // Return null if no subscription found
  const customer = await db.query.customers.findFirst({
    where: eq(customers.id, customerId),
    columns: { userId: true }
  });
  if (!customer?.userId) return null;
  const user = await db.query.users.findFirst({
    where: eq(users.id, customer.userId),
    columns: { metadata: true }
  });
  return user?.metadata?.pushSubscription ?? null;
}
```
