<objective>
Create RabbitMQ event consumers that listen to booking, payment, and review lifecycle events and enqueue notification delivery jobs to BullMQ with idempotent processing.

Purpose: This is the bridge between domain events and notification delivery. When a booking is created, the consumer fetches booking details, renders the confirmation template, and enqueues an email job. When a booking is completed, it schedules a review request with a 2-hour delay. When a review comes in, it routes to external platforms (high rating) or internal feedback (low rating). Idempotency via BullMQ jobId prevents duplicate sends.

Output: Three RabbitMQ consumers (booking, payment, review) that process domain events and enqueue notification jobs. Worker entrypoint updated to start consumers alongside BullMQ workers.
</objective>

Create `services/notification-worker/src/consumers/booking-consumer.ts`:

1. `setupBookingConsumer(channel: amqp.Channel, emailQueue: Queue, smsQueue: Queue)`:
   - Assert durable queue `notification-worker.booking`
   - Bind to routing keys: `booking.created`, `booking.confirmed`, `booking.completed`, `booking.cancelled`, `booking.rescheduled`
   - Set prefetch to 10
   - Consume messages with try/catch, ACK on success, NACK+requeue on error

2. Event handler `handleBookingCreated(event, emailQueue, smsQueue)`:
   - Parse CloudEvent<BookingCreatedPayload> from message
   - Fetch booking from DB with relations: `db.query.bookings.findFirst({ where: eq(bookings.uuid, event.data.bookingUuid), with: { customer: true, service: true, employee: true } })`
   - If no booking or no customer email, log warning and return (don't enqueue)
   - Fetch active notification template from DB: `notificationTemplates` where companyId + type='booking_confirmation' + channel='email' + isActive=true
   - If DB template exists, use it. If not, use default file template via renderTemplateFile('booking-confirmation', 'email', data)
   - Template data: `{ customer_name, service_name, booking_date: booking.startTime, booking_time: booking.startTime, employee_name (or 'nas tym'), price, currency, company_name, cancel_url: '{appUrl}/bookings/{uuid}/cancel' }`
   - Render template with renderTemplate() (template string from DB or file)
   - Render subject similarly (from DB template.subject or default "Potvrzeni rezervace - {{service_name}}")
   - Enqueue to emailQueue with: jobId = `confirm-${event.id}` (CRITICAL for idempotency), attempts: 3, backoff: { type: 'exponential', delay: 5000 }
   - Also check for SMS template: if customer has phone and SMS template exists, enqueue to smsQueue with jobId = `confirm-sms-${event.id}`

3. Event handler `handleBookingCompleted(event, emailQueue)`:
   - Similar booking fetch
   - Enqueue review request email with 2-HOUR DELAY: `delay: 2 * 60 * 60 * 1000`
   - jobId = `review-request-${event.id}` for deduplication
   - Template: 'review-request' with review_url = `{appUrl}/reviews/submit/{booking.uuid}`

4. Event handler `handleBookingCancelled(event, emailQueue)`:
   - Fetch booking + customer
   - Send cancellation notification email
   - jobId = `cancel-${event.id}`

Create `services/notification-worker/src/consumers/payment-consumer.ts`:

1. `setupPaymentConsumer(channel: amqp.Channel, emailQueue: Queue)`:
   - Assert durable queue `notification-worker.payment`
   - Bind to: `payment.completed`, `payment.failed`
   - Prefetch 10

2. Event handler `handlePaymentCompleted(event, emailQueue)`:
   - Fetch payment with booking + customer relations
   - Render payment_confirmation template with amount, currency, gateway
   - Enqueue with jobId = `payment-confirm-${event.id}`

3. Event handler `handlePaymentFailed(event, emailQueue)`:
   - Fetch payment with customer
   - Render payment failure notification
   - Enqueue with jobId = `payment-failed-${event.id}`

IMPORTANT: Use the consumer helper from @schedulebox/events (createConsumerConnection, consumeMessages) if it provides the right abstraction, OR use amqplib directly on the channel parameter (since consumers receive channel from the orchestrator in index.ts). The channel is passed in, not created here.

All consumers must import CloudEvent types from @schedulebox/events for type-safe event parsing. Use JSON.parse(msg.content.toString()) to deserialize.
