    **security/zap/zap-baseline.yaml:**
    ZAP configuration for baseline web application scan:
    - Context: ScheduleBox, target URL from environment (staging.schedulebox.cz)
    - Include paths: `https://TARGET/.*` (scan all pages)
    - Exclude paths:
      - `/embed/.*` (widget embedding has different CSP requirements)
      - `/api/v1/webhooks/.*` (webhook endpoints expect specific payloads)
      - `/api/metrics` (internal monitoring endpoint)
    - Rules to IGNORE (false positives for JWT-based SaaS):
      - 10038: Content Security Policy Header Not Set (configured differently per route)
      - 10098: Cross-Domain JavaScript Source File Inclusion (CDN usage)
      - 10202: Absence of Anti-CSRF Tokens (JWT-based auth, not cookie sessions)
      - 40025: Proxy Disclosure (internal proxy headers normal in K8s)
    - Rules with lowered threshold (MEDIUM instead of HIGH):
      - 10037: Server Leaks Information via X-Powered-By (Next.js header, non-critical)
    - All other rules at default severity

    **security/zap/zap-api-scan.yaml:**
    ZAP configuration for API-specific scanning:
    - Target: API endpoint URL (staging/api/v1)
    - Rules to IGNORE for API context:
      - 10202: Anti-CSRF Tokens (APIs use Authorization header)
      - 90022: Application Error Disclosure (structured error responses are intentional)
    - Enable:
      - SQL Injection tests (critical for Drizzle ORM queries)
      - XSS tests (critical for user-generated content in reviews, templates)
      - Authentication bypass tests
      - IDOR tests (important for multi-tenant: cross-company data access)
    - API format: OpenAPI (if /api/v1/openapi.json exists) or import URL list

    **security/headers/security-headers.ts:**
    Export a `securityHeaders` array compatible with Next.js `headers()` config:

    ```typescript
    export const securityHeaders = [
      {
        key: 'X-DNS-Prefetch-Control',
        value: 'on',
      },
      {
        key: 'Strict-Transport-Security',
        value: 'max-age=63072000; includeSubDomains; preload', // 2 years HSTS
      },
      {
        key: 'X-Frame-Options',
        value: 'SAMEORIGIN', // Allow same-origin framing (needed for embed widget preview)
      },
      {
        key: 'X-Content-Type-Options',
        value: 'nosniff',
      },
      {
        key: 'Referrer-Policy',
        value: 'strict-origin-when-cross-origin',
      },
      {
        key: 'Permissions-Policy',
        value: 'camera=(), microphone=(self), geolocation=(), interest-cohort=()', // Microphone allowed for voice booking
      },
      {
        key: 'Content-Security-Policy',
        value: buildCSP(),
      },
    ];
    ```

    `buildCSP()` function constructs Content Security Policy:
    - `default-src 'self'`
    - `script-src 'self' 'unsafe-inline' 'unsafe-eval'` (Next.js requires unsafe-eval in dev, remove in production via NODE_ENV check)
    - `style-src 'self' 'unsafe-inline'` (Tailwind CSS injects inline styles)
    - `img-src 'self' data: blob: https:` (allow data URIs for base64 images, blob for generated content)
    - `font-src 'self'` (Inter font loaded locally)
    - `connect-src 'self' https://*.schedulebox.cz wss://*.schedulebox.cz` (API and WebSocket)
    - `frame-src 'self' https://pay.comgate.cz` (Comgate payment iframe)
    - `frame-ancestors 'self'` (prevent clickjacking except same origin)

    Add NODE_ENV conditional: in production, remove 'unsafe-eval' from script-src.

    **apps/web/next.config.js:**
    Read the existing next.config.js first. Add/update the `headers()` async function to apply security headers to all routes:

    ```javascript
    async headers() {
      return [
        {
          source: '/(.*)',
          headers: securityHeaders,
        },
        {
          // Relaxed CSP for embed widget routes
          source: '/embed/:path*',
          headers: [
            ...securityHeaders.filter(h => h.key !== 'Content-Security-Policy' && h.key !== 'X-Frame-Options'),
            { key: 'X-Frame-Options', value: 'ALLOWALL' }, // Widget must be embeddable
            { key: 'Content-Security-Policy', value: "frame-ancestors *" }, // Allow embedding from any domain
          ],
        },
      ];
    },
    ```

    Import securityHeaders from the security/headers module. If next.config.js uses CommonJS (require), convert the import accordingly or use dynamic import.

    NOTE: The security headers file lives in security/ directory (not packages/shared) because it's specific to the web deployment, not a shared library.
