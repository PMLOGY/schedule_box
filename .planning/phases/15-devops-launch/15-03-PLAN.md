  <files>
    packages/shared/src/telemetry/tracer.ts

  <action>
    Install OpenTelemetry packages in packages/shared:
    ```
    pnpm --filter @schedulebox/shared add @opentelemetry/sdk-node @opentelemetry/exporter-trace-otlp-grpc @opentelemetry/resources @opentelemetry/semantic-conventions @opentelemetry/instrumentation-http @opentelemetry/api
    ```

    Install winston for structured logging:
    ```
    pnpm --filter @schedulebox/shared add winston
    ```

    **packages/shared/src/telemetry/tracer.ts:**
    - `initTracer(serviceName: string)` function that creates and starts an OpenTelemetry NodeSDK
    - Resource attributes: SERVICE_NAME from param, SERVICE_VERSION from APP_VERSION env var or '1.0.0'
    - Trace exporter: OTLPTraceExporter pointing to `process.env.OTEL_EXPORTER_OTLP_ENDPOINT || 'http://localhost:4317'`
    - Instrumentations: HttpInstrumentation (auto-instruments all http/https requests)
    - Wrap in try/catch -- if OTEL endpoint is unreachable, log warning but do NOT crash the application (graceful degradation, matching project pattern from AI circuit breakers)
    - `shutdownTracer()` function for graceful shutdown (call sdk.shutdown())
    - Register SIGTERM handler to call shutdownTracer
    - Export `initTracer` and `shutdownTracer`

    **packages/shared/src/telemetry/index.ts:**
    Barrel export: re-export everything from ./tracer.

    **packages/shared/src/logger/index.ts:**
    Create winston-based structured logger:
    - Level from `process.env.LOG_LEVEL || 'info'`
    - Format: combine(timestamp(), errors({ stack: true }), json())
    - Default meta: `service` from `process.env.SERVICE_NAME || 'schedulebox'`
    - Single Console transport (stdout -- Docker/K8s will collect)
    - `log(level, message, meta)` function that:
      1. Gets active span from `trace.getSpan(context.active())`
      2. Extracts traceId and spanId from span context (if available)
      3. Calls winston logger with merged meta including trace_id, span_id
    - Export convenience functions: `logInfo(msg, meta?)`, `logError(msg, meta?)`, `logWarn(msg, meta?)`, `logDebug(msg, meta?)`
    - Export the winston logger instance directly as `logger` for advanced usage

    Update `packages/shared/src/index.ts` to add barrel exports:
    ```typescript
    export * from './telemetry/index.js';
    export * from './logger/index.js';
    ```

    NOTE: Do NOT integrate the tracer initialization into the app entry point in this plan. The tracer module is created here; wiring it into app startup is an incremental task. The /api/metrics endpoint from 15-02 and the tracer from this plan are independent observability layers.
  </action>

  <action>
    Create Helm values file for jaegertracing/jaeger chart deployment.

    **k8s/monitoring/jaeger-values.yaml:**
    - Deploy as all-in-one mode (suitable for MVP/staging; production would use separate collector/query/storage)
    - `allInOne.enabled: true`
    - Storage type: badger (embedded, no external Cassandra/ClickHouse needed)
    - Badger persistence: 20Gi PVC
    - Badger maintenance: purge traces older than 168h (7 days) per research recommendation
    - OTLP receiver enabled on port 4317 (gRPC) and 4318 (HTTP) -- this is what OpenTelemetry SDK connects to
    - Collector sampling: 100% in staging, 10% in production (add comment noting production sampling should be configured via env-specific override)
    - Ingress for Jaeger UI: enabled, path /jaeger, host from values
    - Resource limits: requests 250m/256Mi, limits 1000m/1Gi
    - Service type: ClusterIP
    - Query UI: enabled (Jaeger web interface for viewing traces)

    Add comments throughout explaining:
    - How to install: `helm install jaeger jaegertracing/jaeger --namespace monitoring --values k8s/monitoring/jaeger-values.yaml`
    - How to access UI: `kubectl port-forward svc/jaeger-query 16686:16686 -n monitoring`
    - How apps connect: set OTEL_EXPORTER_OTLP_ENDPOINT=http://jaeger-collector.monitoring:4317
    - When to upgrade from all-in-one to distributed: when trace volume exceeds 10k spans/second or need HA
  </action>
