1. Create `apps/web/app/[locale]/(marketing)/_components/cookie-consent-banner.tsx`:
   - This IS a 'use client' component (needs useState, useEffect, localStorage)
   - Import `useState`, `useEffect` from 'react'
   - Import `useTranslations` from 'next-intl' (client hook)
   - Import `Button` from `@/components/ui/button`
   - Import `Link` from `@/lib/i18n/navigation`

   Constants:
   ```typescript
   const CONSENT_KEY = 'sb_cookie_consent';
   type ConsentState = 'accepted' | 'rejected' | null;
   ```

   Component logic:
   - `const [consent, setConsent] = useState<ConsentState>(null)`
   - `const [mounted, setMounted] = useState(false)` — CRITICAL: prevents hydration mismatch
   - `const t = useTranslations('landing.cookie')`
   - useEffect on mount: `setMounted(true); const stored = localStorage.getItem(CONSENT_KEY) as ConsentState; setConsent(stored);`
   - `handleAccept`: `localStorage.setItem(CONSENT_KEY, 'accepted'); setConsent('accepted');`
   - `handleReject`: `localStorage.setItem(CONSENT_KEY, 'rejected'); setConsent('rejected');`
   - Return `null` if `!mounted || consent !== null` — banner only shows when mounted and no prior choice

   Render:
   - `<div role="dialog" aria-label="Souhlas s cookies" className="fixed bottom-0 left-0 right-0 z-50 border-t bg-white p-4 shadow-lg">`
   - Inner container: `<div className="mx-auto max-w-4xl flex flex-col gap-4 sm:flex-row sm:items-center sm:justify-between">`
   - Text: `<p className="text-sm text-gray-700">{t('message')} <Link href="/privacy" className="underline">{t('privacyLink')}</Link></p>`
   - Button row: `<div className="flex gap-3 flex-shrink-0">`
     - Reject button: `<Button variant="outline" size="sm" onClick={handleReject}>{t('reject')}</Button>`
     - Accept button: `<Button size="sm" onClick={handleAccept}>{t('accept')}</Button>`
   - CRITICAL COMPLIANCE: Both buttons must be similar size and prominence. The outline vs default variant gives slight visual difference but both are clearly clickable. Do NOT make reject button smaller, grayed out, or less prominent.

   Czech ECA 2022 compliance checklist (enforce all):
   - [x] No pre-checked boxes (no checkboxes at all in simple accept/reject)
   - [x] Equal-weight Accept and Reject buttons
   - [x] Closing/dismissing without choosing is NOT consent (no X close button)
   - [x] Banner does NOT block page content (fixed bottom overlay, content scrollable behind it)
   - [x] Link to privacy policy is present

2. Update `apps/web/app/[locale]/(marketing)/layout.tsx`:
   - Read the existing layout.tsx file first
   - Add import: `import { CookieConsentBanner } from './_components/cookie-consent-banner';`
   - Add `<CookieConsentBanner />` inside the layout div, AFTER `<MarketingFooter />`
   - The layout should now render: `<MarketingNavbar />`, `<main>`, `<MarketingFooter />`, `<CookieConsentBanner />`
   - Do NOT change anything else in the layout

   **PATH CLARIFICATION:** The research architecture diagram references `(marketing)/cs/privacy` and similar `cs/`-prefixed paths. IGNORE these — the actual file structure uses `(marketing)/privacy/page.tsx` (no `cs/` prefix). The `[locale]` dynamic segment and `localePrefix: 'as-needed'` routing config handle locale prefixing at the URL level automatically. When the banner links to `/privacy`, the i18n `Link` component handles locale-aware routing correctly.

IMPORTANT: The mounted guard pattern is essential. Without it, the server renders the banner (consent is null on server), but the client immediately reads localStorage and may hide it, causing a React hydration mismatch error. The mounted flag ensures the banner is NEVER rendered on the server.

IMPORTANT: Do NOT add a close (X) button. Under Czech law, dismissing without explicit choice is NOT valid consent. The user MUST click Accept or Reject.
