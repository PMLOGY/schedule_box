1. Update `apps/web/app/api/v1/payments/comgate/create/route.ts`:
   - Import `resolveComgateCredentials` from `@/lib/payment-provider/resolve`
   - After getting `companyId`, call `const creds = await resolveComgateCredentials(companyId)`
   - Pass `{ merchantId: creds.merchantId, secret: creds.secret, testMode: creds.testMode }` as the second argument to `initComgatePayment()`
   - This ensures booking payments use the business's own Comgate credentials when configured, falling back to platform otherwise (PAY-02)
   - Add a log line: `console.log(\`[Payment Create] Using ${creds.source} Comgate credentials for company ${companyId}\`)`

2. Update `apps/web/app/api/v1/webhooks/comgate/route.ts`:
   - The webhook handler receives payment notifications. It needs to verify the secret against the correct merchant.
   - After finding the payment record (step 6 in current code, `findPaymentByGatewayTx`), use the payment's `companyId` to resolve credentials: `const creds = await resolveComgateCredentials(payment.companyId)`
   - Replace the current `verifyComgateWebhookSecret(receivedSecret)` call — move it AFTER the payment lookup, and pass the resolved secret: `verifyComgateWebhookSecret(receivedSecret, creds.secret)`
   - IMPORTANT: The verification currently happens at step 3 (before payment lookup). We need to restructure:
     a. Parse body (step 2) — keep as-is
     b. Extract transId — keep as-is
     c. Idempotency check — keep as-is
     d. Find payment by gateway tx — keep as-is
     e. NEW: resolve credentials using payment.companyId
     f. Verify secret using resolved credentials (moved from step 3)
     g. If payment not found (unknown transId), fall back to platform secret for verification
   - Also pass credentials to `getComgatePaymentStatus(transId, { merchantId: creds.merchantId, secret: creds.secret })` in the defense-in-depth check

3. **DO NOT modify** `apps/web/app/api/v1/billing/subscribe/route.ts` or `apps/web/app/api/v1/billing/service.ts` — these use platform Comgate credentials for subscription billing (PAY-04). They call `initComgatePayment()` without the credentials parameter, so they automatically use the env var fallback. Add a comment at the top of billing/subscribe/route.ts: `// NOTE: Subscription billing intentionally uses platform Comgate credentials (PAY-04). Do NOT pass per-company credentials here.`
