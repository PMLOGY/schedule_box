<objective>
Create the provider-agnostic payment_providers DB table, build the API for managing per-company Comgate credentials (encrypted at rest), and refactor the Comgate client to accept credential overrides so booking payments can route through the business's own merchant account.

Purpose: PAY-03 requires a provider-agnostic schema. PAY-04 requires that platform subscription billing still uses the platform account. This plan builds the foundation — schema, credential storage, and credential resolution logic — that Plan 02 will wire into the UI and booking payment flow.

Output: payment_providers table, settings API, credential resolver, updated Comgate client
</objective>

1. Create `apps/web/lib/payment-provider/resolve.ts`:

   ```typescript
   export interface ComgateCredentials {
     merchantId: string;
     secret: string;
     testMode: boolean;
     source: 'company' | 'platform';
   }
   ```

   Export `async function resolveComgateCredentials(companyId: number): Promise<ComgateCredentials>`:
   - Query `payment_providers` WHERE `company_id = companyId AND provider = 'comgate' AND is_active = true`
   - If found: decrypt credentials using `decrypt()` + `getEncryptionKey()`, parse JSON `{ merchantId, secret }`, return with `source: 'company'` and the row's `test_mode`
   - If NOT found (or is_active=false): fall back to platform env vars `COMGATE_MERCHANT_ID` / `COMGATE_SECRET` / `COMGATE_TEST_MODE`, return with `source: 'platform'`
   - If neither exists, throw AppError('PAYMENT_GATEWAY_ERROR', 'No Comgate credentials configured', 500)

   Export `function getPlatformComgateCredentials(): ComgateCredentials`:
   - Reads from env vars only (COMGATE_MERCHANT_ID, COMGATE_SECRET, COMGATE_TEST_MODE)
   - Returns with `source: 'platform'`
   - Used by subscription billing to explicitly bypass per-company credentials (PAY-04)

2. Update `apps/web/app/api/v1/payments/comgate/client.ts`:

   Add an optional `credentials` parameter to the key functions. The current `getComgateCredentials()` remains as internal fallback.

   - Change `initComgatePayment` signature to accept an optional `credentials?: { merchantId: string; secret: string; testMode?: boolean }` parameter. If provided, use those credentials instead of calling `getComgateCredentials()`. If `testMode` is provided, use it instead of `COMGATE_TEST_MODE` env var.
   - Change `getComgatePaymentStatus` to accept optional `credentials?: { merchantId: string; secret: string }`.
   - Change `refundComgatePayment` to accept optional `credentials?: { merchantId: string; secret: string }`.
   - `verifyComgateWebhookSecret` — add optional `secret?: string` parameter. If provided, compare against it instead of env var. This is needed because webhook verification must use the merchant's secret, not the platform's.
   - Keep `getComgateCredentials()` as the fallback (backward compatible — existing callers without credentials param still work).

   **IMPORTANT**: Do NOT change the `chargeRecurringPayment` function — it is used only for platform subscription billing and MUST always use platform credentials (PAY-04 separation). If anything, add a comment documenting this intentional design.
