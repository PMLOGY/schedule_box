<objective>
Create the 4-step business setup wizard that guides new business owners from registration to sharing their live booking link. The wizard calls existing v1.0 APIs (company update, service creation, working hours) and culminates in a "Your booking link is ready" moment with QR code and copy-to-clipboard.

Purpose: This is the primary aha moment for new owners. Without a guided wizard, cold-start churn exceeds 70% within 48 hours (research). The wizard reduces time-to-value from "figure it out yourself" to under 5 minutes.

Output: Onboarding wizard route + 4 step components + Zustand store + QR code generation + i18n messages
</objective>

    First, install the qrcode package for QR code generation:
    ```
    cd apps/web && pnpm add qrcode && pnpm add -D @types/qrcode
    ```

    Create `apps/web/stores/onboarding-wizard.store.ts`:
    - Follow the pattern from `booking-wizard.store.ts` (same Zustand create pattern)
    - Define `OnboardingStep` as `1 | 2 | 3 | 4`
    - Step 1 data: `companyName`, `phone`, `description`, `addressStreet`, `addressCity`, `addressZip`, `industryType` (string, one of the industry_type CHECK constraint values), `logoUrl` (optional)
    - Step 2 data: `serviceName`, `serviceDuration` (number, minutes), `servicePrice` (number, CZK), `serviceDescription` (optional)
    - Step 3 data: `workingHours` (array of `{ dayOfWeek: number, startTime: string, endTime: string, isActive: boolean }`)
    - Step 4 data: `bookingUrl` (string), `companySlug` (string)
    - Store state: `step`, `data`, `isSubmitting`, `error`, `completedSteps` (Set<number> tracking which steps are done)
    - Actions: `setStep`, `nextStep`, `prevStep`, `updateData`, `setSubmitting`, `setError`, `markStepCompleted`, `reset`
    - Export `useOnboardingWizard`

    Create `apps/web/validations/onboarding.ts`:
    - `companyDetailsSchema`: z.object with name (min 2, max 255), phone (optional), description (optional, max 500), address_street (optional), address_city (optional), address_zip (optional, regex /^\d{3}\s?\d{2}$/ for Czech postal codes), industry_type (z.enum of industry types from the DB CHECK constraint)
    - `firstServiceSchema`: z.object with name (min 1, max 255), duration_minutes (number, min 5, max 480), price (number, min 0), description (optional, max 500)
    - `workingHoursSchema`: z.array of { day_of_week (0-6), start_time (HH:mm regex), end_time (HH:mm regex), is_active (boolean) }

    Create `apps/web/hooks/use-onboarding.ts`:
    - Export `useOnboardingRedirect()` hook
    - Uses `use-settings-query.ts` pattern to fetch company settings via `/api/v1/settings/company`
    - If `onboarding_completed === false`, returns `{ shouldRedirect: true, isLoading: false }`
    - If `onboarding_completed === true`, returns `{ shouldRedirect: false, isLoading: false }`
    - While loading, returns `{ shouldRedirect: false, isLoading: true }`
    - This hook will be used by the dashboard page to conditionally redirect to `/onboarding`
  </action>

    - `qrcode` and `@types/qrcode` are in package.json dependencies
    - All three files export their named functions
