<objective>
Create the onboarding checklist dashboard widget that tracks 5 setup items, and add action-oriented empty states to all dashboard sections that display data lists. The checklist motivates completion; empty states guide new users toward their first actions.

Purpose: Checklists increase onboarding completion by 21% when the first action shows "25% done" (ProductLed research). Empty states replace confusing blank screens with clear next actions, reducing cold-start abandonment.

Output: Checklist widget component + checklist hook + 6 section-specific empty states + dashboard integration + i18n messages
</objective>

    Create `apps/web/hooks/use-onboarding-checklist.ts`:
    - Export `useOnboardingChecklist()` hook
    - Uses React Query (TanStack Query, same pattern as existing hooks like `use-settings-query.ts`) to fetch data needed to determine completion of 5 items
    - The 5 checklist items and their completion criteria:
      1. **Company profile complete** — fetch `/api/v1/settings/company`, check that `name` is non-empty AND `phone` OR `description` is non-empty (not just the auto-created name from registration)
      2. **First service added** — fetch `/api/v1/services`, check that at least 1 service exists
      3. **Working hours set** — fetch `/api/v1/settings/working-hours`, check that at least 1 working hour entry exists
      4. **First booking received** — fetch `/api/v1/bookings?limit=1`, check that at least 1 booking exists
      5. **Payment or SMS enabled** — fetch `/api/v1/settings/company`, check `settings` JSONB or `features_enabled` for payment/notification config (simplified: check if company has a phone set for SMS readiness)
    - Return: `{ items: ChecklistItem[], completedCount: number, totalCount: 5, isAllComplete: boolean, isLoading: boolean }`
    - Each `ChecklistItem`: `{ id: string, label: string, description: string, completed: boolean, href: string }`
    - The `href` links to the relevant settings/setup page for incomplete items
    - Use `useTranslations('onboarding.checklist')` for item labels

    Create `apps/web/components/onboarding/onboarding-checklist.tsx`:
    - 'use client' directive
    - Import `useOnboardingChecklist` hook

    - Show a progress bar (use shadcn Progress component): `completedCount / 5 * 100`%
    - Show percentage text: "3 of 5 complete"
    - List all 5 items as rows with:
      - Checkbox-style indicator (green check circle if complete, empty circle if not)
      - Item label (bold) + description (muted text)
      - If incomplete, the entire row is a link to `href` for that item
    - When `isAllComplete === true`, show a "Dismiss" button (X icon in top-right corner)
    - Dismissal: store `onboarding_checklist_dismissed` in localStorage (key: `sb_checklist_dismissed_{companyUuid}`)
    - If dismissed, do not render the checklist at all
    - While loading, show skeleton (2 lines)
    - If `onboarding_completed === false` (fresh registration, wizard not done), do not show the checklist — the wizard takes priority. Show checklist only AFTER wizard completion.
    - Use `useTranslations('onboarding.checklist')` for all text
