  <action>
Modify `apps/web/app/[locale]/(dashboard)/page.tsx` to wire the existing `useOnboardingRedirect` hook and render the welcome banner when onboarding is not completed.

**Imports to add:**
- `useEffect` from `'react'`
- `useRouter` from `'@/lib/i18n/navigation'` (already used in adjacent files, e.g. notifications/page.tsx)
- `useOnboardingRedirect` from `'@/hooks/use-onboarding'`
- `useTranslations` is already imported — add `'onboarding'` namespace OR use a second `useTranslations('onboarding')` call
- `Button` from `'@/components/ui/button'`
- `Card, CardContent` from `'@/components/ui/card'`
- `Link` from `'@/lib/i18n/navigation'` (for the CTA link to /onboarding)

**Logic:**
1. Call `const { shouldRedirect, isLoading } = useOnboardingRedirect();`
2. Call `const router = useRouter();`
3. Add `const tOnboarding = useTranslations('onboarding');`
4. Add a `useEffect` that calls `router.replace('/onboarding' as any)` when `shouldRedirect === true` and `!isLoading`. Use `router.replace` (not `push`) so the user cannot navigate back to dashboard mid-wizard.
5. If `isLoading`, render `null` (prevents flash of dashboard content before redirect status is known).
6. If `shouldRedirect` is `true`, render the welcome banner card ONLY (not the full dashboard), using `onboarding.welcomeBanner` keys:
   - `tOnboarding('welcomeBanner.title')` — card title
   - `tOnboarding('welcomeBanner.description')` — card description text
   - `tOnboarding('welcomeBanner.action')` — CTA button label, linking to `/onboarding`

**Welcome banner structure (Tailwind + shadcn/ui):**
```tsx
<div className="flex min-h-[60vh] items-center justify-center">
  <Card className="max-w-md w-full text-center">
    <CardContent className="pt-8 pb-8 space-y-4">
      <h1 className="text-2xl font-bold">{tOnboarding('welcomeBanner.title')}</h1>
      <p className="text-muted-foreground">{tOnboarding('welcomeBanner.description')}</p>
      <Button asChild>
        <Link href="/onboarding">{tOnboarding('welcomeBanner.action')}</Link>
      </Button>
    </CardContent>
  </Card>
</div>
```

7. Otherwise (onboarding completed), render the existing dashboard JSX unchanged.

**Do NOT remove existing imports** — PageHeader, DashboardGrid, QuickActions, AiInsightsPanel, OnboardingChecklist, DemoDataCard must all remain for the completed state.

**Branching logic summary:**
```
if isLoading → return null
if shouldRedirect → useEffect triggers router.replace('/onboarding'), render welcome banner
else → render existing dashboard JSX
```
