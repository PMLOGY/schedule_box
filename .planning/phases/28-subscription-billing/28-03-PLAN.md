  <action>
    1. Create `services/notification-worker/src/schedulers/billing-scheduler.ts`:

    Follow the pattern from `reminder-scheduler.ts` but use the newer `upsertJobScheduler` API (BullMQ 5.16+) instead of `Queue.add(..., { repeat })` as recommended by research.

    **Constants:**
    ```typescript
    const BILLING_QUEUE_NAME = 'subscription-billing';
    const DUNNING_GRACE_DAYS = 14;
    ```

    **Main function: `startBillingScheduler(emailQueue: Queue, redisConnection: ConnectionOptions)`**
    - Create billing queue
    - Use `upsertJobScheduler` to schedule daily scan at 06:00 UTC:
      ```typescript
      await billingQueue.upsertJobScheduler(
        'daily-renewal-scanner',
        { pattern: '0 6 * * *' },
        {
          name: 'scan-renewals',
          data: {},
          opts: { attempts: 3, backoff: { type: 'exponential', delay: 60000 }, removeOnFail: 100 },
        },
      );
      ```
    - Create Worker that processes the scan job by calling `processRenewals(emailQueue)`
    - Worker concurrency: 1 (only one scanner at a time)
    - Add event listeners for completed/failed (console.log pattern from reminder-scheduler)
    - Return `{ queue, worker }` for graceful shutdown

    **processRenewals(emailQueue: Queue):**
    - Query subscriptions WHERE status = 'active' AND currentPeriodEnd <= now()
    - For each subscription:
      a. Load company data (email, name, currency, addressCountry)
      b. Check if subscription has cancelAtPeriodEnd=true -- if so, handle pending downgrade:
         - If pending downgrade plan is in metadata, update subscription plan to downgrade plan
         - If downgrade to 'free', transition to 'expired' and set companies.subscriptionPlan='free'
         - Log subscription_event 'plan.downgraded'
         - Skip payment
      c. If NOT cancelling: call `chargeRecurringPayment` from Comgate client:
         ```typescript
         const result = await chargeRecurringPayment({
           initRecurringId: subscription.comgateInitTransactionId,
           price: planPrice, // from PLAN_CONFIG
           currency: subscription.currency || 'CZK',
           label: `ScheduleBox ${planName} - ${format(new Date(), 'yyyy-MM')}`,
           refId: `${subscription.uuid}-${format(new Date(), 'yyyy-MM')}`,
           email: companyEmail,
         });
         ```
      d. If result.code === '0' (success):
         - Extend currentPeriodStart = old currentPeriodEnd
         - Extend currentPeriodEnd = currentPeriodStart + 30 days (monthly) or + 365 days (annual)
         - Reset dunningStartedAt = null
         - Update companies.subscriptionValidUntil = new currentPeriodEnd
         - Log subscription_event 'payment.success'
         - Create subscription_invoice record (next plan picks this up for PDF generation)
      e. If result.code !== '0' (failure):
         - If currently 'active': transition to 'past_due', set dunningStartedAt = now
         - Log subscription_event 'payment.failed' with { comgateCode: result.code, message: result.message }
         - Enqueue dunning email

    **processDunning(emailQueue: Queue):**
    - Query subscriptions WHERE status = 'past_due'
    - For each:
      a. Calculate days since dunningStartedAt
      b. If >= DUNNING_GRACE_DAYS (14):
         - Transition to 'expired'
         - Update companies.subscriptionPlan = 'free'
         - Update companies.subscriptionValidUntil = null
         - Log subscription_event 'subscription.expired'
         - Enqueue final expiration email
      c. If dunningStartedAt + 7 days <= now AND no 'dunning.warning_7d' event exists:
         - Enqueue 7-day warning dunning email
         - Log subscription_event 'dunning.warning_7d'

    **Call both processRenewals and processDunning from the worker handler.**

    **IMPORTANT:** The notification-worker is a separate service. It connects to the same database. Import chargeRecurringPayment by direct path import or create a shared billing utility. Since the Comgate client is in the web app, the simplest approach is to duplicate the `chargeRecurringPayment` function in the worker OR extract it to a shared location. The cleanest approach: create `packages/shared/src/billing/comgate-recurring.ts` with the recurring charge function, and import from both places. However, the existing Comgate client uses `fetchWithTimeout` and `getComgateCredentials` which are local to the web app.

    Pragmatic approach: Inline the chargeRecurringPayment logic in billing-scheduler.ts using fetch + URLSearchParams directly (same pattern but self-contained). Read COMGATE_MERCHANT_ID, COMGATE_SECRET, COMGATE_API_URL from environment variables. This avoids cross-package coupling between the web app and the worker.

    2. Update `services/notification-worker/src/schedulers/index.ts`:
    - Import `startBillingScheduler` from `./billing-scheduler.js`
    - Add billingQueue and billingWorker to SchedulerResources interface
    - Call `startBillingScheduler(queues.emailQueue, redisConnection)` in `startSchedulers`
    - Return billing resources alongside existing reminder resources

    3. Update `services/notification-worker/src/index.ts`:
    - In the `shutdown` function, close billingWorker and billingQueue (via schedulerResources)
    - The schedulerResources type already extends, so just ensure shutdown handles the new fields
