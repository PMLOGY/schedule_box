  <name>Task 1: Create subscription invoice service with SEQUENCE-based numbering and PDF generation</name>
  <files>apps/web/app/api/v1/billing/invoice-service.ts</files>
  <action>
    Create `apps/web/app/api/v1/billing/invoice-service.ts` following the pattern from `apps/web/app/api/v1/invoices/generate.ts` but adapted for subscription invoices.

    **Function 1: `generateSubscriptionInvoiceNumber(tx: Database): Promise<string>`**
    - Use PostgreSQL SEQUENCE (NOT MAX+1 pattern -- race condition under concurrent renewals):
      ```typescript
      const [result] = await tx.execute(sql`SELECT nextval('subscription_invoice_seq') as num`);
      const seqNum = String(result.num).padStart(6, '0');
      const year = new Date().getFullYear();
      return `SB-${year}-${seqNum}`;
      ```
    - Format: SB-YYYY-NNNNNN (e.g., SB-2026-000001)
    - Globally unique (not per-company) since SEQUENCE is global

    **Function 2: `createSubscriptionInvoice(params: CreateSubInvoiceParams, tx?: Database): Promise<InvoiceRecord>`**
    - Params: subscriptionId, companyId, amount, currency, comgateTransactionId, period
    - Get company data for sellerSnapshot (name, ICO, DIC, address -- from companies.settings)
    - Determine VAT rate using `getVatRate(company.addressCountry)` from shared/types/billing
    - Calculate taxAmount = amount * vatRate / 100
    - Generate invoice number using `generateSubscriptionInvoiceNumber`
    - Create sellerSnapshot JSON with company details frozen at invoice time:
      ```typescript
      const sellerSnapshot = {
        companyName: company.name,
        ico: settings.ico || '',
        dic: settings.dic || '',
        addressStreet: company.addressStreet || '',
        addressCity: company.addressCity || '',
        addressZip: company.addressZip || '',
        addressCountry: company.addressCountry || 'CZ',
      };
      ```
    - Insert into subscription_invoices table with status='paid', paidAt=now
    - Return the invoice record

    **Function 3: `generateSubscriptionInvoicePDF(invoiceId: number, companyId: number): Promise<Buffer>`**
    - Follow the EXACT layout from `apps/web/app/api/v1/invoices/generate.ts` but query from subscription_invoices table
    - Query invoice with subscription and company data:
      ```
      subscription_invoices
        JOIN subscriptions ON subscriptionId
        JOIN companies ON companyId
      ```
    - Use PDFKit (already installed, in serverExternalPackages)
    - Czech invoice layout (A4 page):
      a. **Header:** Company name, ICO, DIC, address (from sellerSnapshot, NOT current company data)
      b. **Title:** "FAKTURA" centered, invoice number below
      c. **Buyer section:** "Odberatel" -- ScheduleBox s.r.o. (or the platform billing entity)
         Actually, for SaaS subscription: the SELLER is ScheduleBox (platform), the BUYER is the company.
         Wait -- re-read: the company is the CUSTOMER paying ScheduleBox for the subscription.
         So: Seller = ScheduleBox platform entity, Buyer = the company subscribing.
         BUT the existing booking invoice has the company as seller and customer as buyer.
         For subscription invoices, the roles are reversed:
         - Seller (Dodavatel): ScheduleBox s.r.o. (platform) -- hardcode or use env vars
         - Buyer (Odberatel): The company subscribing -- use sellerSnapshot data
         Define ScheduleBox platform billing details as constants or env vars:
         ```typescript
         const PLATFORM_SELLER = {
           name: process.env.PLATFORM_COMPANY_NAME || 'ScheduleBox s.r.o.',
           ico: process.env.PLATFORM_ICO || '',
           dic: process.env.PLATFORM_DIC || '',
           address: process.env.PLATFORM_ADDRESS || '',
         };
         ```
      d. **Invoice details:** Datum vystaveni, Datum splatnosti, Variabilni symbol
      e. **Line item:** Plan name, 1x, price excl. VAT, VAT rate%, price incl. VAT
      f. **Totals:** Zaklad dane, DPH XX%, Celkem
      g. **Payment info:** Comgate platebni brana, transaction ID
      h. **Footer:** "Vystaveno v systemu ScheduleBox"

    - IMPORTANT: Use VAT rate from the invoice record (vatRate column), NOT hardcoded 21%
    - IMPORTANT: Use sellerSnapshot for buyer details (frozen at invoice time per Czech law)

    **Function 4: `getSubscriptionInvoicesForCompany(companyId: number): Promise<InvoiceRecord[]>`**
    - Query subscription_invoices WHERE companyId = companyId, ordered by createdAt DESC
    - Return list with id, uuid, invoiceNumber, amount, taxAmount, vatRate, currency, status, period, paidAt, pdfUrl, createdAt
  </action>

    - Invoice number format is SB-YYYY-NNNNNN using SEQUENCE
    - PDF generation uses PDFKit with Czech layout matching existing invoice pattern
    - VAT rate comes from invoice record, not hardcoded
