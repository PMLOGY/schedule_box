    export function useCurrentSubscription()  // GET /api/v1/billing/subscription
    export function useBillingInvoices()       // GET /api/v1/billing/invoices
    export function useBillingStatus()         // GET /api/v1/billing/status (polling)

    // Mutation hooks:
    export function useSubscribe()   // POST /api/v1/billing/subscribe -> returns { redirectUrl }

    export function useDowngrade()   // POST /api/v1/billing/downgrade

    For subscribe mutation, on success: redirect to `redirectUrl` using `window.location.href` (Comgate payment page).

    For upgrade mutation, on success: **check the `charged` field in the response:**
    - If `result.charged === true`: proration was charged server-side, no redirect needed. Show success toast and invalidate subscription query to refetch updated plan.
    - If `result.redirectUrl`: redirect to Comgate using `window.location.href` (only happens when subscriber has no existing recurring token, e.g., upgrading during trial).

<task type="auto">
  <name>Task 2: Build billing portal page with plan comparison, subscription management, and invoice history</name>
  <files>apps/web/app/[locale]/(dashboard)/settings/billing/page.tsx</files>
  <action>
    Create `apps/web/app/[locale]/(dashboard)/settings/billing/page.tsx` as a client component.

    Use the existing UI components from the codebase: Card, CardHeader, CardTitle, CardDescription, CardContent, Badge, Button, PageHeader, and any others from `@/components/ui/`. Follow the layout pattern from the settings page (`settings/page.tsx`).

    **Page structure (3 sections):**

    **Section 1: Current Subscription Card**
    - Show current plan name with Badge (color-coded by plan)
    - Status badge (active=green, past_due=amber, cancelled=red, trialing=blue)
    - Next billing date (from subscription.currentPeriodEnd)
    - Monthly price
    - If past_due: warning banner "Vaše platba se nezdařila. Aktualizujte platební údaje."
    - If no subscription: show "Bezplatný plán" with "Upgradovat" button
    - Cancel subscription button (opens confirmation dialog)

    **Section 2: Plan Comparison Grid**
    - 4 columns (Free, Essential, Growth, AI-Powered)
    - Use data from `useBillingPlans()` hook (PLAN_CONFIG)
    - Each column shows: plan name, price/month, key features (bookings limit, employees limit, services limit, AI features yes/no)
    - Current plan column highlighted with "Aktualni" badge
    - Each non-current plan has action button:
      - Higher tier: "Upgradovat" (primary button)
      - Lower tier: "Snizit plan" (outline button)
      - Free: no action needed
    - On mobile: stack cards vertically (responsive grid: `grid-cols-1 md:grid-cols-2 lg:grid-cols-4`)

    **Section 3: Invoice History Table**
    - Use data from `useBillingInvoices()` hook
    - Table columns: Invoice number, Period, Amount + currency, Status badge (paid/issued/failed), Download PDF button
    - PDF download: link to `/api/v1/billing/invoices/{id}/pdf` (opens in new tab)
    - Empty state: "Zatim nemate zadne faktury"
    - Pagination if >10 invoices

    **Payment return flow:**
    - Check URL search params for `?payment=pending`
    - If present: show a loading overlay "Cekame na potvrzeni platby..." with spinner
    - Enable `useBillingStatus()` polling (every 2 seconds)
    - When status returns `activated: true`, show success toast, remove `?payment=pending` from URL, refetch subscription data
    - Timeout after 15 seconds: show message "Platba se zpracovava, zkuste obnovit stranku za chvili"

    **Upgrade flow (handles both server-side charge and redirect):**
    - When user clicks "Upgradovat" on a higher tier plan:
      a. Call useUpgrade mutation with the target plan
      b. If response has `charged: true`: show success toast with "Plan byl uspesne upgradovan. Pomerny doplatek byl automaticky strzen." and refetch subscription data. No redirect needed.
      c. If response has `redirectUrl`: redirect to Comgate using `window.location.href`. This only happens for subscribers without an existing recurring token (rare edge case, e.g., upgrading during trial).
    - Show loading state on the upgrade button during mutation

    **Downgrade flow:**
    - Show confirmation dialog explaining "Snizeni planu bude provedeno na konci aktualniho obdobi". On confirm, call useDowngrade mutation, show toast.

    **Styling:**
    - Use Tailwind CSS classes consistent with the settings page
    - Plan comparison cards should have subtle borders, the current plan card should have a primary-colored border
    - Use `text-muted-foreground` for secondary text
    - Loading states: show skeleton or loading text per existing pattern
