Purpose: This is the core enforcement layer (LIMIT-02, LIMIT-03). Without these checks, plan limits are purely cosmetic. The checks must be server-side to prevent bypass.
Output: Modified route.ts files for bookings, employees, and services with limit checks integrated.

  <files>apps/web/app/api/v1/bookings/route.ts</files>
  <action>
    Modify `apps/web/app/api/v1/bookings/route.ts` to add limit enforcement:

    1. Add import at the top of the file:
    ```typescript
    import { checkBookingLimit, incrementBookingCounter } from '@/lib/usage/usage-service';
    ```

    2. In the POST handler, AFTER `findCompanyId(userSub)` resolves `companyId` (currently line 61) and BEFORE the user record lookup (line 64), add the limit check:
    ```typescript
    // Check booking limit for company's plan tier
    await checkBookingLimit(companyId);
    ```
    This will throw AppError(402) if the limit is exceeded, and `handleRouteError` in the route-handler middleware will convert it to the proper JSON error response. No additional catch logic needed.

    3. AFTER the successful `createBooking()` call (currently line 76-85) and BEFORE the return statement, increment the Redis counter:
    ```typescript
    // Increment booking counter for usage tracking (fire-and-forget, non-blocking)
    incrementBookingCounter(companyId).catch((err) => {
      console.error('[UsageCounter] Failed to increment booking counter:', err);
    });
    ```
    Use fire-and-forget (`.catch()`) so the counter increment does not block or fail the booking response. If Redis is down, the booking still succeeds (fail-open).

    The final POST handler flow becomes:
    1. `findCompanyId(userSub)` — resolve company
    2. `checkBookingLimit(companyId)` — 402 if exceeded
    3. DB lookup for user record
    4. `createBooking(...)` — actual booking creation
    5. `incrementBookingCounter(companyId)` — fire-and-forget
    6. Return 201

    IMPORTANT: Do NOT modify the GET handler. Only the POST handler needs changes.
    IMPORTANT: Do NOT remove or change any existing imports, validation, or business logic. Only ADD the two function calls and the import.
