    Import `PLAN_CONFIG` and `SubscriptionPlan` from `@schedulebox/shared` (these are created by Phase 28 in `packages/shared/src/types/billing.ts`). If Phase 28 has not yet been executed when this plan runs, create a temporary inline fallback that matches the same structure — but prefer the import.

    ```typescript
    import { PLAN_CONFIG, type SubscriptionPlan } from '@schedulebox/shared';

    export interface PlanLimits {
      maxBookingsPerMonth: number;
      maxEmployees: number;
      maxServices: number;
    }

    export function getLimitsForPlan(plan: SubscriptionPlan): PlanLimits {
      const config = PLAN_CONFIG[plan];
      return {
        maxBookingsPerMonth: config.features.maxBookingsPerMonth,
        maxEmployees: config.features.maxEmployees,
        maxServices: config.features.maxServices,
      };
    }

    export function isUnlimited(limit: number): boolean {
      return !isFinite(limit);
    }
    ```

    2. Create `apps/web/lib/usage/usage-service.ts`:

    This file provides all usage counting and limit checking functions used by both the API routes (Plan 29-02) and the usage endpoint.

    **Redis booking counter functions:**
    ```typescript
    import { redis } from '@/lib/redis/client';
    import { eq, and, isNull, sql } from 'drizzle-orm';
    import { db, employees, services, companies } from '@schedulebox/database';
    import { AppError } from '@schedulebox/shared';
    import { getLimitsForPlan, isUnlimited, type PlanLimits } from './plan-limits';
    import type { SubscriptionPlan } from '@schedulebox/shared';

    // Redis key format: usage:bookings:{companyId}:{YYYY-MM}
    function bookingCounterKey(companyId: number, period: string): string {
      return `usage:bookings:${companyId}:${period}`;
    }

    function getCurrentPeriod(): string {
      const now = new Date();
      return `${now.getFullYear()}-${String(now.getMonth() + 1).padStart(2, '0')}`;
    }

    function getSecondsUntilEndOfMonth(): number {
      const now = new Date();
      const endOfMonth = new Date(now.getFullYear(), now.getMonth() + 1, 1);
      return Math.ceil((endOfMonth.getTime() - now.getTime()) / 1000);
    }
    ```

    **incrementBookingCounter(companyId: number): Promise<number>**
    - Uses `redis.incr(bookingCounterKey(companyId, getCurrentPeriod()))`
    - On first increment (result === 1), sets TTL to `getSecondsUntilEndOfMonth()` via `redis.expire()`
    - Returns the new count after increment
    - On Redis error, log and return 0 (fail-open, same pattern as rate-limit.ts)

    **getBookingCount(companyId: number): Promise<number>**
    - Uses `redis.get(bookingCounterKey(companyId, getCurrentPeriod()))`
    - Returns `parseInt(result, 10) || 0`
    - On Redis error, fallback to DB count query: `SELECT COUNT(*) FROM bookings WHERE company_id = ? AND created_at >= start_of_month`

    **getEmployeeCount(companyId: number): Promise<number>**
    - DB query: `SELECT COUNT(*) FROM employees WHERE company_id = companyId AND deleted_at IS NULL`
    - Use Drizzle: `db.select({ count: sql<number>`count(*)` }).from(employees).where(and(eq(employees.companyId, companyId), isNull(employees.deletedAt)))`
    - Return the count value

    **getServiceCount(companyId: number): Promise<number>**
    - Same pattern as getEmployeeCount but against `services` table
    - `db.select({ count: sql<number>`count(*)` }).from(services).where(and(eq(services.companyId, companyId), isNull(services.deletedAt)))`

    **getCompanyPlan(companyId: number): Promise<SubscriptionPlan>**
    - Query: `db.select({ plan: companies.subscriptionPlan }).from(companies).where(eq(companies.id, companyId)).limit(1)`
    - Return `result.plan ?? 'free'` (cast to SubscriptionPlan)

    **Limit check functions (throw AppError 402 if exceeded):**

    **checkBookingLimit(companyId: number): Promise<void>**
    - Get plan via `getCompanyPlan(companyId)`
    - Get limits via `getLimitsForPlan(plan)`
    - If `isUnlimited(limits.maxBookingsPerMonth)`, return immediately
    - Get current count via `getBookingCount(companyId)`
    - If `current >= limits.maxBookingsPerMonth`, throw:
      `new AppError('PLAN_LIMIT_EXCEEDED', 'Monthly booking limit reached (${current}/${limits.maxBookingsPerMonth})', 402, { resource: 'bookings', current, limit: limits.maxBookingsPerMonth, plan, upgradeUrl: '/settings/billing' })`

    **checkEmployeeLimit(companyId: number): Promise<void>**
    - Same pattern: get plan, get limits, get count, throw 402 if `count >= limits.maxEmployees`
    - Error details: `{ resource: 'employees', current: count, limit: limits.maxEmployees, plan, upgradeUrl: '/settings/billing' }`

    **checkServiceLimit(companyId: number): Promise<void>**
    - Same pattern for services: throw 402 if `count >= limits.maxServices`
    - Error details: `{ resource: 'services', current: count, limit: limits.maxServices, plan, upgradeUrl: '/settings/billing' }`

    **getUsageSummary(companyId: number): Promise<UsageSummary>**
    ```typescript
    export interface UsageItem {
      resource: string;
      current: number;
      limit: number;
      unlimited: boolean;
      percentUsed: number;  // 0-100, capped at 100
      warning: boolean;     // true when >= 80%
    }

    export interface UsageSummary {
      plan: SubscriptionPlan;
      period: string;  // YYYY-MM
      items: UsageItem[];
    }
    ```
    - Get plan, limits, and all three counts in parallel (Promise.all)
    - For each resource, compute `percentUsed = unlimited ? 0 : Math.round((current / limit) * 100)`
    - Set `warning = !unlimited && percentUsed >= 80`
    - Return the summary object

    IMPORTANT: The `sql` template tag usage for count queries should be:
    ```typescript
    const [result] = await db
      .select({ count: sql<number>`cast(count(*) as int)` })
      .from(employees)
      .where(and(eq(employees.companyId, companyId), isNull(employees.deletedAt)));
    return result?.count ?? 0;
    ```

    ```typescript
    /**
     * GET /api/v1/usage
     * Returns current usage consumption vs tier limits for the authenticated company.
     * Used by the dashboard usage widget to show progress bars and warnings.
     */
    import { createRouteHandler } from '@/lib/middleware/route-handler';
    import { findCompanyId } from '@/lib/db/tenant-scope';
    import { successResponse } from '@/lib/utils/response';
    import { getUsageSummary } from '@/lib/usage/usage-service';

    export const GET = createRouteHandler({
      requiresAuth: true,
      handler: async ({ user }) => {
        const userSub = user?.sub ?? '';
        const { companyId } = await findCompanyId(userSub);
        const usage = await getUsageSummary(companyId);
        return successResponse(usage);
      },
    });
    ```

    No specific permissions required beyond authentication — any authenticated user in the company can view usage stats. This matches the dashboard access pattern.
