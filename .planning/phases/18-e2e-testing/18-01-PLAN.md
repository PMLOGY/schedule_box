  <name>Task 1: Install Playwright and create configuration</name>
  <files>

  <action>
1. Install Playwright as a dev dependency in the web package:
   `pnpm --filter @schedulebox/web add -D @playwright/test`

2. Install browser binaries:
   `npx playwright install --with-deps chromium firefox webkit`

3. Add E2E scripts to `apps/web/package.json`:
   - `"test:e2e": "playwright test --config e2e/playwright.config.ts"`
   - `"test:e2e:ui": "playwright test --config e2e/playwright.config.ts --ui"`

4. Add root-level E2E script to root `package.json`:
   - `"test:e2e": "pnpm --filter @schedulebox/web test:e2e"`

5. Create `apps/web/e2e/playwright.config.ts`:
   - Import `defineConfig, devices` from `@playwright/test`
   - `testDir: './tests'`
   - `fullyParallel: true`
   - `forbidOnly: !!process.env.CI`
   - `retries: process.env.CI ? 2 : 0`
   - `workers: process.env.CI ? 1 : undefined`
   - Reporter: CI uses `[['html', { open: 'never' }], ['github']]`, local uses `[['html', { open: 'on-failure' }]]`
   - `use.baseURL: 'http://localhost:3000'`
   - `use.trace: 'on-first-retry'`
   - `use.screenshot: 'only-on-failure'`
   - `use.video: 'retain-on-failure'`
   - Setup project: `{ name: 'setup', testMatch: /.*\.setup\.ts/ }`
   - Chromium project: `devices['Desktop Chrome']`, storageState `path.join(__dirname, 'playwright/.auth/user.json')`, dependencies: ['setup']
   - Firefox project: `devices['Desktop Firefox']`, same storageState, dependencies: ['setup']
   - WebKit project: `devices['Desktop Safari']`, same storageState, dependencies: ['setup']
   - webServer: `{ command: 'pnpm start', url: 'http://localhost:3000', reuseExistingServer: !process.env.CI, timeout: 120_000, env: { NODE_ENV: 'test' } }`
   - `outputDir: './test-results'`

6. Create `apps/web/e2e/auth.setup.ts`:
   - Import `test as setup, expect` from `@playwright/test`
   - Define `authFile = path.join(__dirname, 'playwright/.auth/user.json')`
   - Setup named 'authenticate as test owner':
     - `page.goto('/login')` — handles locale redirect automatically (localePrefix: 'as-needed', defaultLocale: 'cs')
     - Fill email with `test@example.com` using `page.getByLabel(/email/i)` or `page.locator('input[type="email"]')`
     - Fill password with `password123` using `page.locator('input[type="password"]')`
     - Click submit button using `page.getByRole('button', { name: /prihlasit|sign in|submit/i })`
     - Wait for dashboard: `page.waitForURL('**/')` then `expect(page.locator('body')).not.toContainText('login')`
     - Save state: `page.context().storageState({ path: authFile })`

7. Create `apps/web/e2e/helpers/test-data.ts`:
   - Export `TEST_OWNER = { email: 'test@example.com', password: 'password123', name: 'Test Owner' }`
   - Export `ADMIN_USER = { email: 'admin@schedulebox.cz', password: 'password123' }`
   - Export `NEW_USER = { name: 'E2E Test User', email: 'e2e-test-${Date.now()}@example.com', password: 'E2eTest123!@#', companyName: 'E2E Test Company' }` (function that generates unique user per test)
   - Export `MOCK_SERVICE = { id: 1, uuid: 'svc-001', name: 'Strih damsky', duration_minutes: 30, price: '500', currency: 'CZK', category_id: 1, is_active: true }`
   - Export `MOCK_EMPLOYEE = { id: 1, uuid: 'emp-001', name: 'Jana Novakova' }`

8. Update `.gitignore` to add:
   ```
   # Playwright
   apps/web/e2e/playwright/.auth/
   apps/web/e2e/test-results/
   apps/web/e2e/playwright-report/
   ```

  <action>
1. Create `apps/web/e2e/page-objects/login.page.ts` — LoginPage class:
   - Constructor takes `Page` from Playwright
   - Locators: `emailInput = page.locator('input[type="email"]')`, `passwordInput = page.locator('input[type="password"]')`, `submitButton = page.getByRole('button', { name: /prihlasit|sign in|submit/i })`, `errorMessage = page.locator('.text-destructive')`, `registerLink = page.getByRole('link', { name: /registr/i })`
   - Methods: `goto()` navigates to `/login`, `login(email, password)` fills and submits, `getErrorMessage()` returns error text
   - Note: The login form uses `useTranslations('auth.login')` so button text depends on locale. Use regex patterns that match Czech (`Prihlasit`), Slovak, and English (`Sign in`).

2. Create `apps/web/e2e/page-objects/register.page.ts` — RegisterPage class:
   - Constructor takes `Page`
   - Locators based on register-form.tsx: name input (`input[type="text"]` first one or by label), email input, company name input, password input (first `input[type="password"]`), confirmPassword input (second `input[type="password"]`), submit button
   - Use `page.getByLabel()` with i18n-safe regex patterns matching field labels
   - Methods: `goto()` navigates to `/register`, `register(data: { name, email, companyName, password })` fills all 5 fields and submits, `getErrorMessage()`, `getSuccessMessage()` reads the green success div
   - Password requirements: min 12 chars, uppercase + lowercase + number + special char

3. Create `apps/web/e2e/page-objects/booking-wizard.page.ts` — BookingWizardPage class:
   - Constructor takes `Page`
   - Locators: step indicator, service cards (`[data-testid="service-card"]` or by text), next button (`page.getByRole('button', { name: /next|dalsi|pokracovat/i })`), back button, confirm button (`page.getByRole('button', { name: /confirm|potvrdit/i })`), date picker, time slots, customer info fields (name, email, phone, notes)
   - Methods:
     - `goto()` navigates to `/bookings/new`
     - `selectService(name: string)` clicks service by text
     - `selectDateTime(date: string, time: string)` interacts with date picker and time slot
     - `fillCustomerInfo(data: { name, email, phone, notes? })` fills step 3 fields
     - `confirm()` clicks confirm button
     - `getCurrentStep()` reads step indicator
   - The wizard has 4 steps: 1=Service, 2=DateTime, 3=CustomerInfo, 4=Confirmation
   - Each step likely has a "Next" button to proceed

4. Create `apps/web/e2e/page-objects/dashboard.page.ts` — DashboardPage class:
   - Constructor takes `Page`
   - Locators: sidebar nav items, main content area, booking stats, quick actions
   - Methods: `goto()` navigates to `/`, `isVisible()` checks for dashboard-specific content, `navigateTo(section: string)` clicks sidebar links
   - Used primarily to verify successful login (dashboard loads)

5. Create `apps/web/e2e/helpers/mock-api.ts` — reusable `page.route()` interceptors:
   - `mockServicesAPI(page, services)` — intercepts `**/api/v1/services*` and returns provided service array
   - `mockEmployeesAPI(page, employees)` — intercepts `**/api/v1/employees*`
   - `mockAvailabilityAPI(page, slots)` — intercepts `**/api/v1/availability*` with time slots
   - `mockComgatePaymentCreate(page, response)` — intercepts `**/api/v1/payments/comgate/create` POST and returns mock redirect URL
   - `mockComgateRedirect(page, callbackUrl)` — intercepts `**/payments.comgate.cz/**` and redirects back to app callback
   - `mockAIServiceDown(page)` — intercepts `**/api/v1/ai/**` with 503 status to trigger circuit breaker fallback
   - `mockAIServiceHealthy(page, predictions)` — intercepts AI endpoints with success responses
   - Each function takes `page: Page` as first arg and returns `Promise<void>`
   - Export all functions as named exports

6. Create `apps/web/e2e/fixtures/auth.fixture.ts` — custom test fixture:
   - Extends base `test` from `@playwright/test`
   - Adds `authenticatedPage` fixture that uses storageState
   - Adds `unauthenticatedPage` fixture that has clean state (for testing login/register flows)
   - Export extended `test` and `expect`
