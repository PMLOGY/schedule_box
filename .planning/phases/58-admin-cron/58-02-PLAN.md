  <action>
Create POST /api/v1/cron/gdpr-cleanup endpoint that bulk-anonymizes customer records older than 3 years.

Implementation:

1. **Auth:** Use the same CRON_SECRET timing-safe comparison pattern from the expire-pending cron endpoint (see interfaces above). Return 401 if invalid, 503 if CRON_SECRET not configured.

2. **Query target customers:**
   ```sql
   SELECT id, uuid FROM customers
   WHERE created_at < NOW() - INTERVAL '3 years'
     AND deleted_at IS NULL
     AND email IS NOT NULL  -- skip already-anonymized
   LIMIT 500  -- batch size to prevent timeout
   ```
   Use `db.select()` with Drizzle's `lt()` and `isNull()` operators against `customers.createdAt` and `customers.deletedAt`. Filter where `customers.email` is not null (already-anonymized records have null email).

3. **Batch anonymize:** For each customer in the batch:
   - Update: set name to `Deleted User {uuid}`, email/phone/dateOfBirth/notes to null, marketingConsent to false, deletedAt to now, updatedAt to now
   - Delete all customerTags rows for that customer ID
   - Use a single bulk update if possible, otherwise iterate (matching the existing pattern from the individual anonymize endpoint)

   For efficiency, do the anonymization in a single SQL UPDATE for all matched IDs:
   ```typescript
   // Bulk update all matched customers
   await db.update(customers).set({
     name: sql`'Deleted User ' || uuid`,
     email: null, phone: null, dateOfBirth: null, notes: null,
     marketingConsent: false, deletedAt: new Date(), updatedAt: new Date(),
   }).where(inArray(customers.id, targetIds));

   // Bulk delete tag associations
   await db.delete(customerTags).where(inArray(customerTags.customerId, targetIds));
   ```

4. **Response:** Return JSON with `{ anonymized: number, timestamp: string }` and status 200.

5. **Logging:** Use `console.log('[CronGDPRCleanup] ...')` prefix consistent with other cron endpoints.

6. **Runtime config:** Set `export const runtime = 'nodejs';` and `export const maxDuration = 60;` (longer than other crons due to bulk operations).

7. **Error handling:** Wrap in try/catch, return 500 on failure with error message.

Import `crypto` from Node.js for timing-safe comparison. Import `db, customers, customerTags` from `@schedulebox/database`. Import `inArray, isNull, lt, and, isNotNull` from `drizzle-orm` and `sql` from `drizzle-orm`.
