  <name>Task 1: Create cohort retention API endpoint</name>
  <files>apps/web/app/api/v1/admin/cohort/route.ts</files>
  <action>
Create GET /api/v1/admin/cohort endpoint that returns company retention grouped by signup month.

The endpoint must:
1. Require admin role (same pattern as other admin routes: `if (!user || user.role !== 'admin') throw new ForbiddenError(...)`)
2. Query the companies table to build a cohort analysis:
   - Group companies by signup month (from `companies.createdAt`)
   - For each cohort month, calculate the percentage of companies still active (isActive=true, suspendedAt IS NULL) in each subsequent month up to the current month
   - A company is "active in month X" if it was created before or during month X AND (isActive=true AND suspendedAt IS NULL) — this is a snapshot-based approach since we don't have monthly login data; use current active status as the retention signal
3. Return format:
```json
{
  "data": {
    "cohorts": [
      {
        "month": "2026-01",
        "signups": 15,
        "retention": [100, 87, 73, 60]  // percentage retained in month 0, 1, 2, 3...
      }
    ],
    "months": ["2026-01", "2026-02", "2026-03", "2026-04"]  // column headers
  }
}
```
4. Limit to last 12 months of cohorts
5. Use raw SQL via `db.execute(sql\`...\`)` for the cohort pivot query — this is complex aggregation that Drizzle query builder handles poorly
6. Use `successResponse()` wrapper consistent with other admin endpoints

Since we only have current active/inactive status (no monthly activity logs), the retention calculation is: for each cohort, what % of signups are still active today. This gives a single "survived to now" rate per cohort rather than month-by-month decay. The UI will display this as a cohort table where each row is a signup month and shows total signups + current retention %.
