Define `GradientMeshPreset` type: `'dashboard' | 'marketing' | 'auth'` (matches the three gradient-mesh-* CSS classes defined in Phase 33's globals.css).

Define `GradientMeshProps` interface:
- `preset?: GradientMeshPreset` — defaults to `'dashboard'`
- `className?: string` — for additional non-stacking-context classes only

Create `GradientMesh` as a function component (NOT forwardRef — this is a decorative background div that is never referenced by parent code):

```typescript
function GradientMesh({ preset = 'dashboard', className }: GradientMeshProps) {
  return (
    <div
      className={cn('gradient-mesh', `gradient-mesh-${preset}`, className)}
      aria-hidden="true"
    />
  );
}
```

Export: `{ GradientMesh }` as named export. Also export `GradientMeshPreset` type.

CRITICAL CONSTRAINTS (from Phase 33 architecture and Decision 17):
- The `gradient-mesh` CSS class provides: `position: fixed; inset: 0; z-index: -10; pointer-events: none`
- Do NOT add any of these properties to the component (they are in CSS): `transform`, `filter`, `opacity`, `will-change`, `isolation`
- These properties create stacking contexts that break z-index layering for ALL dropdowns, popovers, and modals on the page
- `aria-hidden="true"` is required because this is a decorative background — screen readers must not announce it
- Do NOT use forwardRef — no parent component needs a ref to a background div
- Do NOT add `style` prop handling that could inject stacking context triggers
