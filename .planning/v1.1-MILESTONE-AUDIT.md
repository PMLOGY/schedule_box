### Broken Flow

**Comgate webhook failure → MON-03 alert:**
Webhook catch block increments Prometheus counter but never calls `markWebhookFailed()` (function doesn't exist). DB record stays at 'processing' status. MON-03 scheduler counts `status = 'failed'` which is always zero. The "stuck" detection (>5 min in processing) is the only DB-level failure signal, firing much later than real-time.

**Fix:** ~20-30 lines — add `markWebhookFailed()` call in catch block + webhook failure check in `handleMonitoringCheck`.
