## v1.2 Requirements

### AI Service — Training Pipeline and Model Deployment

- [x] **AI-01**: Internal training feature-extraction API routes (6 endpoints) protected by AI_SERVICE_API_KEY header validation
- [x] **AI-02**: No-show predictor trained with XGBoost on 500+ synthetic samples, producing confidence > 0.5 predictions
- [x] **AI-03**: CLV predictor trained with Random Forest, producing meaningful customer lifetime value scores
- [x] **AI-04**: Pricing optimizer Multi-Armed Bandit state persisted to Redis (survives Railway container restarts)
- [x] **AI-05**: Model versioning with .meta.json sidecars; startup validation raises RuntimeError on sklearn/xgboost version mismatch
- [x] **AI-06**: Weekly model retraining CI workflow (.github/workflows/train-models.yml) with manual dispatch option
- [x] **AI-07**: Railway AI service deployed with railway.toml, 1.5GB memory limit, Prophet startup warmup, health check passing within 30s
- [x] **AI-08**: CPU-bound ML inference runs in ThreadPoolExecutor (not blocking FastAPI async event loop)

### AI-Powered UI Surfaces

- [x] **AIUI-01**: No-show risk badge on booking management list rows (red >50%, yellow 30-50%, green <30%)
- [x] **AIUI-02**: No-show probability displayed on booking detail page with actionable label (not raw decimal)
- [x] **AIUI-03**: AI insights dashboard panel showing daily digest of high-risk bookings and optimization suggestions
- [x] **AIUI-04**: Confidence transparency UI — show "AI confidence: X%" when trained; show "Insufficient data" when confidence < 0.5
- [x] **AIUI-05**: AI onboarding state for new companies — "AI features activate after 10 bookings" with progress indicator

### Landing Page and Czech Legal Compliance

- [x] **LAND-01**: (marketing) route group with marketing layout (no AuthGuard, no sidebar, marketing navbar + footer)
- [x] **LAND-02**: Czech hero section with live embedded booking widget demo and "Zacit zdarma" primary CTA
- [x] **LAND-03**: Three-tier pricing page (Free / CZK 299 / CZK 699) with annual discount option
- [x] **LAND-04**: Feature grid (6 cards with icons) and trust badge row (GDPR, Czech hosting, Comgate, bank-level security)
- [x] **LAND-05**: Czech privacy policy (/cs/privacy) and terms of service (/cs/terms) pages
- [x] **LAND-06**: Strict opt-in cookie consent (no pre-checked boxes, Czech Electronic Communications Act 2022 compliant)
- [x] **LAND-07**: Footer with company ICO, DIC, registered address visible on every marketing page

### Booking UX Polish and Calendar Upgrade

- [x] **BUX-01**: Playwright visual regression baseline for embed widget established before any globals.css changes
- [x] **BUX-02**: react-big-calendar upgrade with day/week/month views, drag-and-drop rescheduling via react-dnd, shadcn theme
- [x] **BUX-03**: Smart time slot grouping by Morning/Afternoon/Evening (client-side, no API changes)
- [x] **BUX-04**: Mobile tap targets minimum 44px on calendar cells and booking flow buttons
- [x] **BUX-05**: Progress stepper ("Step X of Y") and skeleton loaders on slot fetch (not blank white screens)
- [x] **BUX-06**: Add-to-calendar ICS endpoint (/api/v1/bookings/[id]/calendar.ics) with RFC 5545 format
- [x] **BUX-07**: Micro-animations on booking confirmation (Motion 0.3s fade-in + scale on success icon)

### Onboarding and Business Setup Wizard

- [x] **ONB-01**: 4-step business setup wizard (Company details + logo, First service, Working hours, Share booking link)
- [x] **ONB-02**: "Your booking link is ready" moment with booking URL, QR code, and copy-to-clipboard with browser feedback
- [x] **ONB-03**: 5-item onboarding checklist dashboard widget (dismissible after completion)
- [x] **ONB-04**: Empty states with action prompts on every previously-blank table and list in the dashboard
- [x] **ONB-05**: Demo company data option ("Beauty Studio Praha" with 3 services, 10 bookings, 5 customers, clearly labeled)
- [x] **ONB-06**: Driver.js contextual tooltips on first visit to each dashboard section (never repeated)
