# Milestone v1.2: Product Readiness

**Status:** SHIPPED 2026-02-24
**Phases:** 23-27
**Total Plans:** 20 (19 planned + 1 gap closure)

## Overview

Make ScheduleBox a polished, demo-ready product with working AI models, professional UI, smooth workflows, and a landing page — so the sales team can confidently present it to SMB customers.

**Key insight:** ScheduleBox's AI features (no-show prediction, CLV scoring, dynamic pricing) are genuinely unique in the Czech/Slovak scheduling market. No competitor (Reservio, Bookio, Calendly) offers ML-based predictions.

## Milestone Summary

**Key Decisions:**
- AI first (unblocks UI), Landing Page before Booking UX (drives traffic), Onboarding last (layers on everything)
- GradientBoostingRegressor for capacity optimizer (not LSTM) — 80% accuracy at 10% complexity
- Models baked into Docker image (not R2); migrate to R2 if deploys exceed 5 minutes
- Landing page as (marketing) route group (not separate service)
- react-big-calendar replaces FullCalendar (MIT vs premium license for commercial SaaS)
- Motion for animations (not GSAP — commercial license), Driver.js for onboarding (not react-joyride)
- ICS generated with pure string templating (no external library)
- Static TypeScript const for industry template data (no DB round-trip)
- Demo data tagged in company.settings JSONB (no extra table)

**Issues Resolved:**
- FullCalendar premium license replaced with MIT react-big-calendar
- XGBoost deprecated parameter fixed in training scripts
- Prophet JSON serialization fixed for model persistence
- CompanySettings type extended to include onboarding fields

**Issues Deferred:**
- Real testimonials needed for landing page social proof (placeholder content in place)
- Lighthouse >90 score validation requires production deployment measurement

**Technical Debt Incurred:**
- Visual regression baselines need updating when globals.css changes
- Cookie consent uses localStorage (not server-side) for hydration safety
